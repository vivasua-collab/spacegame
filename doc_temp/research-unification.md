## 1. Проблема: два списка веток

В проекте исторически сложились **два разных списка** веток исследований:

### ARCHITECTURE.md §3.2.1 — Фундаментальные ветки

| # | Ветка | Фокус |
|---|-------|-------|
| 1 | **Химия** | Новые рецепты переработки, новые материалы |
| 2 | **Физика** | Энергетика, щиты, двигатели |
| 3 | **Инженерия** | Корпуса кораблей, модули, верфи |
| 4 | **Биология** | Габитабельность, терраформирование, питание колоний |
| 5 | **Военные науки** | Оружие, тактика, оборона |
| 6 | **Ксеноархеология** | Изучение артефактов Странников |

### 06-research.md §2.1 — Специализированные ветки

| # | Ветка | ID | Фокус |
|---|-------|----|-------|
| 1 | **Энергетика** | `power` | Двигатели, реакторы, щиты, варп |
| 2 | **Материаловедение** | `materials` | Сплавы, броня, конструкционные материалы, сверхпроводники |
| 3 | **Военные науки** | `weapons` | Оружие, боеприпасы, тактика, абордаж |
| 4 | **Вычислительные системы** | `computing` | Датчики, связь, ИИ, автоматизация, квантовые вычисления |
| 5 | **Биотехнологии** | `biology` | Терраформирование, генетика, криогенка, органика |
| 6 | **Ксеноархеология** | `xenoarch` | Артефакты, руины, инопланетные технологии, Предтечи |

### Расхождение

Списки **не идентичны** — фундаментальные ветки описывают **широкие области знаний**, а специализированные — **конкретные технологические направления**. Например:

- Фундаментальная «Химия» не имеет прямого аналога среди специализированных веток
- Специализированная «Вычислительные системы» отсутствует среди фундаментальных веток
- «Военные науки» и «Ксеноархеология» присутствуют в обоих списках, но с разным смыслом

**Решение:** ввести двухуровневую модель, где каждый уровень имеет чёткую роль.

## 2. Двухуровневая модель исследований

### 2.1 Определения

| Уровень | Что описывает | Роль | Документ-источник |
|---------|--------------|------|-------------------|
| **Фундаментальный** | Широкие области научного знания | Определяет, **какие специализированные ветки доступны** и на каком максимальном уровне | ARCHITECTURE.md §3.2.1 |
| **Специализированный** | Конкретные технологии с уровнями 1–10+ | Определяет **конкретные улучшения**, разблокировку зданий, модулей, рецептов | 06-research.md §2.1 |

### 2.2 Аналогия

```
Фундаментальный уровень  ≈  «Кафедра в университете»
Специализированный уровень ≈  «Конкретная специальность / курс»
```

- Фундаментальный уровень Химия (ур.5) → открывает доступ к специализированной ветке Материаловедение на макс. уровне 5
- Фундаментальный уровень Физика (ур.3) → открывает доступ к специализированной ветке Энергетика на макс. уровне 3

### 2.3 Принципы двухуровневой системы

| Принцип | Описание |
|---------|----------|
| **Фундаментальный уровень — ворота** | Без развития фундаментальной ветки специализированные недоступны |
| **Специализированный уровень — глубина** | Конкретные технологии исследуются внутри специализированных веток |
| **Фундаментальный ≠ сумма специализированных** | Фундаментальный уровень растёт отдельно, через инвестиции RP в фундаментальные исследования |
| **Ограничение потолка** | Максимальный уровень специализированной технологии ≤ уровень соответствующей фундаментальной ветки |
| **Перекрытие веток** | Одна специализированная ветка может зависеть от 1–2 фундаментальных (primary/secondary) |

## 3. Таблица соответствия: фундаментальные ↔ специализированные

### 3.1 Полная таблица маппинга

| Фундаментальная ветка | ID | Специализированная ветка (primary) | ID | Специализированная ветка (secondary/partial) | ID | Тип связи |
|----------------------|----|-----------------------------------|----|----------------------------------------------|----|-----------|
| **Химия** | `chemistry` | Материаловедение | `materials` | Вычислительные системы | `computing` | primary + partial |
| **Физика** | `physics` | Энергетика | `power` | — | — | primary |
| **Инженерия** | `engineering` | Материаловедение | `materials` | Военные науки | `weapons` | secondary + partial |
| **Биология** | `biology_fund` | Биотехнологии | `biology` | — | — | primary |
| **Военные науки** | `military_science` | Военные науки | `weapons` | — | — | primary |
| **Ксеноархеология** | `xenoarchaeology` | Ксеноархеология | `xenoarch` | — | — | primary |

### 3.2 Пояснения по маппингу

#### Химия → Материаловедение (primary), Вычислительные системы (partial)

- **Химия** — наука о веществах и их превращениях. Это фундамент для создания новых материалов (сплавы, полимеры, сверхпроводники) — то есть специализированной ветки **Материаловедение**.
- **Partial связь с Вычислительными системами:** химия полупроводников и фотохимия являются основой для микроэлектроники. Однако вычислительные системы опираются на химию лишь частично (на ранних этапах — кремниевые кристаллы, фоторезисты). Поэтому связь — **partial**, а не primary.

#### Физика → Энергетика (primary)

- **Физика** — наука о фундаментальных законах природы, энергии, полях. Прямой переход к **Энергетике**: реакторы, щиты, двигатели — всё это прикладная физика.
- Чистый primary-маппинг: для специализированных технологий Энергетики достаточно фундаментальной Физики.

#### Инженерия → Материаловедение (secondary), Военные науки (partial)

- **Инженерия** — наука о проектировании и конструировании. Она опирается на материаловедение (secondary): чтобы спроектировать корпус, нужно понимать свойства материала.
- **Partial связь с Военными науками:** инженерия проектирует системы оружия, но сама тактика и стратегия боя — это военная наука. Поэтому связь — partial.

#### Биология → Биотехнологии (primary)

- **Биология** — наука о живых системах. Прямой переход к **Биотехнологиям**: терраформирование, генетика, криогенка — всё это прикладная биология.
- Чистый primary-маппинг.

#### Военные науки → Военные науки (primary)

- **Военные науки** как фундаментальная дисциплина (стратегия, тактика, теория боя) переходит в специализированную ветку **Военные науки** (конкретные типы оружия и технологий).
- Чистый primary-маппинг, но с важным нюансом: специализированные Военные науки также частично опираются на Инженерию (конструирование оружия).

#### Ксеноархеология → Ксеноархеология (primary)

- **Ксеноархеология** как фундаментальная дисциплина (методология изучения инопланетных культур) переходит в специализированную ветку **Ксеноархеология** (конкретные артефакты и технологии Предтеч).
- Чистый primary-маппинг.

### 3.3 Обратный маппинг: от специализированных к фундаментальным

| Специализированная ветка | ID | Фундаментальная ветка (primary) | Фундаментальная ветка (secondary/partial) |
|--------------------------|----|--------------------------------|-------------------------------------------|
| **Энергетика** | `power` | Физика | — |
| **Материаловедение** | `materials` | Химия (primary) | Инженерия (secondary) |
| **Военные науки** | `weapons` | Военные науки (primary) | Инженерия (partial) |
| **Вычислительные системы** | `computing` | — | Химия (partial) |
| **Биотехнологии** | `biology` | Биология | — |
| **Ксеноархеология** | `xenoarch` | Ксеноархеология | — |

> **Важно:** Вычислительные системы не имеют primary-связи с какой-либо фундаментальной веткой. Это **междисциплинарная** специализированная ветка, доступная изначально, но получающая бонусы от уровня Химии. Подробнее — в §4.

## 4. Схема зависимостей: фундаментальный уровень → специализированные ветки

### 4.1 Правила доступа

| Правило | Описание |
|---------|----------|
| **Primary unlock** | Специализированная ветка становится доступна, когда соответствующая фундаментальная ветка достигает уровня 1 |
| **Primary ceiling** | Максимальный уровень технологий в специализированной ветке ≤ уровень primary фундаментальной ветки |
| **Secondary ceiling** | Если специализированная ветка имеет secondary-связь, то технологии выше `secondary_level × 1.5` требуют роста secondary-фундаментальной ветки |
| **Partial bonus** | Partial-связь даёт бонус к скорости исследования (+5% за уровень частичной фундаментальной ветки), но не ограничивает потолок |
| **Свободная ветка** | Специализированная ветка без primary-связи (Вычислительные системы) доступна изначально, без требований к фундаментальным веткам |

### 4.2 Таблица правил доступа по каждой специализированной ветке

#### Энергетика (`power`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | Физика |
| **Unlock** | Физика ≥ ур.1 |
| **Потолок технологий** | ≤ уровень Физики |
| **Secondary/Partial** | — |
| **Пример** | Физика ур.5 → можно исследовать Энергетику до ур.5 включительно |

#### Материаловедение (`materials`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | Химия |
| **Unlock** | Химия ≥ ур.1 |
| **Потолок технологий** | ≤ уровень Химии |
| **Secondary фундаментальная** | Инженерия |
| **Secondary-ограничение** | Технологии выше `floor(Инженерия × 1.5)` требуют Инженерию соответствующего уровня |
| **Пример** | Химия ур.7, Инженерия ур.3 → потолок = ур.7, но выше ур.4 (3×1.5) нужна Инженерия |

#### Военные науки (`weapons`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | Военные науки |
| **Unlock** | Военные науки ≥ ур.1 |
| **Потолок технологий** | ≤ уровень Военных наук |
| **Partial фундаментальная** | Инженерия |
| **Partial-бонус** | +5% к скорости исследований в ветке за каждый уровень Инженерии |
| **Пример** | Военные науки ур.6, Инженерия ур.4 → потолок = ур.6, бонус = +20% к RP/сек в ветке |

#### Вычислительные системы (`computing`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | — (свободная ветка) |
| **Unlock** | Доступна с начала игры |
| **Потолок технологий** | Не ограничен фундаментальными ветками |
| **Partial фундаментальная** | Химия |
| **Partial-бонус** | +5% к скорости исследований за каждый уровень Химии |
| **Пример** | Химия ур.5 → бонус = +25% к RP/сек в ветке Вычислительных систем |

#### Биотехнологии (`biology`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | Биология |
| **Unlock** | Биология ≥ ур.1 |
| **Потолок технологий** | ≤ уровень Биологии |
| **Secondary/Partial** | — |
| **Пример** | Биология ур.8 → можно исследовать Биотехнологии до ур.8 включительно |

#### Ксеноархеология (`xenoarch`)

| Параметр | Значение |
|----------|----------|
| **Primary фундаментальная** | Ксеноархеология |
| **Unlock** | Ксеноархеология ≥ ур.1 |
| **Потолок технологий** | ≤ уровень Ксеноархеологии |
| **Secondary/Partial** | — |
| **Пример** | Ксеноархеология ур.4 → можно исследовать спец. Ксеноархеологию до ур.4 |

### 4.3 Сводная таблица формул ограничения

```
max_tech_level(specialized_branch) =
  если есть primary:
    primary_fundamental_level
  иначе (свободная ветка):
    ∞ (без ограничения)

secondary_ceiling(specialized_branch) =
  если есть secondary:
    floor(secondary_fundamental_level × 1.5)
  иначе:
    ∞ (без ограничения)

effective_max_level(specialized_branch) =
  min(max_tech_level, secondary_ceiling)

partial_bonus(specialized_branch) =
  если есть partial:
    1.0 + 0.05 × partial_fundamental_level
  иначе:
    1.0
```

**Итоговая формула:**

```
effective_max = min(primary_level, floor(secondary_level × 1.5))
rp_multiplier = 1.0 + 0.05 × partial_level
```

## 5. Механика ограничения доступа к специализированным веткам

### 5.1 Фундаментальные исследования — как они растут

Фундаментальные ветки исследуются так же, как специализированные, но **не дают прямых игровых эффектов** — они только открывают и поднимают потолок специализированных веток.

| Параметр фундаментальной ветки | Значение |
|-------------------------------|----------|
| **Количество уровней** | 1–10 (как у специализированных) |
| **Стоимость уровня N** | `base_cost × 1.5^(N-1)` (аналогично) |
| **Базовая стоимость** | 200 RP (дешевле специализированных — фундаментальные исследования шире, но менее «осязаемы») |
| **Эффект** | Unlock + потолок для специализированных веток |
| **RP-источник** | Тот же пул RP от лабораторий |

> **Дизайнерское обоснование:** Фундаментальные исследования дешевле специализированных, потому что они «инвестиция в будущее» — не дают немедленных бонусов, но расширяют возможности. Игрок выбирает: тратить RP на фундамент (открывать новые ветки) или на специализированные (получать конкретные бонусы прямо сейчас).

### 5.2 Пример прогрессии

```
Ход 1: Игрок выделяет RP на Химию → Химия ур.1 → разблокировано Материаловедение (макс. ур.1)
Ход 2: Игрок инвестирует в Материаловедение → Сталеплавильное дело ур.1 (доступно, ≤ потолок)
Ход 3: Игрок продолжает Химию → Химия ур.2 → потолок Материаловедения поднят до ур.2
Ход 4: Игрок инвестирует в Инженерию → Инженерия ур.1 → secondary-ceiling для Материаловедения = floor(1×1.5) = 1
Ход 5: Игрок продолжает Химию → Химия ур.3 → потолок Материаловедения = min(3, floor(1×1.5)) = 1 (!)
       → Инженерия отстаёт, ограничивает рост Материаловедения!
Ход 6: Игрок инвестирует в Инженерию → Инженерия ур.2 → secondary-ceiling = floor(2×1.5) = 3 → потолок = min(3, 3) = 3
```

> **Ключевой дизайн-момент:** при наличии secondary-связи нельзя «прокачать» одну фундаментальную ветку и получить доступ к высоким уровням специализированной. Нужно развивать обе. Это создаёт **осмысленный выбор** — ограниченный бюджет RP нужно распределять между несколькими фундаментальными ветками.

| 2 | 1.5 | 300 | 500 |

### 7.3 Функции расчёта ограничений

```typescript
/**
 * Возвращает primary-связь для специализированной ветки.
 * Если нет — ветка свободная (доступна с начала игры).
 */
function getPrimaryLink(specializedId: SpecializedBranchId): BranchLink | undefined {
  return BRANCH_LINKS.find(l => l.specializedId === specializedId && l.linkType === 'primary');
}

/**
 * Возвращает secondary-связь для специализированной ветки.
 */
function getSecondaryLink(specializedId: SpecializedBranchId): BranchLink | undefined {
  return BRANCH_LINKS.find(l => l.specializedId === specializedId && l.linkType === 'secondary');
}

/**
 * Возвращает все partial-связи для специализированной ветки.
 */
function getPartialLinks(specializedId: SpecializedBranchId): BranchLink[] {
  return BRANCH_LINKS.filter(l => l.specializedId === specializedId && l.linkType === 'partial');
}

/**
 * Проверяет, разблокирована ли специализированная ветка.
 * Ветка разблокирована, если primary-фундаментальная ≥ ур.1, или если нет primary-связи (свободная).
 */
function isSpecializedUnlocked(
  specializedId: SpecializedBranchId,
  fundamentalLevels: Record<FundamentalBranchId, number>,
): boolean {
  const primary = getPrimaryLink(specializedId);
  if (!primary) return true; // Свободная ветка
  return (fundamentalLevels[primary.fundamentalId] ?? 0) >= 1;
}

/**
 * Вычисляет максимальный доступный уровень технологии
 * в специализированной ветке.
 */
function getEffectiveMaxLevel(
  specializedId: SpecializedBranchId,
  fundamentalLevels: Record<FundamentalBranchId, number>,
): number {
  const primary = getPrimaryLink(specializedId);
  const secondary = getSecondaryLink(specializedId);

  // Потолок от primary (если нет — ∞)
  const primaryCeiling = primary
    ? (fundamentalLevels[primary.fundamentalId] ?? 0)
    : Infinity;

  // Потолок от secondary (если нет — ∞)
  const secondaryCeiling = secondary
    ? Math.floor((fundamentalLevels[secondary.fundamentalId] ?? 0) * 1.5)
    : Infinity;

  return Math.min(primaryCeiling, secondaryCeiling);
}

/**
 * Вычисляет множитель скорости исследования для специализированной ветки
 * на основе partial-связей.
 */
function getPartialBonus(
  specializedId: SpecializedBranchId,
  fundamentalLevels: Record<FundamentalBranchId, number>,
): number {
  const partials = getPartialLinks(specializedId);
  let bonus = 1.0;
  for (const link of partials) {
    bonus += 0.05 * (fundamentalLevels[link.fundamentalId] ?? 0);
  }
  return bonus;
}
```

### 7.4 Пример использования

```typescript
// Текущие уровни фундаментальных веток
const fundLevels: Record<FundamentalBranchId, number> = {
  chemistry: 5,
  physics: 3,
  engineering: 2,
  biology_fund: 4,
  military_science: 3,
  xenoarchaeology: 1,
};

// Проверяем Материаловедение (materials):
// primary = Химия (ур.5), secondary = Инженерия (ур.2)
getEffectiveMaxLevel('materials', fundLevels);
// → min(5, floor(2 × 1.5)) = min(5, 3) = 3

// Проверяем Военные науки (weapons):
// primary = Военные науки (ур.3), partial = Инженерия (ур.2)
getEffectiveMaxLevel('weapons', fundLevels);
// → min(3, ∞) = 3 (partial не ограничивает)
getPartialBonus('weapons', fundLevels);
// → 1.0 + 0.05 × 2 = 1.10 (+10% к скорости)

// Проверяем Вычислительные системы (computing):
// primary = нет (свободная), partial = Химия (ур.5)
getEffectiveMaxLevel('computing', fundLevels);
// → min(∞, ∞) = ∞ (нет ограничений)
getPartialBonus('computing', fundLevels);
// → 1.0 + 0.05 × 5 = 1.25 (+25% к скорости)

// Проверяем Энергетика (power):
// primary = Физика (ур.3)
getEffectiveMaxLevel('power', fundLevels);
// → min(3, ∞) = 3
```

## Приложение Б: Глоссарий

| Термин | Определение |
|--------|------------|
| **Фундаментальная ветка** | Широкая область научного знания (Химия, Физика и т.д.), определяющая доступ к специализированным веткам |
| **Специализированная ветка** | Конкретное технологическое направление (Энергетика, Материаловедение и т.д.) с деревом технологий |
| **Primary-связь** | Основная зависимость: фундаментальная ветка разблокирует специализированную и ограничивает потолок |
| **Secondary-связь** | Дополнительная зависимость: вторая фундаментальная ветка ограничивает потолок специализированной |
| **Partial-связь** | Частичная зависимость: фундаментальная ветка даёт бонус к скорости, но не ограничивает доступ |
| **Свободная ветка** | Специализированная ветка без primary-связи, доступная с начала игры |
| **Потолок (ceiling)** | Максимальный уровень технологий в специализированной ветке, ограниченный фундаментальными уровнями |
