## Текущий статус проекта

### Что работает ✅
- Генерация галактики (500 систем, спиральная структура, PRNG)
- BakedGalaxyModel (57 элементов, ~50 руд, запекание + lookup-структуры)
- Экономический движок (добыча → переработка → производство, энергия = поток)
- Склад ресурсов (базовый 10000, специализация, резервы, орбитальный буфер)
- UI: карта галактики, система, планета (гекс-сетка), панель склада
- Сохранение/загрузка (SQLite/Prisma)
- Колонизация планет

### Критические проблемы 🔴
1. **Расхождение ID руд**: recipes.ts использует `'Fe-ore'`, но chemistry-generator генерирует другие ID через `elementToOre`. Рецепты крафта могут не находить входные ресурсы.
2. **Нет UI для атмосферы/орбиты**: `buildOnAtmosphereSlot()` и `buildOnOrbitSlot()` существуют в engine.ts, но BuildingDialog не позволяет строить на этих слотах.
3. **Прямая мутация состояния**: game-store.ts мутирует `planet.resources[...]` напрямую, затем делает `set({ gameState: { ...gameState } })`. Поверхностный клон — `systemMap`, `productionQueues` и вложенные объекты не клонируются.

### Средние проблемы 🟡
4. **ResourcePanel не показывает крафтовые материалы**: steel, superconductor, microchip и др. попадают в «Прочие» без категории.
5. **Нет UI для очереди производства**: `enqueueProduction()` работает, но игрок не может добавить рецепт через интерфейс.
6. **Colony Hub бесплатный на всех уровнях**: `costPerLevel: {}` — апгрейд хаба не стоит ресурсов.
7. **Тип `transuranic` объявлен, но элементов нет**: ElementCategory включает 'transuranic', но elements.ts содержит только U (который в 'rare').

### Недостающие подсистемы ⚪
8. **Флот**: типы Ship/Fleet/FleetOrder определены, но 0 реализации
9. **Исследования**: документы есть, кода нет
10. **AI-фракция**: документы есть, кода нет
11. **Здания для атмосферы/орбиты**: нет UI для строительства на атмосферных и орбитальных слотах

## Детальный план P1-P7 (ближайшие задачи)

### P1: Унификация ID руд
**Проблема:** recipes.ts использует ID типа `'Fe-ore'`, а chemistry-generator создаёт ID через формулу минерала (например, `'hematite'`). Это приводит к тому, что рецепты крафта не могут найти ресурсы.

**Решение:**
1. Прочитать `getCurrentLookups().elementToOre` для получения маппинга elementId → oreId
2. Проверить, какие recipe.inputs ссылаются на руды
3. Создать функцию `getRecipeOreId(elementId)` которая использует baked-lookups
4. Обновить recipes.ts: заменить хардкод на динамический lookup
5. Создать скрипт валидации: для каждого рецепта проверить, что все inputs существуют в baked model

### P2: Immutable-паттерн в store
**Проблема:** Прямая мутация `planet.resources[Fe] += 10` + `set({ gameState: { ...gameState } })` — поверхностный клон. Вложенные объекты (systemMap, planets) разделяют ссылки со старым состоянием.

**Решение:**
1. Установить `immer` (уже есть в зависимостях zustand-immer)
2. Обернуть create() в immer-мидлвар: `create<GameStore>()(immer((set, get) => ...))`
3. Переписать все `set()` вызовы на прямую мутацию (immer автоматически создаст иммутабельный клон)
4. Убрать ручные `{ ...gameState }` клоны

### P3: UI — строительство на атмосфере/орбите
**Решение:**
1. Расширить BuildingDialog: добавить tab для Surface / Atmosphere / Orbit
2. Для атмосферы: показывать AtmosphericSlot, фильтровать здания с `layer: ['atmosphere']`
3. Для орбиты: показывать OrbitalSlot, фильтровать здания с `layer: ['orbit']`
4. Вызвать `buildOnAtmosphereSlot()` / `buildOnOrbitSlot()` из engine.ts

### P4: UI — очередь производства
**Решение:**
1. Новая вкладка «Производство» на экране планеты (рядом с «Карта» / «Ресурсы»)
2. Список доступных рецептов, сгруппированных по зданию (processor, synthesizer, refinery, shipyard)
3. Кнопка «Добавить в очередь» → `enqueueProduction()`
4. Визуализация текущей очереди: прогресс-бар, оставшееся время, автоповтор
5. Возможность отмены элемента из очереди

### P5: ResourcePanel — крафтовые материалы
**Решение:**
1. Добавить `'crafted'` в ElementCategory (types.ts)
2. Добавить CATEGORY_LABELS['crafted'] = 'Синтезированные' и CATEGORY_COLORS
3. Создать CRAFTED_MATERIALS map: `{ steel: { name: 'Сталь', category: 'crafted' }, ... }`
4. В ResourcePanel: проверять CRAFTED_MATERIALS если элемент не найден в ELEMENT_MAP

### P6: Colony Hub — стоимость апгрейда
**Решение:**
1. Добавить `costPerLevel: { Fe: 10, Si: 5, Al: 3 }` для colony_hub
2. Обновить описание здания

### P7: Убрать мёртвый тип transuranic
**Решение:**
1. Добавить трансурановые элементы: Np (Нептуний), Pu (Плутоний), Am (Америций)
2. Или убрать 'transuranic' из ElementCategory, если считаем что U достаточно
