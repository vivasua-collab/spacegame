**Дата:** 2026-05-03 16:07 MSK
**Фаза:** 1
**Статус:** in_progress

## Цель этапа

Создать работающее **ядро симуляции** — консольный/минимальный прототип, где можно:
1. Запустить генерацию галактики (10 тестовых систем × 3 планеты)
2. Построить шахту на планете → получать ресурс
3. Переработать ресурс в материал
4. Собрать корабль из модулей
5. Сохранить/загрузить состояние

---

## Способ реализации

### Архитектура: Разделение движка и UI

Ядро симуляции (game engine) — **чистый TypeScript**, независимый от React/UI. Запускается в Web Worker для изоляции от рендеринга. Связь с UI через postMessage/events.

## Детальный план подзадач

### 1.1 Тиковая система (Game Loop)

**Что:** Игровой цикл с настраиваемой скоростью.

**Реализация:**
- Класс `GameLoop` с методами: `start()`, `pause()`, `setSpeed(x1|x5|x15|x50)`
- Один тик = 1 игровая секунда
- При x1: 1 тик/сек, x5: 5 тиков/сек, x15: 15 тиков/сек, x50: 50 тиков/сек
- Активная пауза: во время паузы можно отдавать приказы
- `EventBus` для подписки на события тика (`onTick`, `onDay`, `onMonth`)

**Файлы:** `src/core/game-loop/tick.ts`, `src/core/game-loop/time-control.ts`, `src/core/game-loop/event-bus.ts`

### 1.2 Seed-based генератор галактики

**Что:** Процедурная генерация галактики с воспроизводимостью.

**Реализация:**
- PRNG `xoshiro256**` (быстрый, детерминированный)
- Seed → под-seeds для каждого этапа генерации
- Генерация 10 тестовых систем для MVP (параметризуемо до 2000)
- Непрерывные координаты (x, y) в парсеках
- Спиральная структура (для MVP — простой вариант: равномерное распределение в диске)
- Точки перехода (JP) — граф связности между системами

**Файлы:** `src/core/galaxy/generator.ts`, `src/core/galaxy/prng.ts`, `src/core/galaxy/jump-points.ts`

### 1.3 Звёздные системы

**Что:** Модель звезды + её планетная система.

**Реализация:**
- Типы звёзд: O, B, A, F, G, K, M + спецтипы (белый карлик, нейтронная, чёрная дыра)
- Каждая звезда определяет: массу, светимость, температуру, радиус
- Количество планет = f(тип звезды): M-звезда → 1-4, G-звезда → 2-8, O-звезда → 0-3
- Астероидные поля: 0-2 на систему

**Файлы:** `src/core/galaxy/star.ts`, `src/core/galaxy/system.ts`

### 1.4 Планеты

**Что:** Модель планеты с параметрами и гекс-сеткой.

**Реализация:**
- 7 типов: Rocky, Volcanic, Ice, Oceanic, Desert, GasGiant, Dwarf
- Параметры: radius, gravity, temperature, atmosphere, habitability, magnetic_field, life
- Гекс-сетка застройки: axial-координаты (q, r)
  - Крошечная: 19 гексов, Малая: 37, Средняя: 61, Большая: 91, Огромная: 127
- Типы местности гексов: plains, mountains, desert, ice, ocean, volcano, jungle
- Ресурсные залежи: element → { availability, quantity, depth }

**Файлы:** `src/core/galaxy/planet.ts`, `src/core/galaxy/hex-grid.ts`, `src/core/galaxy/resources.ts`

### 1.6 Система зданий (MVP: 8 зданий)

**Здания MVP:**

### 1.7 Рецепты крафта (MVP)

**Что:** Минимальный набор рецептов для замкнутого цикла.

**Уровень 1: Сырьё → Материалы**
- Fe-руда (10) + Энергия(2) → Fe (8)
- Si-руда (10) + Энергия(2) → Si (7)
- Ti-руда (10) + Энергия(3) → Ti (7)
- Стальной сплав = Fe(10) + C(3) + Энергия(5)
- Титановый сплав = Ti(8) + Al(4) + Энергия(8)
- Пластик = C(5) + H(8) + Энергия(2)
- Кремниевый кристалл = Si(6) + Энергия(10)
- Сверхпроводник = Y(0.5) + Ba(1) + Cu(2) + O(3) + Энергия(15)

**Уровень 2: Материалы → Компоненты**
- Микрочип = Кремниевый кристалл(1) + Au(0.1) + Энергия(5)
- Корпусной элемент = Стальной сплав(5) + Энергия(3)
- Бронеплита = Титановый сплав(3) + W(1) + Энергия(5)
- Двигательная секция = Стальной сплав(3) + Сверхпроводник(1) + Энергия(10)

**Уровень 3: Компоненты → Модули кораблей**
- Ионный двигатель = Двигательная секция(2) + Сверхпроводник(1) + Энергия(12)
- Лазерная установка = Микрочип(2) + Корпусной элемент(1) + Энергия(8)
- Грузовой отсек = Корпусной элемент(3) + Энергия(2)
- Сканер = Микрочип(1) + Энергия(3)

**Уровень 4: Модули → Корабль**
- Разведчик = Корпус(лёгкий)(1) + Ионный двигатель(1) + Сканер(1)
- Транспорт = Корпус(средний)(1) + Ионный двигатель(1) + Грузовой отсек(3)

**Файлы:** `src/data/recipes.json`, `src/core/economy/recipes.ts`

### 1.8 Система энергии

**Что:** Баланс производства/потребления энергии.

**Реализация:**
- Каждая планета имеет `energy_balance = energy_production - energy_consumption`
- Если balance < 0: здания отключаются по приоритету
- Приоритеты: электростанции → добыча → переработка → производство → верфи → лаборатории
- Энергия не хранится (нет аккумуляторов в MVP) — моментальный баланс

**Файлы:** `src/core/economy/energy.ts`

### 1.9 Сохранение/загрузка

**Что:** Полное сохранение состояния игры в SQLite.

**Реализация:**
- Prisma schema: Game (seed, settings), System, Planet, Building, Fleet, Ship, Resource
- Сохранение: serialize всего GameState → INSERT/UPDATE в SQLite
- Загрузка: SELECT → deserialize → GameState
- Автосохранение каждые 60 игровых дней

**Файлы:** `prisma/schema.prisma`, `src/core/save/serializer.ts`, `src/core/save/deserializer.ts`, `src/lib/db.ts`

