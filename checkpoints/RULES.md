# Система чекпоинтов разработки SpaceGame

> Дата создания: 2026-05-03
> Репозиторий: https://github.com/vivasua-collab/spacegame

## Цель

Фиксация этапов разработки проекта SpaceGame — однопользовательской космической 4X-стратегии.

---

## 🗂️ Формат имени файла

```
ММ_ДД_цель.md
```

- **ММ** — месяц (01-12)
- **ДД** — день (01-31)
- **цель** — краткое описание этапа или номер фазы

### Примеры

- `05_03_phase0_complete.md` — завершена фаза 0 (документация)
- `05_03_galaxy_core.md` — чекпоинт ядра галактики
- `05_10_save_system.md` — система сохранений
- `05_15_combat_mvp.md` — MVP боевой системы

---

## 📝 Содержимое чекпоинта

Каждый файл содержит:

```markdown
# Чекпоинт: [Название]

**Дата:** YYYY-MM-DD HH:MM
**Фаза:** X
**Статус:** [in_progress | complete | blocked]

## Выполненные задачи
- Задача 1
- Задача 2

## Текущие задачи
- Задача 3

## Проблемы
- Описание проблем (если есть)

## Следующие шаги
- Что делать дальше

## Изменённые файлы
- путь/к/файлу.ts
```

## 📦 Вынос кода в отдельный файл

Если чекпоинт содержит код (аудит текущего кода, сниппеты, примеры реализации, листинги классов) — код выносится в отдельный файл, а в основном чекпоинте остаётся только планирование и результаты.

### Правила

- **Основной файл** (`ММ_ДД_цель.md`) — планирование, решения, результаты, ссылки на код
- **Файл кода** (`ММ_ДД_цель_code.md`) — вся кодовая база: аудиты, сниппеты, примеры реализации
- Между файлами — взаимные ссылки:
  - В основном файле: `👉 Кодовая база: [ММ_ДД_цель_code.md](ММ_ДД_цель_code.md)`
  - В файле кода: `👉 Основной план: [ММ_ДД_цель.md](ММ_ДД_цель.md)`
- Файл кода наследует дату и статус от основного чекпоинта

### Когда создавать отдельный файл

- ✅ Аудит существующего кода (листинги файлов)
- ✅ Сниппеты реализации (код, который будет написан)
- ✅ Примеры/шаблоны классов и методов
- ✅ Конфигурации, формулы в кодовом представлении
- ❌ Короткие inline-примеры (1-3 строки) — можно оставить в основном файле
- ❌ Псевдокод без привязки к конкретным файлам — можно оставить

### Формат файла кода

```markdown
# 📦 Кодовая база: [Название]

👉 Основной план: [ММ_ДД_цель.md](ММ_ДД_цель.md)

**Дата:** YYYY-MM-DD HH:MM
**Статус:** [in_progress | complete | blocked]

## Аудит текущего кода
...

## Сниппеты реализации
...
```

## 🔄 Логика создания чекпоинта

1. Получить системное время — `DateTime.Now` / `date`
2. Сформировать имя — `{MM}_{DD}_{цель}.md`
3. Записать состояние — список выполненных задач
4. Зафиксировать файлы — какие файлы изменены/созданы
5. Вынести код — если есть кодовые листинги, создать `_code.md` файл и добавить ссылки

## Статусы

| Статус | Описание |
|--------|----------|
| `in_progress` | Активно разрабатывается |
| `complete` | Завершён |
| `blocked` | Заблокирован (ждёт зависимости) |
