## 1. Справка об оригинальной игре

**«Полдень XXI века» (21noon.com)** — браузерная многопользовательская космическая стратегия в реальном времени, запущенная в 2010 году. Название отсылает к повести братьев Стругацких «Полдень, XXII век».

### Установленные механики оригинала

| Аспект | Описание |
|--------|----------|
| **Сеттинг** | Альтернативная реальность: земляне исчерпали ресурсы планеты и расселились по галактике |
| **Пути развития** | Колонизатор, Торговец, Пират — выбор не фиксирован, можно комбинировать |
| **Ресурсы** | 60 видов полезных ископаемых, 50 элементов из таблицы Менделеева |
| **Производство** | Корабли собираются из деталей, а детали — из элементов или более простых компонентов (многоуровневый крафт) |
| **Здания** | Шахты, заводы, верфи, электростанции, лаборатории |
| **Наука** | Ключевая механика — высокий уровень науки позволяет конструировать более мощные корабли |
| **Энергия** | Один из основных ресурсов, вырабатывается электростанциями, нужна для всего производства |
| **Корабли** | Модульная система: корпус, оружие, броня, двигатели, грузовые отсеки |
| **Бой** | Флотские сражения + абордаж (сложнее, но выгоднее — позволяет захватить груз) |
| **Инопланетяне** | Две расы: **Триниты** (понимают только силу) и **Странники** (таинственные, встречаются как заброшенные корабли-источники артефактов) |
| **Артефакты** | Добываются при исследовании космоса и встрече с инопланетянами, дают научный рывок |
| **Колонизация** | Неограниченное количество колоний; домашняя планета защищена от нападений |
| **Экономика** | Внутренняя валюта, торговля между игроками, торговые караваны (можно грабить) |
| **Сложность** | «Категорически противопоказана казуалам» — глубокая и сложная система |

### Что нужно адаптировать для синглплеера

| Механика оригинала | Проблема в сингле | Решение |
|---------------------|-------------------|---------|
| Тысячи живых игроков | Нет социума | AI-фракции (2–4) с экономикой, дипломатией, войной |
| Торговля между игроками | Нет рынка | AI-торговцы с динамическим спросом/предложением |
| Нападения на караваны игроков | Нет караванов | AI-торговые маршруты между системами |
| Глобальная конкурентная экономика | Нет конкуренции | AI-фракции конкурируют за ресурсы и системы |
| Ожидание реального времени | Скучно одному | Ускорение времени (x1, x5, x15, x50) + активная пауза |
| Союзники/враги — живые люди | Нет социума | Дипломатия с AI-фракциями |

## 2. Общая концепция (однопользовательская адаптация)

### Жанр
Однопользовательская песочница / стратегия в реальном времени с возможностью паузы и ускорения времени.

### Цель игры
Достичь одной из победных условий (научная, экономическая, военная, дипломатическая) в своём темпе, либо играть бесконечно в режиме песочницы.

### Ключевые отличия от оригинала
1. **AI-фракции (2–4)** вместо живых игроков — развиваются, торгуют, воюют
2. **Процедурная галактика** (200–1000 звёздных систем) — уникальная при каждом seed
3. **Локальная экономика** — спрос и предложение формируются AI-колониями
4. **Ускорение времени** — x1/x5/x15/x50 + активная пауза (отдача приказов во время паузы)
5. **Условия победы** — научная, экономическая, военная, дипломатическая
6. **Сохранение в любой момент** — никаких «_real-time ожиданий»

## 3. Игровые модули

### Модуль 1: Галактика и звёздные системы

#### 3.1.1 Генератор галактики
- **Seed + процедурная генерация** — воспроизводимые результаты
- Алгоритм «рукава галактики» (как Elite Dangerous / Stellaris)
- **200–1000 звёздных систем** (настраиваемо при старте)
- Типы звёзд: жёлтый карлик (G), красный карлик (M), белый карлик, красный гигант, голубой гигант, нейтронная звезда
- Количество планет в системе: 1–12 (зависит от типа звезды)
- Расстояния между системами — для расчёта времени перемещения флотов

#### 3.1.2 Планеты
Параметры каждой планеты:
- **Радиус** (определяет вместимость построек)
- **Гравитация** (0.1–3.0 g, влияет на типы построек)
- **Атмосфера** (нет / тонкая / плотная / токсичная / инертная)
- **Температура** (определяет тип поверхности и доступные ресурсы)
- **Тип поверхности**: скалистая, вулканическая, ледяная, газовый гигант, океаническая, пустынная
- **Габитабельность** (0–100%) — влияет на скорость работы колонистов
- **Ресурсный потенциал** — какие элементы и в каком количестве добываются

#### 3.1.3 Ресурсы
Система из **50 элементов** (как в оригинале), сгруппированных по распространённости:

| Категория | Примеры | Распространённость |
|-----------|---------|--------------------|
| Повсеместные | H, C, O, Si, Fe, Al, Ca, Na, K, Mg | Почти на каждой планете |
| Распространённые | Ti, Cr, Mn, Ni, Cu, Zn, Sn, Pb, S, P | ~50% планет |
| Редкие | Co, W, Mo, V, Ag, Pt, Au, U, Li, Be | ~15% планет |
| Уникальные | Ir, Os, Re, Ru, Rh, Pd, Hf, Ta, Nb, Zr | ~3% планет |

> **Важно**: не пытаться реализовать все 118 элементов сразу. Начать с 20 ключевых, остальные добавлять по мере развития.

Привязка к типам планет:
- **Вулканические**: Fe, Ti, Cr, Ni, W, S — много тяжёлых металлов и серы
- **Скалистые**: Si, Al, O, C, Ca — стандартный набор для строительства
- **Ледяные**: H, O (вода), He, N — лёткие элементы и вода
- **Газовые гиганты**: H, He, C, N — добыча из атмосферы (спец. оборудование)
- **Океанические**: H, O, Na, Cl, Mg — морские ресурсы
- **Пустынные**: Si, Fe, Al, Ti — эрозия обнажает залежи

#### 3.1.4 Постройки на планетах

| Постройка | Функция | Улучшения |
|-----------|---------|-----------|
| **Шахта** | Добыча элементов из залежей планеты | Уровень 1–10 (скорость добычи) |
| **Завод** | Переработка элементов в компоненты по рецептам | Уровень 1–10 (скорость + сложные рецепты) |
| **Верфь** | Строительство кораблей и модулей | Уровень 1–10 (размер + тип кораблей) |
| **Электростанция** | Выработка энергии для всех построек | Уровень 1–10 (тип: солнечная / ядерная / термоядерная) |
| **Лаборатория** | Исследование технологий | Уровень 1–10 (скорость + уровень технологий) |
| **Склад** | Хранение элементов и компонентов | Уровень 1–10 (вместимость) |
| **Сканер** | Обнаружение ресурсов в системе | Уровень 1–5 (радиус и точность) |
| **Оборонительная платформа** | Защита планеты от атак | Уровень 1–10 (урон + щит) |
| **Торговый хаб** | Автоматическая торговля с AI-фракциями | Уровень 1–5 (объём сделок) |

> **Оригинальная механика**: здания «запоминают» последнее задание и повторяют его автоматически — **сохранить в адаптации**.

### Модуль 2: Экономика и производство

#### 3.2.1 Древо технологий
**Ветви исследований**:

1. **Химия** — новые рецепты переработки, новые материалы
2. **Физика** — энергетика, щиты, двигатели
3. **Инженерия** — корпуса кораблей, модули, верфи
4. **Биология** — габитабельность, Terraforming, питание колоний
5. **Военные науки** — оружие, тактика, оборона
6. **Ксеноархеология** — изучение артефактов Странников

> Каждая технология имеет **уровень** (1–10) и **зависимости** от других технологий. Изучение занимает игровое время и потребляет ресурсы.

#### 3.2.2 Рецепты (многоуровневый крафт)
Как в оригинале — **корабли из деталей, детали из элементов или более простых компонентов**:

**Уровень 1: Сырьё → Материалы**
- Стальной сплав = Fe(10) + C(3) + Энергия(5)
- Титановый сплав = Ti(8) + Al(4) + V(1) + Энергия(8)
- Пластик = C(5) + H(8) + Энергия(2)
- Кремниевый кристалл = Si(6) + Энергия(10)
- Сверхпроводник = Y(1) + Ba(2) + Cu(3) + O(4) + Энергия(15)

**Уровень 2: Материалы + Сырьё → Компоненты**
- Микрочип = Кремниевый кристалл(1) + Au(0.1) + Энергия(5)
- Корпусной элемент = Стальной сплав(5) + Энергия(3)
- Бронеплита = Титановый сплав(3) + W(1) + Энергия(5)
- Двигательная секция = Стальной сплав(3) + Сверхпроводник(1) + Энергия(10)

**Уровень 3: Компоненты → Модули кораблей**
- Лазерная установка = Микрочип(2) + Корпусной элемент(1) + Энергия(8)
- Щитовой генератор = Сверхпроводник(3) + Микрочип(1) + Энергия(15)
- Ионный двигатель = Двигательная секция(2) + Сверхпроводник(1) + Энергия(12)
- Грузовой отсек = Корпусной элемент(3) + Энергия(2)

**Уровень 4: Модули → Корабль**
- Разведчик = Корпус(лёгкий)(1) + Ионный двигатель(1) + Сканер(1)
- Истребитель = Корпус(лёгкий)(1) + Ионный двигатель(1) + Лазер(2)
- Транспорт = Корпус(средний)(1) + Ионный двигатель(1) + Грузовой отсек(3)
- Крейсер = Корпус(тяжёлый)(1) + Ионный двигатель(2) + Лазер(4) + Щит(1)

#### 3.2.3 Производственные линии
- **Очередь производства** — здания автоматически обрабатывают задания по порядку
- **Повтор последнего задания** — как в оригинале, здание запоминает и повторяет
- **Управление складами** — лимиты по видам элементов, приоритеты хранения
- **Цепочки поставок** — автоматическая передача ресурсов между планетами (транспортными кораблями)

#### 3.2.4 Торговля с AI
- **Генерация запросов**: AI-планеты формируют спрос на дефицитные элементы
- **Динамические цены**: зависят от соотношения спрос/предложение в системе
- **Торговые маршруты**: AI-флот перемещается между системами (можно перехватить!)
- **Торговые соглашения**: долгосрочные контракты с AI-фракциями

### Модуль 3: Космические корабли

#### 3.3.1 Конструктор кораблей
Модульная система (как в оригинале):

| Модуль | Влияние на характеристики |
|--------|--------------------------|
| **Корпус** (лёгкий/средний/тяжёлый/сверхтяжёлый) | HP, слоты для модулей, масса |
| **Двигатель** (химический/ионный/фотонный/варп) | Скорость, дальность, потребление энергии |
| **Щиты** (нет/лёгкие/средние/тяжёлые) | Дополнительный HP, потребление энергии |
| **Оружие** (лазер/плазма/ракеты/пушка Гаусса) | Урон, дальность, потребление энергии |
| **Грузовой отсек** | Вместимость для торговли/добычи |
| **Сканер** | Дальность обнаружения в системе |
| **Модуль поддержки** (ремонт/добыча/колонизация) | Спец. возможности |

> **Масса влияет на скорость и дальность** — тяжёлый корабль с слабым двигателем летает медленнее.

#### 3.3.2 Управление флотом
- **Группировка кораблей** во флоты (до 100 кораблей на флот)
- **Приказы**: лететь, атаковать, добывать, торговать, патрулировать, колонизировать
- **Маршруты**: цепочки точек для автоматического патрулирования
- **Очередь приказов** — выполнение задач последовательно

#### 3.3.3 Бой
**Для MVP**: расчётный (авторасчёт) бой — столкновение флотов, учёт характеристик, случайность.

**В будущем**: тактический бой (как в Star Control / FTL) — управление группами кораблей с паузой.

Факторы боя:
- Соотношение огневой мощи и защиты
- Эффективность оружия против щитов/брони
- Скорость и манёвренность
- Мораль (потери приводят к отступлению)
- Абордаж — отдельная мини-механика (шанс захвата корабля + груза)

### Модуль 4: AI-фракции

#### 3.4.1 Состав фракций (3–4 на старте)

| Фракция | Архетип | Бонусы | Поведение |
|---------|---------|--------|-----------|
| **Конфедерация** | Сбалансированная | +10% производство | Равномерное развитие, дипломатичны |
| **Хегемония** | Военная | +15% боевая мощь | Агрессивная экспансия, мало торгуют |
| **Синдикат** | Торговая | +20% торговые доходы | Торговля, наёмники, нейтралитет |
| **Орден** | Научная | +25% скорость исследований | Быстрые технологии, слабый флот |

> **Инопланетяне**: Триниты и Странники — не фракции, а «среда обитания»:
> - Триниты — враги по умолчанию, атакуют любого рядом, можно найти и уничтожить их базы
> - Странники — находки заброшенных кораблей с артефактами (случайные события при исследовании)

#### 3.4.2 Экономический AI
- Каждая фракция владеет 5–20 системами (при старте)
- **Потребности**: AI-планетам нужны элементы для построек (как игроку)
- **Производство**: автоматическая добыча и переработка по формуле
- **Дефицит/избыток**: формирует торговые запросы
- **Упрощённая модель**: для удалённых систем — прирост ресурсов по формуле (не посимвольно)

#### 3.4.3 Военный AI
- **Формирование флотов**: зависит от стратегии фракции
- **Атака**: при превосходстве сил в секторе
- **Защита**: патрули собственных систем
- **Расширение**: колонизация свободных систем

#### 3.4.4 Дипломатия
- **Отношения**: -100 (война) до +100 (союз)
- **Действия**: мир, торговое соглашение, военный союз, обмен технологиями, ультиматум, объявление войны
- **Факторы**: сила армии, близость границ, прошлые действия, идеологические различия
- **События**: просьбы о помощи, угрозы, предложения обмена

### Модуль 5: Интерфейс (UI/UX)

#### 3.5.1 Карта галактики
- **Зум от общей карты до отдельных планет** (как в Stellaris / EVE)

- Список кораблей с параметрами

#### 3.5.4 Технологическое дерево
- Графическое дерево с подсветкой изученного/доступного/заблокированного

- Прогресс текущего исследования

#### 3.5.5 Дипломатический экран
- Отношения с каждой фракцией
- Активные соглашения
- Журнал событий и предложений

## 5. Техническая архитектура

### 5.1 Ключевые технические вызовы

| Вызов | Решение |
|-------|---------|
| Сложность экономики (50 элементов, сотни рецептов) | Начать с 20 элементов, расширять постепенно |
| Баланс в однопользовательской игре | Ускорение времени (x1/x5/x15/x50) + активная пауза |
| AI, который торгует осмысленно | Система голод/избыток: AI покупает недостающее |
| Производительность при 1000+ систем | Событийная модель: активные системы — посекундно, удалённые — по формуле |
| Управление сотнями кораблей | Менеджер очередей и автоматические маршруты |
| Рендеринг большой карты галактики | LOD (Level of Detail) + кластеризация при зуме |

### 5.2 Архитектурные принципы

1. **Разделение движка и UI** — ядро симуляции не зависит от интерфейса
2. **Событийная модель** — изменения распространяются через события, а не опрос
3. **Тиковая система** — игровой цикл с настраиваемой скоростью (тик = 1 игровая секунда)
4. **Слоистая симуляция**:
   - Активная зона (рядом с игроком) — полная симуляция
   - Видимая зона — упрощённая симуляция
   - Дальняя зона — модель прироста по формуле
5. **Сохранение состояния** — полный дамп + инкрементальные сохранения

**Минусы**:
- Производительность ниже чем у нативных движков

## 9. Производительность: стратегия оптимизации

### 9.1 Симуляция
- **Активная зона** (3 системы вокруг игрока): полный тик каждые 100мс при x1
- **Видимая зона** (10 систем): упрощённый тик каждые 1с
- **Дальняя зона**: модель прироста по формуле, обновление раз в 10с
- **Web Worker**: симуляция в отдельном потоке, связь через postMessage

### 9.2 Рендеринг карты
- PixiJS с WebGL рендерером
- LOD: при зуме далеко — точки (кластеризованные), при зуме близко — полноценные иконки
- Спрайт-пулинг для тысяч звёзд
- OffscreenCanvas для тяжёлых вычислений

### 9.3 Память
- Ленивая загрузка систем (не держать все 1000 систем в памяти)
- Частичные обновления (только изменённые сущности)
- Сжатие сохранений

## Приложение B: Что было исправлено/дополнено по сравнению с черновиком

1. **Уточнено количество элементов**: оригинал использовал 50 элементов (не 118), плюс 60 видов полезных ископаемых — это разные сущности. Элементы — чистые химические элементы, полезные ископаемые — природные смеси (руды, минералы).

2. **Добавлены инопланетяне**: Триниты и Странники из оригинала — это не фракции, а элемент среды. Триниты — враги для охоты, Странники — источник артефактов.

3. **Уточнена система строительства**: здания запоминают последнее задание и повторяют его автоматически — важная QoL-механика оригинала.

4. **Добавлен абордаж**: в оригинале можно было брать корабли на абордаж — это сложнее, но выгоднее обычного уничтожения.

5. **Уточнены пути развития**: в оригинале нет выбора рас/классов — развитие определяется действиями игрока. Колонизатор/торговец/пират — это не классы, а стили игры.

6. **Добавлена домашняя планета**: защищена от нападений — важная механика для новичков.

7. **Уточнена экономика**: внутренняя валюта + торговля между игроками (в сингле — с AI) + торговые караваны (можно грабить).

8. **Упрощён первоначальный набор элементов**: начать с 20 ключевых, а не пытаться сразу внедрить все 50.

9. **Добавлены условия победы**: в оригинале цели не были формализованы (игра была бесконечной MMO). Для сингла нужны чёткие условия победы.

10. **Выбран рекомендуемый стек**: Вариант A (Next.js + PixiJS + Web Worker) как оптимальный для среды разработки и скорости итераций.
