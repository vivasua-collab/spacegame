## 1. Форма и структура галактики

### 1.1. Основной вариант: спиральная галактика

Основной тип генерируемой галактики — **спиральная галактика с рукавами**, аналогичная Млечному Пути. Этот тип обеспечивает естественное распределение звёздных систем с выраженными макроструктурами (рукава, перемычка, ядро), что создаёт разнообразный стратегический ландшафт.

#### Структура спиральной галактики

```
                    ┌─────────────────────────────┐
                    │        Внешний ореол         │
                    │   (редкие звёзды, пустоты)   │
                    │                              │
                    │    ╲               ╱         │
                    │     ╲  Рукав 1   ╱           │
                    │      ╲         ╱             │
                    │       ╲       ╱              │
                    │        ╲     ╱               │
                    │  Рукав 3 │●│ Рукав 2        │
                    │  (←––)   │ │   (––→)        │
                    │        ╱ │Я│ ╲              │
                    │       ╱  │Д│   ╲             │
                    │      ╱   │Р│    ╲            │
                    │     ╱    │О│     ╲           │
                    │    ╱     └─┘      ╲          │
                    │   Рукав 4          Рукав 5   │
                    │                              │
                    │        Дисковая зона         │
                    └─────────────────────────────┘

    ● Ядро — максимальная плотность звёзд
    Рукава — средняя плотность, вытянутые структуры
    Ореол — минимальная плотность
```

#### Компоненты спиральной галактики

| Компонент            | Описание                                                         | Доля звёзд |
|----------------------|------------------------------------------------------------------|------------|
| **Ядро** (bulge)     | Центральная область высокой плотности, содержит перемычку (bar)  | ~15%       |
| **Рукава** (arms)    | Спиральные структуры, исходящие из ядра; основная масса звёзд    | ~60%       |
| **Диск** (disk)      | Пространство между рукавами; пониженная, но ненулевая плотность  | ~20%       |
| **Ореол** (halo)     | Внешняя область с редкими звёздами и глобулярными скоплениями    | ~5%        |

#### Алгоритм генерации спиральной галактики

Генерация использует логарифмические спирали с вариациями для каждого рукава:

1. **Определение центра** — точка (0, 0) в парсеках.
2. **Генерация рукавов** — каждый рукав описывается логарифмической спиралью:
   ```
   r(θ) = a · e^(b·θ)
   ```
   где `a` — начальный радиус, `b` — параметр закрутки, `θ` — угол.
3. **Распределение звёзд вдоль рукавов** — для каждой системы:
   - Выбирается случайный угол `θ` в диапазоне рукава.
   - Вычисляется радиус `r(θ)`.
   - Добавляется гауссово смещение `(Δx, Δy)` перпендикулярно рукаву (σ = `arm_width`).
4. **Ядро** — звёзды распределяются по 2D-гауссиане с центром (0, 0), σ = `bulge_radius`.
5. **Диск** — равномерное распределение в пределах `galaxy_radius` с пониженной плотностью.
6. **Ореол** — редкие точки за пределами основного диска, сгруппированные в глобулярные скопления.

### 1.2. Seed-based процедурная генерация

Все случайные процессы генерации используют детерминированный генератор псевдослучайных чисел (PRNG) с заданным **seed**.

#### Принципы

- **Воспроизводимость**: один и тот же seed + параметры → идентичная галактика.
- **Пошаговая генерация**: каждый этап генерации (рукава, ядро, объекты, точки перехода) использует свой под-seed, получаемый хешированием основного seed + идентификатора этапа.
- **Локальность изменений**: изменение параметров одного этапа не влияет на другие (если основной seed не изменён).

#### Цепочка под-seed'ов

```
main_seed
├── arm_seed       = hash(main_seed, "arms")       → позиции рукавов
├── bulge_seed     = hash(main_seed, "bulge")      → ядро
├── disk_seed      = hash(main_seed, "disk")       → диск между рукавами
├── halo_seed      = hash(main_seed, "halo")       → ореол
├── nebula_seed    = hash(main_seed, "nebulae")    → туманности
├── gas_seed       = hash(main_seed, "gas")        → газовые скопления
├── asteroid_seed  = hash(main_seed, "asteroids")  → астероидные пояса
├── void_seed      = hash(main_seed, "voids")      → пустоты
├── relic_seed     = hash(main_seed, "relics")     → реликты
├── jp_seed        = hash(main_seed, "jump_points")→ точки перехода
└── system_seed[i] = hash(main_seed, "sys", i)     → содержимое i-й системы
```

#### Рекомендуемый PRNG

**xoshiro256** — быстрый, высококачественный, с возможностью jump-ahead для параллельной генерации. Seed — 64-битное целое число.

### 1.3. Настраиваемые параметры при старте игры

При создании новой игры игрок задаёт параметры галактики через экран настроек:

| Параметр              | Диапазон    | По умолчанию | Описание                                      |
|-----------------------|-------------|--------------|-----------------------------------------------|
| Количество систем     | 100–2000    | 500          | Общее число звёздных систем в галактике       |
| Количество рукавов    | 2–6         | 4            | Число спиральных рукавов                      |
| Плотность ядра        | 0.5–3.0     | 1.5          | Множитель плотности в ядре относительно нормы  |
| Ширина рукава         | 0.2–1.0     | 0.5          | Стандартное отклонение разброса звёзд от оси рукава (в кпк) |
| Закрутка рукава       | 0.2–1.5     | 0.6          | Параметр `b` логарифмической спирали          |
| Радиус галактики      | 10–50       | 25           | В килопарсеках (кпк)                          |
| Доля диска            | 0.1–0.4     | 0.2          | Доля звёзд вне рукавов, но внутри диска       |
| Доля ореола           | 0.0–0.15    | 0.05         | Доля звёзд во внешнем ореоле                  |

### 1.4. Альтернативные формы галактики

Для разнообразия поддерживаются другие морфологические типы:

#### 1.4.1. Эллиптическая галактика (E)

```
           ·  · ·  ·
        ·  · · · · ·  ·
      ·   · · ·●· · ·   ·
        ·  · · · · ·  ·
           ·  · ·  ·
```

- Звёзды распределены по 2D-гауссиане с возможной эллиптичностью.
- Нет рукавов, нет выраженной структуры.
- Плотность плавно убывает от центра к краю.
- **Стратегический эффект**: отсутствие естественных «коридоров» — война ведётся по всем направлениям, нет узких горл.

#### 1.4.2. Неправильная галактика (Irr)

```
        · ·     · ·
      ·  · ·  ·    ·
         ·  · ·
      · ·    · ·  ·
           ·    ·
        ·    · ·   ·
```

- Случайное распределение кластеров звёзд без общей закономерности.
- Генерация: Poisson-disk sampling с переменной плотностью + несколько случайных кластеров.
- **Стратегический эффект**: непредсказуемая география, изолированные «острова» цивилизаций.

#### 1.4.3. Линзовидная галактика (S0)

```
         ═════════════
        ═══════════════
       ═══════●═════════
        ═══════════════
         ═════════════
```

- Переходный тип между спиральной и эллиптической.
- Ярко выраженный диск (линза) с ядром, но **без рукавов**.
- Генерация: диск — равномерное распределение в эллипсе, ядро — гауссиана.
- **Стратегический эффект**: плоская «арена», удобна для симметричного старта фракций.

#### Сводная таблица форм галактики

| Тип        | Код | Рукава | Структура         | Стратегическое разнообразие |
|------------|-----|--------|--------------------|-----------------------------|
| Спиральная | S   | Да     | Рукава + ядро      | Высокое                     |
| Эллиптическая | E | Нет    | Гауссиана          | Среднее                     |
| Линзовидная| S0  | Нет    | Диск + ядро        | Низкое–среднее              |
| Неправильная | Irr | Нет  | Случайные кластеры | Высокое (непредсказуемость) |

## 2. Сетка галактики

### 2.1. Непрерывное 2D-пространство

Галактика представлена **непрерывным 2D-пространством** с декартовыми координатами. Тайловая сетка **не используется**.

#### Обоснование отказа от тайловой сетки

| Критерий             | Тайловая сетка             | Непрерывное пространство       |
|----------------------|----------------------------|--------------------------------|
| Реалистичность       | Искусственные ограничения  | Естественные расстояния        |
| Гибкость размещения  | Привязка к узлам сетки     | Произвольные координаты        |
| Масштабируемость     | Увеличение сетки = рост памяти | Координаты не зависят от разрешения |
| Вычисление расстояний | Манхэттен/Чебышёв         | Евклидово расстояние           |
| Визуализация         | Квадраты/гексы             | Свободное позиционирование     |

### 2.2. Координаты звёздных систем

Каждая звёздная система имеет координаты `(x, y)`, выраженные в **парсеках** (пк).

#### Система координат

```
  y (пк)
  ▲
  │
  │     ·  · ·
  │   ·    ★ (12.3, 45.7)
  │  · ·    ·
  │    ·  ·
  │      ·
  └──────────────────► x (пк)
  0,0 = центр галактики
```

- **Начало координат** (0, 0) — центр галактики (ядро).
- **Ось X** — горизонтальная (правое направление положительное).
- **Ось Y** — вертикальная (верхнее направление положительное).
- **Диапазон**: зависит от радиуса галактики. При радиусе 25 кпк: примерно `[-25000, 25000]` пк по каждой оси.

#### Хранение координат

Координаты хранятся как числа с плавающей точкой (Float64 / `double`). Точность 0.01 пк (0.01 парсека ≈ 206 а.е.) достаточна для всех игровых расчётов.

### 2.3. Расстояния между системами

Расстояние между двумя системами рассчитывается как **евклидово расстояние**:

```
d(A, B) = √((Ax - Bx)² + (Ay - By)²)  [парсек]
```

Для оптимизации частых запросов расстояний используется **пространственный индекс** (k-d tree или R-tree), позволяющий быстро находить соседей в радиусе без полного перебора всех систем.

#### Влияние расстояния на геймплей

| Механика              | Зависимость от расстояния                              |
|-----------------------|--------------------------------------------------------|
| Время перелёта        | Пропорционально расстоянию (с учётом скорости двигателя) |
| Обнаружение систем    | Датчики имеют радиус обнаружения в парсеках           |
| Связь                 | Задержка сигнала или мгновенная (по настройке)        |
| Снабжение колоний     | Логистический диапазон от ближайшей базы              |

### 2.4. Способы перемещения между системами

В игре поддерживаются **два способа** межзвёздного перемещения. Выбор способа — параметр настройки игры при старте.

#### 2.4.1. Точки перехода (Jump Points)

Модель, аналогичная Aurora 4X. Между определёнными парами систем существуют **точки перехода** (jump points) — естественные разломы пространства-времени.

```
  Система A ════════════════════════ Система B
       ║                               ║
       ║    JP-A ═══════════ JP-B      ║
       ║      (точка перехода)         ║
       ║                               ║
  Система C ──────── нет JP ───── Система D
       (прямого пути нет, нужен маршрут A→B→E→D)
```

**Характеристики:**

| Параметр                      | Значение / Описание                                  |
|-------------------------------|------------------------------------------------------|
| Расположение                  | На окраине звёздной системы (50–500 а.е. от звезды)  |
| Парность                      | Каждый JP имеет парный JP в другой системе           |
| Визуальность                  | JP обнаруживается датчиками; точка выхода — нет      |
| Время перехода                | Мгновенно (или несколько игровых часов)              |
| Требования к кораблю          | Нужен прыжковый двигатель (jump engine)              |
| Неизвестность                 | Неизвестно, куда ведёт JP, пока не пройдён           |

**Преимущества модели:**
- Создаёт **граф связности** — ключевые системы (узлы) получают стратегическое значение.
- Исследование: нужно физически прибыть к JP, чтобы обнаружить его и узнать пункт назначения.
- Тактика: контроль «горл» и узловых систем критичен для обороны.

**Недостатки модели:**
- Ограничивает свободу перемещения — нельзя лететь куда угодно.
- Требует тщательной генерации графа для обеспечения связности.

#### 2.4.2. Варп-двигатели (Warp Drive)

Модель свободного перемещения. Корабль с варп-двигателем может лететь **из любой системы в любую** по прямой.

```
  Система A ════════════════════════ Система B
       ╲                               ╱
        ╲                             ╱
         ╲                           ╱
          ╲                         ╱
           ╲                       ╱
            ╲                     ╱
             Система C ───────── Система D
```

**Характеристики:**

| Параметр                      | Значение / Описание                                  |
|-------------------------------|------------------------------------------------------|
| Маршрут                       | Прямолинейный от системы к системе                   |
| Время перелёта                | d / v_warp (расстояние / скорость варпа)             |
| Ограничения                   | Максимальная дальность за один прыжок (warp_range)   |
| Топливо                       | Расход пропорционален расстоянию                     |
| Перехват                      | Возможен в глубоком космосе при сближении            |

**Преимущества модели:**
- Полная свобода перемещения — стратегия определяется расстояниями и логистикой.
- Простота понимания для игрока.

**Недостатки модели:**
- Нет «естественных» горл — оборона сложнее, нужно прикрывать все направления.
- Граф связности — полный, что снижает стратегическую глубину.

#### 2.4.3. Рекомендуемый вариант: гибридная модель

**Рекомендуется гибридный подход**, сочетающий оба способа:

1. **Точки перехода** — основной способ для ранней и средней стадии игры. Обеспечивают стратегическую глубину, исследование и контроль территорий.
2. **Варп-двигатели** — открываются на поздней стадии (технологический уровень 4+). Позволяют свободное перемещение, но с ограничениями:
   - Ограниченная дальность прыжка (warp_range).
   - Высокий расход топлива (дороже, чем JP).
   - Варп-полёт можно прервать в глубоком космосе (аварийный выход).
   - Варп-сигнал обнаруживается на больших расстояниях — демаскирует флот.

```
Стадия игры     Способ перемещения        Доступность
──────────────  ────────────────────────  ────────────
Ранняя          Только JP                 ★☆☆☆☆
Средняя         JP + варп (короткий)      ★★★☆☆
Поздняя         JP + варп (дальний)       ★★★★★
```

**Это даёт:**
- На ранней стадии — графовую стратегию с горлами и узлами.
- На поздней стадии — свободу действий, обход укреплённых узлов, новые тактические возможности.
- Плавный переход, не обесценивающий ранние инвестиции в оборону узлов.

## 3. Макроскопические объекты

Помимо звёздных систем, галактика содержит макроскопические объекты, влияющие на навигацию, стратегию и экономику. Эти объекты размещаются **между системами** или **в регионах галактики** и не привязаны к конкретным звёздам.

### 3.1. Туманности (Nebulae)

```
  ╔══════════════════════════════╗
  ║  ░░░░░░░░░░░░░░░░░░░░░░░░  ║
  ║  ░░ ТУМАННОСТЬ «ОРИОН» ░░░  ║
  ║  ░░░░░░░░░░░░░░░░░░░░░░░░  ║
  ║  ░░ ░░░░ ★ ░░░ ░░ ★ ░░░░  ║
  ║  ░░░░░░░░░░░░░░░░░░░░░░░░  ║
  ╚══════════════════════════════╝
```

Туманности — обширные области газа и пыли, оказывающие влияние на движение и обнаружение.

#### Свойства

| Свойство                        | Значение / Диапазон                   |
|---------------------------------|---------------------------------------|
| Типичный размер                 | 500–5000 пк в диаметре                |
| Количество на галактику         | 3–15 (зависит от размера галактики)   |
| Замедление движения             | ×0.3–×0.7 от нормальной скорости      |
| Экранирование датчиков          | Радиус обнаружения ×0.1–×0.5          |
| Обнаружение туманности          | Видна с дальнего расстояния (карта)   |
| Редкие ресурсы                  | Да — экзотические газы, органика      |
| Влияние на варп-полёт           | Варп невозможен внутри туманности      |

#### Генерация

- Размещаются преимущественно вдоль спиральных рукавов (туманности — области активного звездообразования).
- Форма — эллиптическая или неправильная (набор перекрывающихся эллипсов).
- Плотность убывает от центра к краю.

#### Стратегическое значение

- **Укрытие**: флот внутри туманности сложно обнаружить — база для пиратов и скрытых операций.
- **Препятствие**: замедляет движение, блокирует варп — создаёт естественные барьеры.
- **Ресурсы**: редкие элементы для продвинутых технологий.
- **Контроль**: контроль туманности = контроль ресурсов + контроль информации.

### 3.2. Газовые скопления (Gas Clouds)

```
     ∿∿∿∿∿∿∿∿∿∿
   ∿∿  H₂/He   ∿∿∿
```

| Добыча                          | Газовые гиганты внутри скопления      |

#### Связь с газовыми гигантами

Газовые скопления **с высокой вероятностью** содержат звёздные системы с газовыми гигантами. Добывающие станции на орбите газового гиганта могут извлекать:
- **Водород** — топливо для двигателей, сырьё для синтеза.
- **Гелий-3** — топливо для термоядерных реакторов (редкий, ценный).
- **Метан** — химическое сырьё.

#### Стратегическое значение

- Источник **топлива** — критический ресурс для логистики.
- Менее опасны, чем туманности — можно размещать базы снабжения.

- **Укрытие**: небольшие корабли могут прятаться среди астероидов.
- **Естественная граница**: заставляет обходить или оборудовать корабли щитами.

#### Стратегическое значение

- **Барьер**: разделяет регионы галактики, создавая изолированные «карманы».

- **Скрытый потенциал**: в центре пустоты может находиться **релкт** (см. 3.5) — награда за риск.

## 4. Связность галактики

### 4.1. Модель связности

Связность галактики определяется тем, как звёздные системы соединены между собой. В гибридной модели (рекомендуемой) связность определяется **двумя слоями**:

1. **Слой точек перехода (JP)** — граф связности, определяющий «магистрали».
2. **Слой варп-перемещения** — полный граф с ограничением по дальности (появляется на поздней стадии).

```
  Слой JP (ранняя игра):        Слой варпа (поздняя игра):
      A ─── B                    A ═══ B
      │     │                    ╲   ╱ │
      │     │                     ╲ ╱  │
      C ─── D ─── E               C ══ D ══ E
            │                     ╲  ╱  ╲  ╱
            F                      F ══════
```

### 4.2. Граф точек перехода

#### Определение связей

Для каждой пары систем A и B точка перехода создаётся, если выполняются условия:

1. Расстояние `d(A, B) ≤ max_jp_distance` (максимальная дальность JP).
2. Угол между A→B и ближайшим рукавом ≤ `arm_alignment_threshold` (JP предпочитают идти вдоль рукавов).
3. Нет макроскопического объекта на пути (туманность может блокировать JP).

#### Алгоритм генерации JP

```
Для каждой системы S:
  1. Найти всех соседей в радиусе max_jp_distance
  2. Отсортировать соседей по расстоянию
  3. Для каждого кандидата C (по возрастанию расстояния):
     a. Если S уже имеет max_jp_per_system точек → прекратить
     b. Если C уже имеет max_jp_per_system точек → пропустить
     c. Проверить отсутствие блокирующего объекта на пути S→C
     d. Проверить arm_alignment (опционально, с вероятностью)
     e. Создать JP между S и C
```

#### Обеспечение связности

После генерации JP проверяется **связность графа** (все системы достижимы из любой другой). Если граф несвязен:

1. Найти компоненты связности (BFS/DFS).
2. Для каждой пары соседних компонент найти ближайшие системы.
3. Создать JP между ними.
4. Повторять до полной связности.

### 4.3. Радиус обнаружения точки перехода

Точка перехода **не видна** на карте по умолчанию. Она должна быть обнаружена.

| Параметр                                    | Значение                       |
|---------------------------------------------|--------------------------------|
| Базовый радиус обнаружения JP               | 50 а.е. (внутри системы)       |
| С улучшенными датчиками                     | до 200 а.е.                    |
| Обнаружение из другой системы               | Невозможно                     |
| Обнаружение при сканировании системы        | Автоматическое (scan_system)   |
| Время обнаружения при патрулировании        | Зависит от размера системы     |
| Обнаружение JP после прохождения            | Точка выхода обнаруживается мгновенно (но не связь) |

#### Механика обнаружения

1. Корабль прибывает в звёздную систему.
2. Если корабль находится в пределах радиуса обнаружения JP — JP появляется на карте.
3. **Направление JP**: при обнаружении видно, что это точка перехода, но **не видно**, куда она ведёт.
4. **Исследование JP**: корабль проходит через JP → пункт назначения становится известен навсегда.
5. После исследования JP на карте отображается связь между системами.

### 4.4. Количество точек перехода на систему

| Параметр               | Значение / Диапазон      |
|------------------------|--------------------------|
| Минимум JP на систему  | 1 (обеспечение связности) |
| Максимум JP на систему | 6                        |
| Среднее JP на систему  | 2–3                      |
| Распределение          | Правостороннее (больше систем с 1–2 JP, меньше с 5–6) |

#### Распределение количества JP

```
Частота
  ▲
  │ █
  │ █ █
  │ █ █ █
  │ █ █ █ █
  │ █ █ █ █ █
  │ █ █ █ █ █ █
  └──────────────► Количество JP
    1 2 3 4 5 6

  ~35% систем имеют 1 JP (тупиковые)
  ~30% систем имеют 2 JP (линейные)
  ~20% систем имеют 3 JP (узловые)
  ~10% систем имеют 4 JP (хабы)
  ~4%  систем имеют 5 JP (мажорные хабы)
  ~1%  систем имеют 6 JP (сверххабы, обычно в ядре)
```

#### Влияние на стратегию

- **1 JP (тупик)**: безопасная глубинка, но уязвима при потере единственного JP.
- **2 JP (линейная)**: транзитная система, легко защищаемая с двух сторон.
- **3 JP (узловая)**: важный перекрёсток, требует гарнизона.
- **4+ JP (хаб)**: стратегический центр, контроль = контроль региона.

### 4.5. Узловые системы и стратегия

```
                    [Система X — хаб, 5 JP]
                   ╱    │    ╲
                  ╱     │     ╲
          [A]───[X]────[B]────[C]
                  ╲     │
                   ╲    │
                    [D]

  Контроль X → контроль путей между A, B, C, D
  Потеря X  → разделение территорий, удлинение логистики
```

#### Классификация систем по стратегическому значению

| Класс            | JP  | Пример расположения            | Стратегическая ценность |
|------------------|-----|--------------------------------|-------------------------|
| Тупиковая        | 1   | Окраина рукава                 | Низкая (глубинка)       |
| Линейная         | 2   | Средняя часть рукава           | Средняя (транзит)       |
| Узловая          | 3   | Пересечение рукава и моста     | Высокая                 |
| Хаб              | 4   | Ядро, пересечение рукавов      | Очень высокая           |
| Мажорный хаб     | 5   | Центр ядра                     | Критическая             |
| Сверххаб         | 6   | Центр галактики                | Уникальная              |

#### Последствия для ИИ и игрока

- ИИ должен **оценивать узловые системы** и стремиться к их контролю.
- Оборона узловой системы требует **нескольких фортов/станций** по числу JP.
- **Логистические маршруты** строятся через узловые системы — их перехват парализует снабжение.

### 4.6. Вормхолы Предтеч (дополнительная связность)

Вормхолы Предтеч (разновидность реликта, см. 3.5) создают **дополнительные связи** в графе, которые:

- Соединяют **удалённые** регионы галактики.
- Стабильны и двунаправленны (в отличие от JP, не требуют двигателей).
- Обнаруживаются только при исследовании соответствующего реликта.
- Могут быть **разрушены** или **деактивированы** специальным оружием.

Вормхолы добавляют элемент неожиданности: даже «безопасный» тыл может оказаться уязвим через обнаруженный вормхол.

### 5.3. Порядок генерации

Генерация галактики выполняется в строгом порядке для обеспечения корректности:

```
Шаг 1: Инициализация PRNG(seed)
  │
  Шаг 2: Генерация позиций звёздных систем
  │   ├── Ядро (bulge_seed)
  │   ├── Рукава (arm_seed)
  │   ├── Диск (disk_seed)
  │   └── Ореол (halo_seed)
  │
  Шаг 3: Размещение макроскопических объектов
  │   ├── Туманности (nebula_seed)  ← приоритет, т.к. блокируют JP
  │   ├── Пустоты (void_seed)
  │   ├── Газовые скопления (gas_seed)
  │   ├── Астероидные пояса (asteroid_seed)
  │   └── Реликты (relic_seed)
  │
  Шаг 4: Генерация точек перехода
  │   ├── Создание JP между системами (jp_seed)
  │   ├── Проверка связности графа
  │   ├── Добавление JP для обеспечения связности
  │   └── Генерация вормхолов Предтеч (если включены)
  │
  Шаг 5: Генерация содержимого систем
  │   ├── Для каждой системы: звезда, планеты, орб. пояса (system_seed[i])
  │   └── Размещение добывающих ресурсов на планетах
  │
  Шаг 6: Валидация
      ├── Все системы доступны (связность)
      ├── Нет систем внутри туманности без выхода
      ├── Ресурсы достаточно разнообразны
      └── Распределение JP корректно
```
