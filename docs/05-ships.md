## 1. Модульная система кораблей

### 1.1 Принцип модульности

Корабль — это набор модулей, размещённых в слотах корпуса. Не существует «типовых кораблей» как монолитных сущностей: каждый корабль собирается из модулей, и его характеристики полностью определяются их набором.

```
Корабль = Корпус + [Модуль_1, Модуль_2, ..., Модуль_N]
```

### 1.2 Единица объёма — HS (Hull Space)

**HS (Hull Space)** — базовая единица внутреннего объёма корабля. Каждый модуль занимает определённое количество HS. Корпус определяет общее количество доступных HS.

- HS — это мера **внутреннего объёма**, а не длины или площади
- Модуль размером 5 HS занимает 5 единиц объёма корпуса
- Сумма HS всех установленных модулей ≤ HS корпуса
- Свободные HS не дают бонусов, но позволяют модернизировать корабль

### 1.3 Формулы расчёта характеристик

#### Общая масса корабля

```
mass_total = Σ(modules[i].mass) + hull.mass
```

Где:
- `modules[i].mass` — масса i-го модуля в тоннах
- `hull.mass` — масса пустого корпуса (обшивка + каркас) в тоннах

#### Скорость корабля

```
speed = Σ(engines[i].thrust) / mass_total
```

Где:
- `engines[i].thrust` — тяга i-го двигателя в тоннах-силы
- `mass_total` — полная масса корабля в тоннах
- `speed` — условная скорость, измеряется в км/с

**Пороговые значения скорости:**

| Скорость (км/с) | Класс манёвренности | Эффект в бою |
|------------------|---------------------|--------------|
| 0–50 | Неподвижный | Штраф -50% к уклонению |
| 51–200 | Малая | Штраф -25% к уклонению |
| 201–500 | Средняя | Без бонусов/штрафов |
| 501–1000 | Высокая | Бонус +25% к уклонению |
| 1001+ | Сверхманёвренный | Бонус +50% к уклонению |

#### Энергобаланс

```
energy_balance = Σ(energy_sources[i].output) - Σ(modules[j].consumption)
```

- `energy_sources` — реакторы и генераторы (в МВт)
- `modules[j].consumption` — энергопотребление каждого модуля (в МВт)
- Если `energy_balance < 0` — корабль не может функционировать на полную мощность
- При дефиците энергии приоритет: двигатели → управление → щиты → оружие → вспомогательные

#### Стоимость производства

```
cost_total = hull.cost + Σ(modules[i].cost)
```

Стоимость — в условных единицах ресурсоёмкости (у.е.р.), каждая у.е.р. соответствует набору ресурсов, определённому рецептом крафта.

### 1.4 Структура модуля

Каждый модуль описывается следующими атрибутами:

```typescript
interface ShipModule {
  id: string;                    // Уникальный идентификатор ("engine_ion_mk1")
  name: string;                  // Название ("Ионный двигатель Mk.I")
  category: ModuleCategory;      // Категория (двигатель, оружие, щит и т.д.)
  size: number;                  // Занимаемый объём в HS
  mass: number;                  // Масса в тоннах
  energyConsumption: number;     // Потребление энергии в МВт (0 — не потребляет)
  cost: number;                  // Стоимость производства в у.е.р.
  techLevel: number;             // Минимальный технологический уровень (1–10)
  requiredTechs: string[];       // ID необходимых технологий
  slotRestriction?: SlotType;    // Ограничение типа слота (опционально)
  // Специфичные атрибуты зависят от категории
}

type ModuleCategory =
  | 'engine'
  | 'control'
  | 'life_support'
  | 'weapon'
  | 'defense'
  | 'auxiliary';

type SlotType =
  | 'any'        // Любой слот
  | 'weapon'     // Оружейный слот
  | 'engine'     // Двигательный слот
  | 'system'     // Системный слот (управление, вспомогательные)
  | 'defense';   // Оборонительный слот (щиты, броня)
```

### 1.5 Слоты корпуса

Корпус предоставляет слоты разных типов. Не все модули можно установить в любой слот.

```
Корпус = {
  totalHS: number;           // Общий объём в HS
  weaponSlots: number;       // Максимум оружейных слотов
  engineSlots: number;       // Максимум двигательных слотов
  systemSlots: number;       // Максимум системных слотов
  defenseSlots: number;      // Максимум оборонительных слотов
}
```

**Правила размещения:**
- Модуль с `slotRestriction: 'weapon'` — только в оружейный слот
- Модуль с `slotRestriction: 'engine'` — только в двигательный слот
- Модуль с `slotRestriction: 'system'` — только в системный слот
- Модуль с `slotRestriction: 'defense'` — только в оборонительный слот
- Модуль с `slotRestriction: 'any'` — в любой слот
- Сумма HS модулей в слотах одного типа ≤ общее HS, выделенное этому типу
- **Оружейные модули** дополнительно ограничены количеством weaponSlots
- **Двигательные модули** дополнительно ограничены количеством engineSlots

### 1.6 Обязательные модули

Каждый корабль **обязан** содержать:
1. **Корпус** — база, на которую всё устанавливается
2. **Мостик/ЦПУ (Bridge/CPU)** — модуль управления (без него корабль — груда железа)
3. **Как минимум один двигатель** — без двигателя корабль неподвижен

Без этих трёх компонентов корабль не может быть построен и введён в эксплуатацию.

|-------------|-----|-----------|-----------|-------------|-------------|-------------|-------------|---------------------|
| **Скаут** | 25 | 200 | 500 | 1 | 2 | 3 | 1 | 50 |
| **Истребитель** | 50 | 400 | 1 000 | 2 | 2 | 2 | 2 | 120 |
| **Фрегат** | 100 | 1 000 | 2 500 | 4 | 3 | 4 | 3 | 300 |
| **Крейсер** | 250 | 3 000 | 7 000 | 8 | 4 | 6 | 5 | 900 |
| **Линкор** | 500 | 8 000 | 15 000 | 14 | 5 | 8 | 8 | 2 500 |
| **Транспорт** | 150 | 800 | 4 000 | 2 | 3 | 5 | 2 | 250 |
| **Флагман** | 1 000 | 15 000 | 30 000 | 20 | 6 | 12 | 10 | 8 000 |

### 2.2 Параметры корпуса — подробное описание

#### HS (Hull Space) — Общий объём
Определяет, сколько модулей можно установить. Сумма HS всех модулей не может превышать это значение. Свободные HS могут быть зарезервированы для будущей модернизации.

#### Базовый HP (Hit Points)
Прочность корпуса без учёта брони и щитов. Когда HP достигают 0 — корабль уничтожен. HP восстанавливаются только в ремонтном доке или модулем ремонта.

**Формула итоговых HP:**
```
hp_total = hull.baseHP + armor.bonusHP
```

#### Масса корпуса
Масса пустого корпуса (каркас + обшивка). Влияет на общую массу и скорость. Не включает массу модулей.

#### Слоты
Ограничивают количество модулей определённого типа:
- **Оружейные слоты** — максимум независимых оружейных систем
- **Двигательные слоты** — максимум двигателей
- **Системные слоты** — ЦПУ, сканеры, связь, вспомогательные модули
- **Оборонительные слоты** — щиты, броня, ЭМИ-защита, маскировка

#### Стоимость
Базовая стоимость производства корпуса. Зависит от размера и сложности.

### 2.3 Толщина обшивки

Толщина обшивки — дополнительный параметр корпуса, влияющий на HP и массу. Выбирается при проектировании корабля.

| Обшивка | Бонус HP | Бонус массы | Стоимость (множитель) |
|---------|----------|-------------|----------------------|
| **Лёгкая** | +0% | +0% | ×1.0 |
| **Стандартная** | +25% | +10% | ×1.2 |
| **Утолщённая** | +50% | +25% | ×1.5 |
| **Тяжёлая** | +100% | +50% | ×2.0 |

**Пример:** Крейсер со стандартной обшивкой: HP = 3 000 × 1.25 = 3 750, масса = 7 000 × 1.10 = 7 700 т.

## 3. Двигатели (Engines)

### 3.1 Общие принципы

Двигатели обеспечивают тягу, которая в сочетании с массой корабля определяет скорость. Каждый двигатель потребляет топливо и/или энергию. Тип топлива зависит от типа двигателя.

**Формула расчёта:**
```
speed_kms = (Σ engine.thrust) / mass_total × speed_coefficient
```

Где `speed_coefficient` = 10 (константа для приведения к км/с).

**Расход топлива за ход (тика):**
```
fuel_per_tick = Σ(engine.thrust × engine.fuelPerThrust × speed_factor)
```

Где `speed_factor` = текущая скорость / максимальная скорость (0.0–1.0).

### 3.2 Типы двигателей

#### 3.2.1 Химический двигатель

Дешёвый, малая тяга, расходует химическое топливо.

| Параметр | Mk.I | Mk.II | Mk.III | Mk.IV | Mk.V |
|----------|------|-------|--------|-------|------|
| **Тяга (т)** | 500 | 1 000 | 2 000 | 4 000 | 8 000 |
| **Расход топлива (ед/т тяги/тик)** | 0.10 | 0.09 | 0.08 | 0.07 | 0.06 |
| **Тип топлива** | Химическое | Химическое | Химическое | Химическое | Химическое |
| **Энергопотребление (МВт)** | 0 | 0 | 0 | 0 | 0 |
| **HS** | 2 | 3 | 5 | 8 | 12 |
| **Масса (т)** | 50 | 100 | 200 | 400 | 800 |
| **Стоимость (у.е.р.)** | 10 | 25 | 60 | 150 | 400 |
| **Тех. уровень (Физика)** | 1 | 2 | 3 | 5 | 7 |

**Особенности:**
- Не потребляет электроэнергию — полностью автономен
- Наибольший расход топлива на единицу тяги
- Не позволяет совершать варп-прыжки
- Используется для дешёвых кораблей обороны и транспорта на коротких маршрутах

#### 3.2.2 Ионный двигатель

Средняя эффективность, расходует ксенон + электроэнергию.

| Параметр | Mk.I | Mk.II | Mk.III | Mk.IV | Mk.V |
|----------|------|-------|--------|-------|------|
| **Тяга (т)** | 800 | 1 600 | 3 200 | 6 400 | 12 000 |
| **Расход ксенона (ед/т тяги/тик)** | 0.02 | 0.018 | 0.016 | 0.014 | 0.012 |
| **Энергопотребление (МВт)** | 5 | 10 | 18 | 30 | 50 |
| **HS** | 3 | 4 | 7 | 11 | 16 |
| **Масса (т)** | 80 | 150 | 300 | 600 | 1 100 |
| **Стоимость (у.е.р.)** | 25 | 55 | 130 | 300 | 700 |
| **Тех. уровень (Физика)** | 2 | 3 | 4 | 6 | 8 |

**Особенности:**
- Оптимальное соотношение тяги к расходу топлива
- Требует ксенон (редкий ресурс) и электроэнергию
- Основной двигатель для большинства кораблей
- Не позволяет совершать варп-прыжки

#### 3.2.3 Плазменный двигатель

Мощный, расходует водород + электроэнергию.

| Параметр | Mk.I | Mk.II | Mk.III | Mk.IV | Mk.V |
|----------|------|-------|--------|-------|------|
| **Тяга (т)** | 1 500 | 3 000 | 6 000 | 12 000 | 24 000 |
| **Расход водорода (ед/т тяги/тик)** | 0.04 | 0.035 | 0.03 | 0.025 | 0.02 |
| **Энергопотребление (МВт)** | 15 | 25 | 45 | 80 | 140 |
| **HS** | 5 | 7 | 11 | 17 | 25 |
| **Масса (т)** | 150 | 280 | 550 | 1 100 | 2 200 |
| **Стоимость (у.е.р.)** | 50 | 110 | 260 | 600 | 1 400 |
| **Тех. уровень (Физика)** | 3 | 4 | 6 | 7 | 9 |

**Особенности:**
- Максимальная тяга среди стандартных двигателей
- Водород — дешёвый и распространённый ресурс
- Высокое энергопотребление — требует мощного реактора
- Не позволяет совершать варп-прыжки
- Используется для боевых кораблей и крупных транспортов

#### 3.2.4 Варп-двигатель

Позволяет прыжки между звёздными системами без точек перехода. Огромный расход энергии.

| Параметр | Mk.I | Mk.II | Mk.III | Mk.IV | Mk.V |
|----------|------|-------|--------|-------|------|
| **Тяга (т)** | 600 | 1 200 | 2 400 | 5 000 | 10 000 |
| **Расход водорода (ед/т тяги/тик)** | 0.06 | 0.055 | 0.05 | 0.045 | 0.04 |
| **Энергопотребление (МВт, крейсерский)** | 10 | 18 | 30 | 55 | 90 |
| **Энергопотребление (МВт, варп-прыжок)** | 500 | 450 | 400 | 350 | 300 |
| **Дальность варп-прыжка (св. лет)** | 2 | 4 | 8 | 15 | 25 |
| **HS** | 8 | 12 | 18 | 26 | 36 |
| **Масса (т)** | 400 | 700 | 1 200 | 2 000 | 3 500 |
| **Стоимость (у.е.р.)** | 200 | 450 | 1 000 | 2 200 | 5 000 |
| **Тех. уровень (Физика)** | 5 | 6 | 7 | 8 | 10 |

**Особенности:**
- Позволяет прыгать между системами без точки перехода (jump point)
- Варп-прыжок требует накопления энергии (зарядка 5–20 тиков в зависимости от расстояния)
- Во время зарядки варп-двигателя щиты отключены
- После прыжка — перезарядка 10 тиков
- В крейсерском режиме работает как плазменный двигатель с пониженной тягой
- Двигатель и прыжковый модуль (Jump Drive) — **разные вещи**: варп-двигатель позволяет свободный прыжок, Jump Drive — проход через точку перехода

### 3.3 Энергоснабжение двигателей

Двигатели, потребляющие электроэнергию, требуют источника питания. Энергию обеспечивают:

| Источник | Выход (МВт) | HS | Масса (т) | Стоимость | Топливо | Тех. уровень |
|----------|-------------|-----|-----------|-----------|---------|-------------|
| **Солнечная батарея** | 5 | 1 | 20 | 10 | нет | 1 |
| **Топливный генератор** | 20 | 2 | 80 | 30 | Химическое | 1 |
| **Ядерный реактор Mk.I** | 50 | 4 | 200 | 100 | U (0.01/тик) | 3 |
| **Ядерный реактор Mk.II** | 120 | 6 | 400 | 250 | U (0.02/тик) | 5 |
| **Термоядерный реактор Mk.I** | 300 | 10 | 800 | 600 | H (0.05/тик) | 7 |
| **Термоядерный реактор Mk.II** | 600 | 14 | 1 500 | 1 200 | H (0.08/тик) | 9 |
| **Антиматериальный реактор** | 1 500 | 20 | 2 500 | 4 000 | Антиматерия (0.001/тик) | 10 |

> **Важно**: реакторы — это не двигатели. Реакторы производят энергию, двигатели — тягу. Корабль должен иметь достаточную энергетическую установку для питания всех энергозависимых модулей.

## 4. Система управления (Control Systems)

### 4.1 Общие принципы

Система управления — обязательный набор модулей, обеспечивающих функционирование корабля. В мире игры все корабли управляются ИИ/роботами, поэтому вместо классического «мостика с экипажем» используется ЦПУ (центральный процессор управления).

### 4.2 Мостик/ЦПУ (Bridge/CPU) — ОБЯЗАТЕЛЬНЫЙ МОДУЛЬ

Без ЦПУ корабль не может быть построен. ЦПУ обрабатывает приказы, координирует модули, управляет навигацией и боем.

| Тип ЦПУ | HS | Масса (т) | Энергия (МВт) | Стоимость | Базовый бонус | Макс. корпус |
|---------|-----|-----------|---------------|-----------|---------------|-------------|
| **ЦПУ-Микро** | 1 | 10 | 1 | 15 | — | Скаут, Истребитель |
| **ЦПУ-Лёгкий** | 2 | 30 | 3 | 35 | — | Истребитель, Фрегат |
| **ЦПУ-Стандарт** | 4 | 80 | 8 | 80 | — | Фрегат, Транспорт |
| **ЦПУ-Усиленный** | 6 | 180 | 15 | 180 | — | Крейсер, Линкор |
| **ЦПУ-Тактический** | 8 | 350 | 25 | 350 | +5% боевая эффективность | Крейсер, Линкор |
| **ЦПУ-Флагманский** | 12 | 700 | 40 | 700 | +10% боевая эффективность флота | Флагман |

**Правило масштабирования ЦПУ:**
- Корпус требует ЦПУ определённого минимального уровня
- Установка ЦПУ ниже минимального уровня для корпуса — **невозможна**
- Установка более мощного ЦПУ на малый корпус — возможна, но занимает больше HS

**Ограничения ЦПУ по корпусам:**

## 5. Система жизнеобеспечения (Life Support)

### 5.1 Ключевой принцип: ВСЕ корабли — автоматические

> **ВАЖНО**: В мире игры «Полдень XXI века» космический флот полностью автоматизирован. Корабли управляются ИИ и роботами. Биологический экипаж — исключение, а не правило.

Это принципиальное дизайнерское решение, которое влияет на всю игровую механику:

1. **По умолчанию** — на корабле НЕТ экипажа, НЕТ жизнеобеспечения
2. Корабль управляется ЦПУ и роботами
3. Life Support — **опциональный модуль**, устанавливаемый только для специфических задач
4. Отсутствие Life Support не является недостатком — это норма

### 5.2 Когда нужен Life Support

Life Support устанавливается **только** в следующих сценариях:

| Сценарий | Причина | Бонус |
|----------|---------|-------|
| **Транспорт с биопассажирами** | Перевозка людей между колониями | Нет боевого бонуса |
| **Колониальный транспорт** | Перевозка поселенцев для основания колонии | Нет боевого бонуса |
| **Флагман с адмиралом** | Человек-адмирал управляет флотом | +10% к боевому решению (адмиральский бонус) |
| **Научный корабль с ксенобиологами** | Изучение биологических артефактов | +20% к скорости ксеноархеологических исследований |
| **Дипломатический курьер** | Перевозка живых послов между фракциями | Открывает дипломатические опции |

### 5.3 Характеристики модулей Life Support

| Тип | HS | Масса (т) | Энергия (МВт) | Стоимость | Вместимость | Потребление |
|------|-----|-----------|---------------|-----------|-------------|-------------|
| **ЖО-Кабина** | 2 | 50 | 5 | 30 | 1 чел. | O₂(0.1/тик), H₂O(0.05/тик), Пища(0.02/тик) |
| **ЖО-Отсек** | 5 | 120 | 12 | 70 | 5 чел. | O₂(0.5/тик), H₂O(0.25/тик), Пища(0.1/тик) |
| **ЖО-Квартал** | 10 | 250 | 25 | 150 | 20 чел. | O₂(2.0/тик), H₂O(1.0/тик), Пища(0.4/тик) |
| **ЖО-Блок** | 20 | 500 | 50 | 300 | 100 чел. | O₂(10.0/тик), H₂O(5.0/тик), Пища(2.0/тик) |
| **ЖО-Модуль колонии** | 30 | 800 | 75 | 500 | 500 чел. | O₂(50.0/тик), H₂O(25.0/тик), Пища(10.0/тик) |

### 5.4 Адмиральский бонус

Когда на флагмане установлен модуль Life Support с адмиралом:

```
fleet_combat_bonus = +10% к боевой эффективности всего флота
```

- Адмирал — уникальный персонаж (как в Stellaris / Endless Space)
- Адмирал имеет уровень (1–5), растущий с опытом
- Бонус = 5% + (admiral.level × 5%), максимум +30% на уровне 5
- Адмирал может погибнуть при уничтожении флагмана
- Адмирал может быть переведён на другой флагман

**Уровни адмиралов:**

| Уровень | Бонус | Опыт (боёв) | Спец. способность |
|---------|-------|-------------|-------------------|
| 1 | +10% | 0 | — |
| 2 | +15% | 5 | +5% уклонение флота |
| 3 | +20% | 15 | +10% скорость ремонта |
| 4 | +25% | 30 | +15% эффективность щитов флота |
| 5 | +30% | 50 | +20% шанс критического удара |

### 5.5 Рекомендация для MVP

> **Для минимально жизнеспособного продукта (MVP)**: все корабли автоматические. Life Support реализован как опциональный модуль, но:
> - Адмиральский бонус — **реализовать** (важно для геймплея)
> - Колониальный транспорт — **реализовать** (нужен для колонизации)
> - Остальные сценарии — отложить до пост-MVP

### 5.6 Структура данных

```typescript

  category: 'life_support';
  capacity: number;              // Количество биопассажиров
  oxygenPerTick: number;         // Расход O₂ за тик
  waterPerTick: number;          // Расход H₂O за тик
  foodPerTick: number;           // Расход пищи за тик
  hasAdmiralSlot: boolean;       // Есть ли место для адмирала
  isColonyModule: boolean;       // Является ли колонизационным
}
```

## 6. Оружейные системы (Weapons)

### 6.1 Общие принципы

Каждое оружие характеризуется:
- **Урон** — базовое повреждение за выстрел
- **Дальность** — максимальная дистанция стрельбы (в км)
- **Скорострельность** — выстрелов за тик
- **Энергопотребление** — МВт за выстрел
- **Точность** — базовый % попадания
- **Тип урона** — энергетический / кинетический / ионный / плазменный
- **Размер (HS)** — занимаемый объём
- **Тех. уровень** — минимальный уровень Военных наук

**Модификаторы урона по типу защиты:**

| Тип урона | Против щита | Против брони | Против корпуса | Особенности |
|-----------|-------------|--------------|----------------|-------------|
| Энергетический (лазер) | ×0.8 | ×1.0 | ×1.0 | Стандартный |
| Кинетический (Гаусс) | ×1.5 | ×0.7 | ×1.0 | Пробивает щиты |
| Ионный | ×2.0 | ×0.1 | ×0.05 | Урон по электронике |
| Плазменный | ×0.6 | ×1.3 | ×1.2 | Эффективен против брони |
| Ракетный | ×1.0 | ×1.0 | ×1.5 | Можно перехватить |
| Торпедный | ×0.5 | ×1.5 | ×2.0 | Медленная, мощная |

## 7. Системы щитов и маскировки (Defense & Stealth)

### 7.1 Энергетический щит

Поглощает урон, потребляет энергию, регенерирует со временем.

**Фазы щита:**

```
Заряжен (100%) → Повреждён (1–99%) → Истощён (0%) → Перезарядка → Заряжен
```

| Фаза | Описание | Механика |
|------|----------|----------|
| **Заряжен** | Щит на полную мощность | Поглощает урон полностью |
| **Повреждён** | Щит частично истощён | Поглощает урон пропорционально оставшимся HP |
| **Истощён** | HP щита = 0 | Не поглощает урон, весь урон идёт в броню/корпус |
| **Перезарядка** | Восстановление после истощения | 5 тиков без получения урона, затем начинается регенерация |

**Регенерация щита:**
```
shield_regen_per_tick = shield.maxHP × 0.02 × (energy_available / energy_required)
```

Щит регенерирует 2% от максимума за тик при достаточной энергии. Если энергия в дефиците — регенерация замедляется пропорционально.

- Эффективность регенерации не увеличивается от количества щитов

### 7.3 Маскировочное поле (Stealth)

Снижает сигнатуру корабля, усложняя обнаружение.

| Тип | HS | Масса (т) | Энергия (МВт) | Стоимость | Снижение сигнатуры | Тех. уровень |
|------|-----|-----------|---------------|-----------|-------------------|-------------|
| **Маскировка Mk.I** | 2 | 40 | 8 | 40 | -30% сигнатуры | 4 |
| **Маскировка Mk.II** | 4 | 80 | 15 | 100 | -50% сигнатуры | 6 |
| **Маскировка Mk.III** | 6 | 150 | 25 | 220 | -70% сигнатуры | 8 |
| **Полный камуфляж** | 10 | 300 | 50 | 500 | -90% сигнатуры | 10 |

**Механика обнаружения:**
```
detection_chance = scanner.power × (1 - stealth.signature_reduction) / distance
```

- Сигнатура корабля = базовая (зависит от HS корпуса) + модули
- Маскировка снижает эффективную сигнатуру
- При сигнатуре ≤ порога сканера — корабль невидим
- Маскировка отключается при стрельбе (сигнатура ×3 на 3 тика)

Защита от ионного оружия.

| Тип | HS | Масса (т) | Энергия (МВт) | Стоимость | Снижение ионного урона | Тех. уровень |
|------|-----|-----------|---------------|-----------|----------------------|-------------|
| **ЭМИ-щит Mk.I** | 2 | 50 | 5 | 30 | -40% ионного урона | 4 |
| **ЭМИ-щит Mk.II** | 3 | 100 | 10 | 70 | -60% ионного урона | 6 |
| **ЭМИ-щит Mk.II** | 5 | 180 | 18 | 150 | -80% ионного урона | 8 |

**Особенности:**
- Не защищает от других типов урона
- Снижает шанс отключения модулей от ионного оружия
- ЭМИ-щит занимает оборонительный слот (конкурирует с обычными щитами)
- Рекомендуется для флагманов и крупных кораблей

### 7.5 Броня

Пассивная защита — не потребляет энергию, добавляет HP и массу.

| Тип брони | Бонус HP за HS | Масса за HS | Стоимость за HS | Тех. уровень |
|-----------|---------------|-------------|-----------------|-------------|
| **Стальная обшивка** | +50 HP | +40 т | 5 | 1 |
| **Титановая обшивка** | +80 HP | +30 т | 12 | 3 |
| **Вольфрамовая обшивка** | +120 HP | +50 т | 25 | 5 |
| **Композитная броня** | +160 HP | +35 т | 40 | 7 |
| **Нано-броня** | +250 HP | +25 т | 80 | 9 |

**Особенности:**
- Броня устанавливается в оборонительные слоты
- Количество HS брони = количество HS, выделенных в слот
- Бонус HP = бонус за HS × количество HS
- Масса брони добавляется к массе корабля (снижает скорость!)
- Броня не регенерируется — ремонт только на базе или модулем ремонта
- В отличие от щитов — работает всегда, даже при нулевой энергии

**Пример:** 5 HS композитной брони = 160 × 5 = 800 HP, масса = 35 × 5 = 175 т.

### 7.6 Структура данных оборонительных модулей

```typescript
interface DefenseModule extends ShipModule {
  category: 'defense';
  defenseType: 'shield' | 'stealth' | 'emi_shield' | 'armor';
  // Щиты
  shieldHP?: number;             // Максимум HP щита
  regenRate?: number;            // % регенерации за тик
  // Маскировка
  signatureReduction?: number;   // % снижения сигнатуры
  // ЭМИ-щит
  ionDamageReduction?: number;   // % снижения ионного урона
  // Броня
  hpPerHS?: number;              // Бонус HP за единицу HS
  massPerHS?: number;            // Масса за единицу HS
}
```

### 8.8 Модуль прыжка (Jump Drive)

Проезд через точку перехода (jump point) между системами.

| Тип | HS | Масса (т) | Энергия (МВт) | Стоимость | Макс. масса прыжка (т) | Тех. уровень |
|------|-----|-----------|---------------|-----------|----------------------|-------------|
| **Прыжковый Mk.I** | 5 | 200 | 30 | 120 | 10 000 | 3 |
| **Прыжковый Mk.II** | 7 | 350 | 50 | 250 | 30 000 | 5 |
| **Прыжковый Mk.III** | 10 | 600 | 80 | 500 | 80 000 | 7 |
| **Прыжковый Mk.IV** | 14 | 1 000 | 120 | 900 | 200 000 | 9 |

**Механика:**
- Jump Drive ≠ Варп-двигатель:
  - **Jump Drive** — проход через существующую точку перехода (jump point) между системами
  - **Варп-двигатель** — свободный прыжок в любую систему (без точки перехода)
- Jump Drive дешевле и компактнее варп-двигателя
- Без Jump Drive или варп-двигателя корабль заперт в своей системе
- Точки перехода обнаруживаются сканером
- Корабль с Jump Drive может «буксировать» корабли без прыжкового модуля (если суммарная масса ≤ максимума прыжка)

## Приложение A: Формулы боя (кратко)

### Расчёт урона

```
actual_damage = base_damage × damage_modifier_vs_target × accuracy_roll × range_modifier
```

Где:
- `damage_modifier_vs_target` — из таблицы типов урона (раздел 6.1)
- `accuracy_roll` = случайное число от 0.5 до 1.0, если `random(0,100) <= accuracy`
- `range_modifier` = 1.0 - (distance / max_range × 0.3), минимум 0.5

### Поглощение урона

```
if shield.hp > 0:
    shield_damage = actual_damage × shield_absorption_rate
    hull_damage = actual_damage × (1 - shield_absorption_rate)
    shield.hp -= shield_damage
    hull.hp -= hull_damage × armor_reduction
else:
    hull.hp -= actual_damage × armor_reduction
```

Где `armor_reduction` = 1 - (armor_hp / max_armor_hp × 0.3), минимум 0.5.

### Уклонение

```
evasion_chance = (speed / 500) × 5%   // максимум 50% при скорости 5000+ км/с
if random(0, 100) < evasion_chance:
    attack_misses = true
```

## Приложение B: Валидация корабля

При проектировании корабля система проверяет:

```typescript
function validateShip(design: ShipDesign): ValidationResult {
  const errors: string[] = [];

  // 1. Обязательные модули
  if (!design.modules.some(m => m.category === 'control' && m.controlType === 'cpu'))
    errors.push('Отсутствует ЦПУ — обязательный модуль');
  if (!design.modules.some(m => m.category === 'engine'))
    errors.push('Отсутствует двигатель — корабль не может двигаться');

  // 2. Объём
  const totalHS = design.modules.reduce((sum, m) => sum + m.size, 0);
  if (totalHS > design.hull.totalHS)
    errors.push(`Превышен объём корпуса: ${totalHS} > ${design.hull.totalHS} HS`);

  // 3. Слоты
  const weaponCount = design.modules.filter(m => m.slotRestriction === 'weapon').length;
  if (weaponCount > design.hull.weaponSlots)
    errors.push(`Превышено количество оружейных слотов: ${weaponCount} > ${design.hull.weaponSlots}`);

  const engineCount = design.modules.filter(m => m.slotRestriction === 'engine').length;
  if (engineCount > design.hull.engineSlots)
    errors.push(`Превышено количество двигательных слотов: ${engineCount} > ${design.hull.engineSlots}`);

  // 4. Энергобаланс
  const energyBalance = calculateEnergyBalance(design);
  if (energyBalance < 0)
    errors.push(`Дефицит энергии: ${energyBalance} МВт`);

  // 5. ЦПУ соответствует корпусу
  const cpu = design.modules.find(m => m.category === 'control' && m.controlType === 'cpu');
  if (cpu && !isCpuSufficient(cpu, design.hull))
    errors.push('ЦПУ недостаточной мощности для данного корпуса');

  // 6. Технологические требования
  for (const module of design.modules) {
    for (const techId of module.requiredTechs) {
      if (!player.hasTech(techId))
        errors.push(`Технология ${techId} не исследована для модуля ${module.name}`);
    }
  }

  return { valid: errors.length === 0, errors };
}
```

Корабли строятся на **верфях**. Требования верфи:

| Корпус | Мин. уровень верфи | Время постройки (тиков) |
|--------|-------------------|------------------------|
| Скаут | 1 | 50 |
| Истребитель | 1 | 80 |
| Фрегат | 2 | 150 |
| Крейсер | 3 | 400 |
| Линкор | 5 | 1 000 |
| Транспорт | 2 | 120 |
| Флагман | 7 | 3 000 |

Время постройки можно сократить:
- Улучшение верфи: -10% за уровень сверх минимального
- Модули быстрее корпуса: время = время_корпуса + Σ(время_модуля_i)
- Параллельная сборка: верфь уровня N может одновременно строить N/2 кораблей (округл. вверх)
