## 1. Общие принципы

### 1.1 Размещение зданий

Здания размещаются на поверхности планеты по **гекс-сетке**. Каждый гекс может содержать одно здание. Размер гекс-сетки определяется радиусом планеты:

| Радиус планеты (g-класс) | Гексов на планете | Пример |
|--------------------------|-------------------|--------|
| Крошечная (R < 0.3) | 19 | Меркурий |
| Малая (0.3 ≤ R < 0.7) | 37 | Марс |
| Средняя (0.7 ≤ R < 1.3) | 61 | Земля |
| Большая (1.3 ≤ R < 2.0) | 91 | Суперземля |
| Огромная (R ≥ 2.0) | 127 | Мегаземля |

> **Примечание для программиста**: гекс-сетка — стандартная axial-координатная система (q, r). Соседние гексы определяются шестью направлениями: [(+1,0), (-1,0), (0,+1), (0,-1), (+1,-1), (-1,+1)].

### 1.2 Уровни зданий

Каждое здание имеет **уровень от 1 до 10**. Повышение уровня:
- Увеличивает скорость работы (добыча, переработка, производство)
- Снижает стоимость эксплуатации (энергия на 1 единицу продукции)
- Может открывать доступ к новым рецептам / ресурсам
- Требует ресурсов для улучшения и времени строительства

### 1.3 Автоповтор задания

Как в оригинале «Полдень XXI века»: здание **запоминает последнее задание** и автоматически повторяет его, пока:
- Есть необходимые входные ресурсы на складе
- Есть энергия
- Игрок не отменил задание явно

### 1.4 Энергопотребление

Каждое здание потребляет энергию. Если энергии недостаточно:
1. Здания с **низшим приоритетом** отключаются первыми
2. Приоритет задаётся игроком (по умолчанию: добыча → переработка → производство → исследование)
3. Отключённое здание не производит, но и не потребляет ресурсы

## 2. Система добывающих зданий

Добывающие здания извлекают сырьевые элементы из природных источников планеты. Каждое здание ориентировано на определённый тип источника.

### 2.1 Шахта (Mine)

**Назначение**: добыча металлических руд из залежей планеты.

**Доступные ресурсы**: Fe, Ti, Al, Cr, Mn, Ni, Cu, Zn, Sn, Pb, Co, W, Mo, V, Ag, Pt, Au, U, Li и другие металлы.

**Формула добычи** (ед./тик):

```
production_rate = resource_availability × mine_level × gravity_bonus × workforce_efficiency
```

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| `resource_availability` | Доступность конкретного элемента на планете (0.0–1.0) | Определяется при генерации планеты |
| `mine_level` | Уровень шахты (1–10) | Целое число |
| `gravity_bonus` | Бонус гравитации | См. таблицу ниже |
| `workforce_efficiency` | Эффективность рабочих (зависит от габитабельности) | 0.1–1.0 |

**Бонус гравитации**:

| Гравитация (g) | Бонус | Обоснование |
|----------------|-------|-------------|
| < 0.3 | 0.5 | Низкая гравитация — породы рыхлые, но оборудование работает хуже |
| 0.3 – 0.7 | 0.8 | Слабая гравитация |
| 0.7 – 1.3 | 1.0 | Землеподобная — оптимальные условия |
| 1.3 – 2.0 | 1.2 | Высокая гравитация — плотные породы, больше руды |
| > 2.0 | 1.1 | Экстремальная — труднее работать, но залежи богаты |

**Ограничения**: нельзя строить на газовых гигантах.

**Пример**: Шахта уровня 5 на планете с availability(Fe) = 0.7, gravity = 1.0g, workforce = 0.9:
```
production_rate = 0.7 × 5 × 1.0 × 0.9 = 3.15 Fe-руды/тик
```

### 2.2 Карьер (Quarry)

**Назначение**: добыча неметаллических ресурсов (Si, O, C, S, P, Ca, Na, K, Mg и т.д.).

**Доступные ресурсы**: Si, O, C, S, P, Ca, Na, K, Mg, Cl, F, N (связанный в породах).

**Формула добычи** (ед./тик):

```
production_rate = resource_availability × quarry_level × workforce_efficiency
```

> **Отличие от шахты**: карьер не получает бонус гравитации — неметаллические породы менее плотные и гравитация не влияет на их концентрацию.

**Ограничения**: нельзя строить на газовых гигантах и чисто ледяных планетах (льда > 90% поверхности).

### 2.3 Газовый экстрактор (Gas Extractor)

**Назначение**: добыча газов из атмосферы планеты (H, He, N, CO₂, CH₄, NH₃, H₂S).

**Доступные ресурсы**: зависит от типа атмосферы.

| Тип атмосферы | Доступные газы | Эффективность |
|---------------|----------------|---------------|
| Нет | — | Строить нельзя |
| Тонкая | CO₂, N | 0.3 |
| Плотная | H, He, N, CO₂, O₂ | 0.7 |
| Токсичная | CO₂, H₂S, SO₂, N, CH₄ | 0.6 |
| Инертная | He, Ne, Ar, N | 0.5 |
| Газовый гигант | H, He, C (в виде CH₄), N | 1.0 |

**Формула добычи** (ед./тик):

```
production_rate = atmospheric_composition × gas_extractor_level × atmosphere_efficiency
```

| Параметр | Описание |
|----------|----------|
| `atmospheric_composition` | Доля нужного газа в атмосфере (0.0–1.0) |
| `gas_extractor_level` | Уровень экстрактора (1–10) |
| `atmosphere_efficiency` | Зависит от типа атмосферы (см. таблицу выше) |

**Ограничения**: **только на планетах с атмосферой**. На планетах без атмосферы здание построить невозможно.

### 2.4 Бурильная установка (Drilling Rig)

**Назначение**: глубокое бурение — доступ к редким элементам, залегающим на глубине.

**Доступные ресурсы**: редкие и уникальные элементы — Ir, Os, Re, Ru, Rh, Pd, Hf, Ta, Nb, Zr, Be, а также геотермальные источники.

**Формула добычи** (ед./тик):

```
production_rate = deep_availability × drill_level × geological_activity_bonus
```

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| `deep_availability` | Доступность редкого элемента в глубоких слоях (0.0–1.0) | Обычно ниже, чем поверхностная |
| `drill_level` | Уровень установки (1–10) | Каждый уровень увеличивает глубину бурения |
| `geological_activity_bonus` | Бонус геологической активности | См. таблицу ниже |

**Бонус геологической активности**:

| Тип планеты | Бонус | Обоснование |
|-------------|-------|-------------|
| Вулканическая | 1.5 | Магма выносит глубинные породы |
| Скалистая | 1.0 | Стандартные условия |
| Океаническая | 0.8 | Бурение через воду сложнее |
| Пустынная | 1.1 | Обнажённые породы |
| Ледяная | 0.6 | Лёд затрудняет бурение |

**Особенность**: на уровнях 1–4 доступна только верхняя мантия (распространённые элементы). Уровни 5–7 открывают нижнюю мантию (редкие элементы). Уровни 8–10 открывают ядро (уникальные элементы).

**Ограничения**: нельзя строить на газовых гигантах. Требует технология «Глубокое бурение» (уровень 1 для постройки, уровень 2 для уровня 5+, уровень 3 для уровня 8+).

### 2.5 Ледодобывающая станция (Ice Harvester)

**Назначение**: добыча воды и замёрзших газов на ледяных планетах, а также на планетах с полярными шапками.

**Доступные ресурсы**: H₂O (вода/лёд), замёрзший CO₂, замёрзший N₂, замёрзший CH₄, NH₃ (аммиачный лёд).

**Формула добычи** (ед./тик):

```
production_rate = ice_availability × harvester_level × temperature_bonus
```

**Бонус температуры** (парадоксально — чем холоднее, тем больше льда):

| Температура поверхности | Бонус | Обоснование |
|------------------------|-------|-------------|
| > 50°C | 0.0 | Лёд отсутствует, строить нельзя |
| 0°C – 50°C | 0.3 | Сезонный лёд / полярные шапки |
| −50°C – 0°C | 0.7 | Многолетний лёд |
| −150°C – −50°C | 1.0 | Сплошной ледяной покров |
| < −150°C | 1.2 | Толстый ледяной панцирь (Европа, Плутон) |

**Ограничения**: можно строить только на планетах со средней температурой ниже +50°C. На более горячих планетах здание недоступно.

### 2.6 Сводная таблица добывающих зданий

| Здание | ID | Источник | Ключевые ресурсы | Требование к планете | Тех-требование |
|--------|-----|----------|-------------------|---------------------|-----------------|
| Шахта | `mine` | Поверхностные залежи | Fe, Ti, Al, Cr, Ni… | Не газовый гигант | Нет |
| Карьер | `quarry` | Поверхностные породы | Si, O, C, S, Ca… | Не газовый гигант, не чистый лёд | Нет |
| Газовый экстрактор | `gas_extractor` | Атмосфера | H, He, N, CO₂, CH₄… | Атмосфера ≠ «нет» | Нет |
| Бурильная установка | `drilling_rig` | Глубокие слои | Ir, Os, Re, Ru, Hf… | Не газовый гигант | «Глубокое бурение» |
| Ледодобывающая станция | `ice_harvester` | Лёд и замёрзшие газы | H₂O, CO₂(лёд), N₂(лёд)… | Температура < +50°C | Нет |

## 3. Автоматизированные линии переработки

Перерабатывающие здания превращают сырьё (руды, газы, лёд) в чистые элементы и материалы. Каждая линия — цепочка связанных зданий.

### 3.1 Плавильня (Smelter)

**Назначение**: руда → металл. Переплавка руды в чистый металл.

**Рецепты**:

| Вход | Выход | Энергия/ед. | Мин. уровень |
|------|-------|-------------|--------------|
| Fe-руда (10) | Fe (8) | 2 | 1 |
| Ti-руда (10) | Ti (7) | 3 | 1 |
| Al-руда (10) | Al (7) | 3 | 1 |
| Ni-руда (10) | Ni (6) | 3 | 2 |
| Cr-руда (10) | Cr (6) | 3 | 2 |
| Cu-руда (10) | Cu (7) | 2 | 1 |
| W-руда (10) | W (5) | 5 | 4 |
| Co-руда (10) | Co (5) | 4 | 3 |
| Pb-руда (10) | Pb (7) | 2 | 1 |
| Sn-руда (10) | Sn (7) | 2 | 1 |

**Формула выхода** (ед./тик):

```
smelting_rate = base_recipe_rate × smelter_level × workforce_efficiency × adjacency_bonus
```

> Коэффициент выхода < 1.0 отражает потери при переработке (шлак). На более высоких уровнях плавильни потери снижаются (см. раздел 9).

**Ограничения**: не может перерабатывать неметаллические руды. Требует входной ресурс на складе.

### 3.2 Химический завод (Chemical Plant)

**Назначение**: базовые химические реакции — разложение и синтез простых соединений.

**Рецепты**:

| Вход | Выход | Энергия/ед. | Мин. уровень |
|------|-------|-------------|--------------|
| H₂O (10) | H₂ (8) + O₂ (8) | 4 | 1 |
| Атмосферный CO₂ (10) | C (5) + O₂ (9) | 5 | 1 |
| CaCO₃ (10) | Ca (4) + C (2) + O₂ (6) | 3 | 2 |
| NaCl (10) | Na (4) + Cl (6) | 4 | 2 |
| Fe₂O₃ (10) | Fe (7) + O₂ (3) | 3 | 1 |
| NH₃ (10) | N (8) + H₂ (12) | 3 | 2 |
| H₂S (10) | H₂ (6) + S (8) | 2 | 1 |

**Формула** (ед./тик):

```
reaction_rate = base_recipe_rate × plant_level × workforce_efficiency
```

**Особенность**: химический завод не требует смежности с источником сырья — он берёт ресурсы со склада. Но смежность даёт бонус (см. раздел 5).

### 3.3 Нефтехимический завод (Petrochem Plant)

**Назначение**: производство сложных органических соединений — пластик, топливо, смазочные материалы.

**Рецепты**:

| Вход | Выход | Энергия/ед. | Мин. уровень |
|------|-------|-------------|--------------|
| C (5) + H (8) | Пластик (6) | 2 | 1 |
| C (8) + H (12) + S (1) | Синтетическое топливо (10) | 3 | 2 |
| C (3) + H (4) + N (2) | Композитный материал (4) | 4 | 4 |
| C (6) + H (6) + O (3) | Органический растворитель (5) | 2 | 3 |
| Пластик (4) + W (1) | Армированный пластик (3) | 5 | 6 |

**Ограничения**: требует технологию «Нефтехимия» (уровень 1).

### 3.4 Очистительный комплекс (Refinery)

**Назначение**: двухступенчатая переработка — сырая руда → концентрат → чистый элемент. Более высокий выход, чем у плавильни, но медленнее.

**Ступени**:

1. **Концентрация**: сырая руда → концентрат (удаление пустой породы). Выход 50% от массы руды.
2. **Очистка**: концентрат → чистый элемент. Выход 90% от концентрата.

**Итоговый выход**: 0.5 × 0.9 = **45%** от массы руды — но это чистый элемент, в отличие от плавильни, где выход ниже и ниже чистота.

**Рецепты**:

| Вход (сырая руда) | Промежуточный (концентрат) | Выход (чистый элемент) | Энергия/ед. | Мин. уровень |
|--------------------|-----------------------------|------------------------|-------------|--------------|
| Fe-руда (20) | Fe-концентрат (10) | Fe (9) | 6 | 1 |
| Ti-руда (20) | Ti-концентрат (10) | Ti (9) | 7 | 2 |
| Au-руда (20) | Au-концентрат (10) | Au (9) | 10 | 3 |
| Pt-руда (20) | Pt-концентрат (10) | Pt (9) | 12 | 5 |
| U-руда (20) | U-концентрат (10) | U (9) | 15 | 5 |

**Сравнение с плавильней**:

| Параметр | Плавильня | Очистительный комплекс |
|----------|-----------|------------------------|
| Скорость | Высокая | Низкая (2× медленнее) |
| Выход (ед. чистого / ед. руды) | 0.6–0.8 | 0.45, но высшая чистота |
| Качество продукта | Технической чистоты | Высшей чистоты |
| Энергозатраты | Умеренные | Высокие |
| Кому нужен | Для строительства | Для электроники, сверхпроводников |

> **Для программиста**: «чистота» элемента — это числовой параметр 0.0–1.0. Плавильня даёт чистоту 0.7–0.85. Очистительный комплекс — 0.95–0.99. Завод электроники требует чистоту ≥ 0.9.

### 3.5 Цепочки зданий на гекс-сетке

Каждая перерабатывающая линия состоит из **трёх связанных зданий**:

```
[Входное хранилище] → [Здание переработки] → [Выходное хранилище]
```

| Звено цепочки | Здание | Функция |
|---------------|--------|---------|
| Входное хранилище | Склад (спец.) | Принимает сырьё от добывающих зданий |
| Переработка | Плавильня / Химзавод / и т.д. | Превращает сырьё в продукт |
| Выходное хранилище | Склад (спец.) | Хранит готовый продукт до передачи дальше |

**Правила связи**:
- Здания должны быть соединены **конвейером** или **автотранспортом**
- Если здания **смежны** (соседние гексы), конвейер не нужен — ресурсы передаются мгновенно
- При несмежном размещении требуется логистическая связь (см. раздел 4)

## 4. Логистика

> **ВАЖНО**: не переусложнять! Логистика — это автоматическая доставка ресурсов между зданиями. Игрок не управляет каждым грузовиком. Он задаёт связи и приоритеты, система работает сама.

### 4.1 Конвейерная лента (Conveyor)

**Назначение**: связь между зданиями на одной планете по гекс-сетке.

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Скорость передачи | Мгновенно (в рамках игрового тика) |
| Пропускная способность | 50 ед./тик на уровень конвейера |
| Дальность | Только между **смежными** гексами |
| Стоимость строительства | 5 ед. конструкционных материалов + 1 ед. энергии/тик |
| Уровни | 1–5 |

**Ключевое правило**: конвейер **бесполезен**, если здания не смежные. Он не тянется через несколько гексов. Для связи удалённых зданий используйте автотранспорт.

**Визуализация**: на карте планеты конвейер отображается как линия между двумя гексами.

### 4.2 Автоматический транспорт (Auto-Transport)

**Назначение**: роботизированные грузовики между удалёнными зданиями на одной планете.

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Скорость доставки | Зависит от расстояния: `delivery_time = distance_hexes × 2` тиков |
| Пропускная способность | 200 ед./рейс |
| Дальность | В пределах планеты (любое расстояние по гекс-сетке) |
| Стоимость строительства | 20 ед. конструкционных материалов + 2 ед. энергии/тик |
| Уровни | 1–3 (определяют количество одновременных рейсов: 1/2/3) |

**Отличие от конвейера**: автотранспорт медленнее (задержка доставки), но работает на любое расстояние.

### 4.3 Склад (Warehouse)

**Назначение**: буфер хранения ресурсов между звеньями производственной цепочки.

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Вместимость (ур. 1) | 500 ед. |
| Вместимость (ур. 10) | 10 000 ед. |
| Формула вместимости | `500 × level` |
| Тип хранения | Универсальный или специализированный |
| Энергопотребление | 1 ед./тик (независимо от заполненности) |

**Специализация склада**:

Игрок может настроить склад на хранение определённых типов ресурсов:

| Тип склада | Что хранит | Бонус |
|------------|-----------|-------|
| Универсальный | Любые ресурсы | Нет |
| Рудный | Только сырая руда | +10% вместимость для руды |
| Металлический | Только чистые металлы | +10% вместимость для металлов |
| Газовый | Только газы | +10% вместимость для газов |
| Компонентный | Только компоненты и модули | +10% вместимость для компонентов |

### 4.4 Очередь приоритетов

Когда ресурс нужен нескольким зданиям одновременно, система использует **очередь приоритетов**.

**Приоритеты по умолчанию** (от высшего к низшему):

| Приоритет | Тип здания | Обоснование |
|-----------|-----------|-------------|
| 1 | Энергетика (электростанции) | Без энергии всё останавливается |
| 2 | Добыча (шахты, карьеры) | Без сырья нечего перерабатывать |
| 3 | Переработка (плавильни, химзаводы) | Промежуточное звено |
| 4 | Производство (заводы компонентов) | Конечный продукт |
| 5 | Верфи | Долгосрочные проекты |
| 6 | Исследование (лаборатории) | Не критично в моменте |

**Настройка**: игрок может переназначить приоритеты для каждого склада индивидуально. Например, если срочно нужен корабль — поднять верфь на приоритет 1.

**Механика**:
1. Склад получает 100 ед. Fe
2. Fe запрошен Плавильней (приоритет 3) и Заводом компонентов (приоритет 4)
3. Fe сначала отправляется в Плавильню, остаток — на Завод компонентов
4. Если Fe не хватает — Завод компонентов ждёт

### 4.5 Межпланетная логистика

**Назначение**: автоматическая доставка ресурсов между планетами с помощью транспортных кораблей.

**Компоненты**:

| Элемент | Описание |
|---------|----------|
| **Транспортный корабль** | Грузовой корабль с грузовым отсеком. Строится на верфи |
| **Торговый маршрут** | Настройка: откуда → куда, что везти, количество |
| **Точка загрузки** | Орбитальный склад или космопорт на планете-отправителе |
| **Точка выгрузки** | Орбитальный склад или космопорт на планете-получателе |

**Типы маршрутов**:

| Тип | Описание | Когда используется |
|-----|----------|-------------------|
| **Автоматический** | Система сама создаёт маршруты для обеспечения дефицита | По умолчанию |
| **Настраиваемый** | Игрок задаёт конкретный маршрут с параметрами | Для оптимизации |
| **Экстренный** | Одноразовая доставка, максимальный приоритет | При кризисе |

**Формула времени доставки**:

```
delivery_time_ticks = (distance_between_systems × warp_speed_factor) + orbit_departure_time + orbit_arrival_time
```

| Параметр | Значение |
|----------|----------|
| `distance_between_systems` | Расстояние в парсеках |
| `warp_speed_factor` | Зависит от двигателя: 1 тик/парсек (химический), 0.3 (ионный), 0.1 (варп) |
| `orbit_departure_time` | 10 тиков |
| `orbit_arrival_time` | 10 тиков |

**Космопорт**: здание на планете (уровни 1–5), определяет сколько транспортных кораблей могут одновременно загружаться/разгружаться. Уровень 1 = 1 корабль, уровень 5 = 5 кораблей.

## 5. Бонусы смежных зданий

Смежные здания (расположенные в соседних гексах) дают **бонусы соседства** (adjacency bonuses). Это стимул планировать застройку и создавать производственные кластеры.

### 5.1 Таблица бонусов

| Комбинация | Бонус | Условие |
|------------|-------|---------|
| Шахта + Плавильня | **+15%** скорость переработки | Оба здания смежны |
| Плавильня + Завод компонентов | **+10%** скорость производства | Оба здания смежны |
| Электростанция + Любое здание | **−5%** энергопотребление | Электростанция смежна с потребителем |
| Склад + Производство | **+20%** скорость загрузки/разгрузки | Склад смежен с производственным зданием |
| Лаборатория + Лаборатория | **+10%** скорость исследований | Кластерный эффект (2+ лаборатории смежны) |

### 5.2 Правила стекания бонусов

Бонусы **стекаются** (stack), но с убывающей отдачей:

- Если здание получает **один** бонус от одного соседа — полная величина
- Если здание получает **два и более** одинаковых бонуса от разных соседей — каждый следующий бонус уменьшается

**Формула стекания**:

```
total_bonus = bonus_1 + (bonus_2 × 0.5) + (bonus_3 × 0.25) + ... + (bonus_n × 0.5^(n-1))
```

**Пример**: Плавильня окружена тремя шахтами:
```
total_bonus = 0.15 + (0.15 × 0.5) + (0.15 × 0.25) = 0.15 + 0.075 + 0.0375 = 0.2625 = +26.25%
```

> Без убывающей отдачи было бы +45%, что слишком мощно.

### 5.3 Условия действия бонусов

1. **Оба здания должны быть активны** (не отключены из-за нехватки энергии)
2. **Оба здания должны быть построены** (не в процессе строительства)
3. **Бонус применяется мгновенно** при появлении смежности
4. **Бонус исчезает** при сносе или отключении одного из зданий
5. **Бонус электростанции** распространяется только на одно смежное здание. Если электростанция смежна с тремя зданиями, каждое получает −5% энергопотребления (электростанция даёт бонус каждому соседу)

### 5.4 Оптимальные конфигурации (спойлер для игрока)

Несколько эффективных схем размещения:

**Горнодобывающий кластер**:
```
    [Шахта]
[Шахта] [Плавильня] [Склад]
    [Шахта]
```
Плавильня получает +26.25% к переработке. Склад даёт +20% загрузки.

**Научный кластер**:
```
[Лаборатория] [Лаборатория]
[Лаборатория] [Лаборатория]
```
Каждая лаборатория получает +10% от 2 смежных: `0.10 + 0.05 = +15%`

**Энергосберегающий ряд**:
```
[Электростанция] [Шахта] [Плавильня] [Завод]
```
Шахта, Плавильня и Завод получают по −5% энергопотребления. Плавильня + Завод дают друг другу +10% и +15% бонусов.

## 7. Заводы производства частей космических кораблей

### 7.1 Верфь (Shipyard)

**Назначение**: сборка кораблей из модулей.

**Уровни верфи и доступные размеры кораблей**:

| Уровень верфи | Доступные корпуса | Классы кораблей |
|---------------|-------------------|-----------------|
| 1 | Лёгкий | Разведчик, Истребитель, Шаттл |
| 2 | Лёгкий | + Перехватчик |
| 3 | Лёгкий, Средний | + Фрегат, Транспорт |
| 4 | Лёгкий, Средний | + Эсминец |
| 5 | Лёгкий, Средний | + Крейсер |
| 6 | Средний | + Тяжёлый крейсер |
| 7 | Средний, Тяжёлый | + Линкор |
| 8 | Тяжёлый | + Авианосец |
| 9 | Тяжёлый, Сверхтяжёлый | + Дредноут |
| 10 | Сверхтяжёлый | + Титан |

**Параллельная сборка**:

| Уровень верфи | Одновременных проектов |
|---------------|------------------------|
| 1–3 | 1 |
| 4–6 | 2 |
| 7–8 | 3 |
| 9–10 | 4 |

**Очередь строительства**: верфь имеет очередь из модулей для сборки. Каждый модуль собирается за определённое время, зависящее от сложности.

**Формула времени сборки модуля**:

```
assembly_time_ticks = base_module_time / (shipyard_level × workforce_efficiency × adjacency_bonus)
```

`base_module_time` — базовое время сборки модуля (определяется рецептом модуля).

### 7.2 Орбитальная верфь (Orbital Shipyard)

**Назначение**: верфь, строящая корабли прямо в космосе. Не занимает гекс на поверхности планеты.

**Отличия от планетарной верфи**:

| Параметр | Планетарная верфь | Орбитальная верфь |
|----------|-------------------|-------------------|
| Расположение | Гекс на планете | Орбита планеты |
| Стоимость строительства | Дешевле | Дороже (×2 ресурсы) |
| Доступные размеры | Ограничены уровнем | +1 уровень эквивалента (ур. 5 = средние + лёгкий тяжёлый) |
| Время сборки | Базовое | −20% (нет гравитационных ограничений) |
| Требование | Нет | Космопорт ур. 3+, Инженерия 4 |
| Максимальное количество | 1 на планете | Зависит от размера орбиты (1–3) |

**Построение орбитальной верфи**:
1. На планетарной верфи (ур. 4+) собираются секции орбитальной верфи
2. Секции запускаются на орбиту (10 тиков на секцию)
3. Секции собираются в орбитальную верфь (занимает 100 тиков)

**Ограничение**: орбитальная верфь не может строить другую орбитальную верфь.

## 8. Энергетика

Электростанции — единственный источник энергии. Без энергии все здания останавливаются.

### 8.1 Солнечная станция (Solar Plant)

**Назначение**: выработка энергии из звёздного излучения.

**Формула мощности** (ед./тик):

```
power_output = base_output × solar_station_level × star_luminosity / distance_factor
```

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| `base_output` | Базовая мощность | 10 ед./тик |
| `solar_station_level` | Уровень станции (1–10) | Целое число |
| `star_luminosity` | Светимость звезды | 0.01 (красный карлик) – 10 000 (голубой гигант) |
| `distance_factor` | Квадрат расстояния от звезды | `distance_au²` |

**Эффективность по типу звезды** (на расстоянии 1 АЕ):

| Тип звезды | Светимость | Мощность станции ур. 1 |
|------------|-----------|----------------------|
| Красный карлик (M) | 0.01 | 0.1 ед./тик |
| Жёлтый карлик (G) | 1.0 | 10 ед./тик |
| Белый карлик | 0.01 | 0.1 ед./тик |
| Красный гигант | 100 | 1 000 ед./тик |
| Голубой гигант | 10 000 | 100 000 ед./тик |

**Ограничения**: не работает на планетах без солнечного освещения (глубокие подземелья, обратная сторона приливно-захваченных планет). Эффективность снижается на планетах с плотной атмосферой (−20%).

### 8.2 Ядерный реактор (Nuclear Reactor)

**Назначение**: постоянная мощность, потребляет U (уран) или Th (торий).

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Базовая мощность (ур. 1) | 50 ед./тик |
| Максимальная мощность (ур. 10) | 500 ед./тик |
| Формула мощности | `50 × reactor_level` |
| Расход топлива | `0.01 × reactor_level` ед. U/Th за тик |
| Тип топлива | U или Th |

**Преимущества**: постоянная мощность, не зависит от расстояния до звезды, работает на любой планете.

**Недостатки**: потребляет редкое топливо (U, Th), риск аварии (см. ниже).

**Риск аварии**: при уровне выше 7 и износе > 80% — шанс аварии 0.01% за тик. Авария выводит реактор из строя на 200 тиков и наносит урон смежным зданиям.

### 8.3 Термоядерный реактор (Fusion Reactor)

**Назначение**: высокая мощность, потребляет H (водород), D (дейтерий), T (тритий).

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Базовая мощность (ур. 1) | 200 ед./тик |
| Максимальная мощность (ур. 10) | 2 000 ед./тик |
| Формула мощности | `200 × reactor_level` |
| Расход топлива | `0.1 × reactor_level` ед. H/D/T за тик |
| Тип топлива | H, D (дейтерий), T (тритий) — чем тяжелее, тем мощнее |

**Модификаторы топлива**:

| Топливо | Бонус к мощности |
|---------|-----------------|
| H (водород) | ×1.0 |
| D (дейтерий) | ×1.5 |
| T (тритий) | ×2.0 |
| D+T смесь | ×3.0 |

**Требование**: технология «Термоядерный синтез» (уровень 1 для постройки, уровень 2 для ур. 4+, уровень 3 для ур. 7+).

**Преимущества**: в 4 раза мощнее ядерного, топливо распространено (H есть везде).

**Недостатки**: дорого в строительстве, требует высокую технологию.

### 8.4 Геотермальная станция (Geothermal Plant)

**Назначение**: дешёвая энергия из недр планеты.

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Базовая мощность (ур. 1) | 80 ед./тик |
| Максимальная мощность (ур. 10) | 800 ед./тик |
| Формула мощности | `80 × plant_level × geological_activity` |
| Расход топлива | Нет |
| Энергопотребление | 0 |

**Модификатор геологической активности**:

| Тип планеты | geological_activity | Можно строить? |
|-------------|---------------------|----------------|
| Вулканическая | 1.5 | Да (идеально) |
| Скалистая (тектонически активная) | 1.0 | Да |
| Океаническая | 0.8 | Да |
| Скалистая (тектонически неактивная) | 0.4 | Да, но слабо |
| Пустынная | 0.5 | Да |
| Ледяная | 0.1 | Нет (недостаточно тепла) |
| Газовый гигант | 0.0 | Нет |

**Ограничение**: **только на тектонически активных планетах**. На ледяных планетах и газовых гигантах строить нельзя.

**Преимущества**: бесплатное топливо, дешёвое строительство.

**Недостатки**: ограничена типом планеты, мощность ниже термоядерной.

### 8.5 Антиматериальный реактор (Antimatter Reactor)

**Назначение**: эндгейм — максимальная мощность.

**Характеристики**:

| Параметр | Значение |
|----------|----------|
| Базовая мощность (ур. 1) | 1 000 ед./тик |
| Максимальная мощность (ур. 10) | 10 000 ед./тик |
| Формула мощности | `1 000 × reactor_level` |
| Расход топлива | `0.001 × reactor_level` ед. антиматерии за тик |
| Тип топлива | Антиматерия (производится на спец. установке) |

**Требование**: технология «Антиматерия» (уровень 1 для постройки, уровень 2 для ур. 5+, уровень 3 для ур. 8+).

**Производство антиматерии**: требуется «Генератор антиматерии» (отдельное здание, строится только при наличии технологии «Антиматерия» ур. 2+). Выработка: 0.01 ед. антиматерии/тик при ур. 1. Потребляет 500 ед. энергии/тик (парадокс — нужно много энергии, чтобы создать антиматерию).

**Риск аварии**: при уровне выше 5 и износе > 80% — шанс аварии 0.05% за тик. Авария уничтожает реактор и наносит тяжёлый урон всем зданиям в радиусе 3 гексов.

**Преимущества**: абсолютный максимум мощности.

**Недостатки**: крайне дорогой в производстве и эксплуатации, высокий риск аварии, требует редчайшую технологию.

|----------------|---------------|----------------|---------|----------------|-----------------|
| Солнечная | 0.1–100 000* | ×10 | Нет | Нет | Зависит от звезды |

## 9. Система уровней зданий

### 9.1 Общие правила

- Все здания имеют **уровни от 1 до 10**
- Повышение уровня — это **модернизация** существующего здания (не снос и перестройка)
- Модернизация занимает время и требует ресурсы
- Время модернизации: `upgrade_time = base_build_time × level` тиков

### 9.2 Стоимость модернизации

Формула стоимости ресурсов для улучшения с уровня N до N+1:

```
upgrade_cost(resource) = base_cost(resource) × N × 1.2
```

Где `base_cost` — стоимость постройки здания уровня 1.

**Пример**: Шахта уровня 1 стоит 20 Fe, 10 Si. Улучшение до уровня 3:
```
уровень 1→2: (20 × 1 × 1.2) Fe + (10 × 1 × 1.2) Si = 24 Fe + 12 Si
уровень 2→3: (20 × 2 × 1.2) Fe + (10 × 2 × 1.2) Si = 48 Fe + 24 Si
```

### 9.3 Эффекты уровней

| Уровень | Скорость работы | Энергопотребление /ед. | Вместимость (склад) | Доступные рецепты |
|---------|-----------------|------------------------|---------------------|-------------------|
| 1 | ×1.0 | ×1.0 | ×1 | Базовые |
| 2 | ×1.2 | ×0.95 | ×2 | +1 рецепт |
| 3 | ×1.5 | ×0.90 | ×3 | +1 рецепт |
| 4 | ×1.8 | ×0.85 | ×4 | +1 рецепт |
| 5 | ×2.2 | ×0.80 | ×5 | +2 рецепта |
| 6 | ×2.7 | ×0.75 | ×6 | +1 рецепт |
| 7 | ×3.2 | ×0.70 | ×7 | +1 рецепт |
| 8 | ×3.8 | ×0.65 | ×8 | +2 рецепта |
| 9 | ×4.5 | ×0.60 | ×9 | +1 рецепт |
| 10 | ×5.0 | ×0.55 | ×10 | +1 рецепт |

**Формула скорости работы**:
```
work_speed = base_speed × level_multiplier(level)
```

**Формула энергоэффективности**:
```
energy_per_unit = base_energy × energy_multiplier(level)
```

> Энергоэффективность означает, что на высоких уровнях здание тратит меньше энергии на единицу продукции. Общее потребление энергии растёт (здание работает быстрее), но удельное — падает.

### 9.4 Потери при переработке

Уровень перерабатывающего здания также влияет на **коэффициент выхода** (yield):

| Уровень | Yield (плавильня) | Yield (очистительный) |
|---------|-------------------|-----------------------|
| 1 | 0.60 | 0.450 |
| 2 | 0.63 | 0.455 |
| 3 | 0.66 | 0.460 |
| 4 | 0.69 | 0.465 |
| 5 | 0.72 | 0.470 |
| 6 | 0.75 | 0.475 |
| 7 | 0.78 | 0.480 |
| 8 | 0.81 | 0.485 |
| 9 | 0.84 | 0.490 |
| 10 | 0.85 | 0.500 |

**Формула yield для плавильни**:
```
yield = 0.60 + 0.03 × (level - 1)   (максимум 0.85 на уровне 10)
```

**Формула yield для очистительного комплекса**:
```
yield = 0.450 + 0.005 × (level - 1)   (максимум 0.500 на уровне 10)
```

> Очистительный комплекс на уровне 10 даёт 50% чистого элемента из руды высшей чистоты — это лучше, чем плавильня (85% технической чистоты), когда нужна высшая чистота.

## 11. Формулы и расчёты

### 11.1 Полная формула добычи

```
final_production = base_production × level_multiplier
                   × planet_modifier
                   × workforce_efficiency
                   × adjacency_bonus
                   × energy_efficiency
```

| Множитель | Диапазон | Источник |
|-----------|----------|----------|
| `base_production` | 1.0 | Базовый параметр здания |
| `level_multiplier` | 1.0–5.0 | Таблица из раздела 9.3 |
| `planet_modifier` | 0.1–1.5 | Зависит от типа здания и планеты |
| `workforce_efficiency` | 0.1–1.0 | `= habitability / 100` |
| `adjacency_bonus` | 1.0–1.3+ | См. раздел 5 |
| `energy_efficiency` | 0.0–1.0 | `= min(1.0, available_energy / required_energy)` |

### 11.2 Формула энергопотребления здания

```
actual_energy_cost = base_energy × level_multiplier / energy_efficiency_per_level
```

| Параметр | Описание |
|----------|----------|
| `base_energy` | Базовое энергопотребление здания уровня 1 |
| `level_multiplier` | Множитель скорости (здание быстрее работает → больше потребляет) |
| `energy_efficiency_per_level` | Уменьшение удельного потребления (0.55 на ур. 10) |

**Пример**: Плавильня уровня 5:
```
actual_energy = 10 × 2.2 / 0.80 = 27.5 ед./тик
```

### 11.3 Формула стоимости строительства

```
build_cost(resource) = base_cost(resource) × (1 + 0.2 × (level - 1))
build_time_ticks = base_build_time × level
```

### 11.4 Износ зданий

Каждое здание имеет **прочность** (hit points), которая медленно снижается со временем:

```
wear_per_tick = 0.001   (базовый износ)
wear_modifier = 1.0     (нормальные условия)
                   1.5  (токсичная атмосфера)
                   2.0  (вулканическая планета)
                   0.5  (герметичное здание на комфортной планете)
```

При прочности < 30% — здание работает с штрафом −50% скорости.
При прочности 0% — здание останавливается.

**Ремонт**: стоит 10% от стоимости строительства, восстанавливает 25% прочности. Можно заказать автоматический ремонт при прочности < 50%.
