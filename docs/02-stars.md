## 1. Классификация звёзд (спектральные классы)

Звёзды классифицируются по спектральному классу (гарвардская классификация), который определяется температурой фотосферы. Основная последовательность — **O → B → A → F → G → K → M** — от горячих голубых звёзд к холодным красным.

Каждый спектральный класс делится на 10 подклассов (0–9), где 0 — самый горячий в классе, 9 — самый холодный. Например, Солнце — G2V.

В игре используются **целые классы** (без подклассов) для упрощения генерации, но при отображении игроку подкласс генерируется косметически.

### 1.3 Косметическая генерация подкласса

При отображении звезды игроку генерируется подкласс (0–9) для реалистичности:

```
подкласс = floor(random() * 10)   // 0–9, равномерно
```

Полная метка звезды: `{класс}{подкласс}V` для звёзд главной последовательности.  
Пример: `G2V` (Солнце), `M5V` (типичный красный карлик), `B3V`.

Значение подкласса используется для лёгкой вариации параметров внутри класса:

```
// t — температура подкласса, линейная интерполяция
t_min = T_min_класса + подкласс * (T_max_класса - T_min_класса) / 10
t_max = T_min_класса + (подкласс + 1) * (T_max_класса - T_min_класса) / 10
T_фактическая = lerp(t_min, t_max, random())

// Аналогично для массы, радиуса, светимости (в логарифмической шкале)
```

## 2. Специальные типы звёзд

Помимо основной последовательности, в галактике встречаются звёзды на поздних стадиях эволюции и экзотические объекты. Они добавляют разнообразие и уникальный игровой опыт.

### 2.1 Сводная таблица специальных типов

| Тип | Игровой ID | Частота (%) | Масса (M☉) | Светимость (L☉) | Температура (K) | Радиус (R☉) |
|---|---|---|---|---|---|---|
| Белый карлик | `STAR_WD` | 0.4 | 0.5–1.4 | 0.001–0.01 | 8 000–40 000 | 0.008–0.02 |
| Красный гигант | `STAR_RG` | 0.2 | 0.5–8 | 100–10 000 | 3 000–5 000 | 10–200 |
| Нейтронная звезда | `STAR_NS` | 0.1 | 1.1–2.1 | ≈0 (в оптике) | ≈600 000 | 0.00001 |
| Пульсар | `STAR_PULSAR` | 0.05 | 1.1–2.1 | Пульсирующая | ≈1 000 000 | 0.00001 |
| Чёрная дыра | `STAR_BH` | 0.05 | 3–50 | 0 (аккреция) | — | ~0 (горизонт событий) |

> **Итого специальные типы**: 0.4 + 0.2 + 0.1 + 0.05 + 0.05 = 0.8%. Вместе с основной последовательностью (99.203%) суммарно ≈100%.

### 2.2 Белый карлик (`STAR_WD`)

**Описание**: Остаток звезды средней массы после сброса оболочки. Очень горячий, но очень маленький — низкая светимость.

| Свойство | Значение |
|---|---|
| **Масса** | 0.5–1.4 M☉ (среднее 0.6 M☉) |
| **Радиус** | ≈ R⊕ (0.008–0.02 R☉) |
| **Температура** | 8 000–40 000 K (остывает миллиарды лет) |
| **Светимость** | 0.001–0.01 L☉ |

**Влияние на планетную систему**:
- **Планеты**: 0–3 (большинство потеряно при сбросе оболочки)
- **Зона обитаемости**: практически отсутствует — слишком тусклый, зона сузилась до узкой полосы
- **Астероидные поля**: 1–3 (остатки разрушенных планет — богатые редкими элементами)
- **Кометное облако**: отсутствует (развеялось)

**Уникальная механика**: Белые карлики — источник редких тяжёлых элементов. Остатки разрушенных планет富含 платиноидов (Pt, Ir, Os, Pd). Добыча на астероидах вокруг белых карликов — один из лучших источников уникальных ресурсов.

**Формула температуры (охлаждение)**:
```
T_поверхности = T_начальное / sqrt(1 + возраст / τ_охлаждения)
где τ_охлаждения ≈ 1 млрд лет
T_начальное = 30 000 K (типично)
```

### 2.3 Красный гигант (`STAR_RG`)

**Описание**: Звезда на поздней стадии эволюции, расширившаяся после выгорания водорода в ядре.

| Свойство | Значение |
|---|---|
| **Масса** | 0.5–8 M☉ (среднее 1.5 M☉) |
| **Радиус** | 10–200 R☉ (среднее 50 R☉) |
| **Температура** | 3 000–5 000 K |
| **Светимость** | 100–10 000 L☉ |

**Влияние на планетную систему**:
- **Планеты**: 1–5 (внутренние планеты поглощены при расширении; остались только дальние)
- **Зона обитаемости**: формально далеко от звезды, но нестабильна — звезда нестабильна
- **Астероидные поля**: 0–2
- **Кометное облако**: частично разрушено

**Уникальная механика**: Красные гиганты — нестабильные системы. Внутренние планеты уничтожены (поглощены), но дальние планеты нагреты — ранее ледяные планеты могли стать обитаемыми на короткое время (эффект «отложенной обитаемости»). Однако нестабильность звезды означает, что это временно.

**Событие «Сброс оболочки»**: С вероятностью 5% за игровую кампанию красный гигант может сбросить оболочку, превратившись в белый карлик. При этом:
- Все планеты в радиусе < 3 АЕ уничтожаются
- Астероидные поля обновляются (богатый материал оболочки)
- Тип звезды меняется на `STAR_WD`

### 2.4 Нейтронная звезда (`STAR_NS`)

**Описание**: Сверхплотный остаток массивной звезды после вспышки сверхновой.

| Свойство | Значение |
|---|---|
| **Масса** | 1.1–2.1 M☉ (среднее 1.4 M☉) |
| **Радиус** | ≈10 км (0.00001 R☉) |
| **Температура** | ≈600 000 K (поверхность) |
| **Светимость** | Пренебрежимо мала в оптике (рентгеновское излучение) |

**Влияние на планетную систему**:
- **Планеты**: 0–2 (почти все уничтожены при вспышке сверхновой)
- **Зона обитаемости**: отсутствует
- **Астероидные поля**: 1–4 (остатки сверхновой — богатейшие уникальными элементами)
- **Кометное облако**: уничтожено

**Уникальная механика**: 
- **Урон излучением**: корабли в системе получают урон 1 HP/тик на расстоянии < 5 АЕ от нейтронной звезды. Щиты снижают урон на 50%.
- **Ресурсный рай**: астероидные поля вокруг нейтронных звёзд — лучшие источники уникальных элементов (Ir, Os, Re, Ru, Rh). Концентрация уникальных ресурсов ×3 по сравнению с обычными системами.

### 2.5 Пульсар (`STAR_PULSAR`)

**Описание**: Нейтронная звезда с сильным магнитным полем, излучающая направленные пучки радиации.

| Свойство | Значение |
|---|---|
| **Масса** | 1.1–2.1 M☉ |
| **Радиус** | ≈10 км |
| **Период вращения** | 0.001–10 секунд |
| **Магнитное поле** | 10⁸–10¹⁵ Тл |

**Влияние на планетную систему**: Как у нейтронной звезды, но с дополнительными эффектами.

**Уникальная механика**:
- **Пульсирующее излучение**: урон кораблям в 3 раза выше, чем от обычной нейтронной звезды (3 HP/тик в радиусе 5 АЕ)
- **Нарушение навигации**: корабли в системе двигаются на 50% медленнее (помехи электронике)
- **Научная ценность**: лаборатории на станциях в системе пульсара получают +50% к скорости исследований физики
- **Астероидные поля**: 2–4, концентрация уникальных ресурсов ×5

### 2.6 Чёрная дыра (`STAR_BH`)

**Описание**: Объект с горизонтом событий, из которого не может вырваться даже свет.

| Свойство | Значение |
|---|---|
| **Масса** | 3–50 M☉ (звёздная чёрная дыра) |
| **Радиус горизонта событий** | 2GM/c² (≈9 км для 3 M☉, ≈150 км для 50 M☉) |
| **Светимость** | 0 (без аккреции) |

**Влияние на планетную систему**:
- **Планеты**: 0–1 (почти всё уничтожено или захвачено)
- **Зона обитаемости**: отсутствует
- **Астероидные поля**: 0–2
- **Кометное облако**: отсутствует

**Уникальная механика**:
- **Гравитационный захват**: корабли ближе 1 АЕ к горизонту событий не могут выйти на варп-скорость
- **Разрушение ближе 0.1 АЕ**: корабли уничтожаются без возможности спасения
- **Аккреционный диск**: при наличии вещества вокруг чёрной дыры формируется аккреционный диск, видимый как яркое кольцо
- **Исследовательская аномалия**: научные корабли могут провести уникальное исследование рядом с чёрной дырой, дающее +1 случайную технологию из ветви «Физика» (одноразово)

### 2.7 Двойные и тройные звёздные системы

**Описание**: Системы с двумя или тремя звёздами, гравитационно связанными друг с другом.

| Тип системы | Игровой ID | Частота среди всех систем (%) |
|---|---|---|
| Одиночная | `BINARY_NONE` | 60 |
| Двойная (близкая) | `BINARY_CLOSE` | 20 |
| Двойная (широкая) | `BINARY_WIDE` | 15 |
| Тройная | `BINARY_TRIPLE` | 5 |

**Определения**:
- **Близкая двойная**: компоненты на расстоянии < 1 АЕ друг от друга; планеты обращаются вокруг центра масс обеих звёзд (циркумбинарные орбиты)
- **Широкая двойная**: компоненты на расстоянии 10–1000 АЕ; каждая звезда имеет собственную планетную систему
- **Тройная**: три компонента; обычно — близкая пара + удалённый третий компонент

**Комбинации звёзд в двойных/тройных системах**:

```
// Алгоритм выбора компонентов
// Первый компонент: обычная генерация по частотам
// Второй компонент: с вероятностью 70% — тот же класс или на 1 класс ниже,
//   с вероятностью 30% — случайный класс
```

| Комбинация | Вероятность | Примечание |
|---|---|---|
| M + M | 45% | Самая частая |
| K + M | 20% | |
| K + K | 10% | |
| G + M | 8% | |
| G + K | 6% | |
| G + G | 4% | |
| F + M/K/G | 3% | |
| A + M/K | 2% | |
| B + M/K | 1.5% | |
| Прочие | 0.5% | |

**Влияние на стабильность орбит**:

| Тип системы | Множитель стабильности | Пояснение |
|---|---|---|
| Одиночная | 1.0 | Базовая стабильность |
| Широкая двойная | 0.9 | Минимальное влияние |
| Близкая двойная | 0.6 | Значительная нестабильность внутренних орбит |
| Тройная | 0.5 | Существенная нестабильность |

Множитель стабильности влияет на:
- Максимальное количество планет: `max_planets = floor(базовое_max × множитель_стабильности)`
- Вероятность разрушения планеты (при генерации): `P_разрушения = 1 - множитель_стабильности`
- Наклонение орбит: больше хаоса в нестабильных системах

**Широкая двойная — генерация подсистем**:

При генерации широкой двойной системы каждый компонент генерирует собственную планетную систему независимо, как одиночная звезда, но с множителем стабильности 0.9. Расстояние между компонентами:

```
d = 10 + random() * 990   // 10–1000 АЕ, равномерное распределение
```

Игрок видит одну звёздную систему на карте галактики, но при приближении видит два (или три) скопления планет.

## 3. Влияние типа звезды на планетную систему

### 3.1 Количество планет и астероидных полей

| Тип звезды | Мин. планет | Макс. планет | Мин. астероидных полей | Макс. астероидных полей | Вероятность ≥1 обитаемой планеты |
|---|---|---|---|---|---|
| O | 0 | 2 | 1 | 2 | 2% |
| B | 0 | 4 | 1 | 3 | 5% |
| A | 1 | 6 | 1 | 3 | 15% |
| F | 2 | 8 | 0 | 3 | 35% |
| G | 2 | 10 | 0 | 3 | 50% |
| K | 1 | 8 | 0 | 4 | 40% |
| M | 1 | 5 | 0 | 2 | 20% |
| Белый карлик | 0 | 3 | 1 | 3 | 3% |
| Красный гигант | 1 | 5 | 0 | 2 | 10% |
| Нейтронная | 0 | 2 | 1 | 4 | 0% |
| Пульсар | 0 | 1 | 2 | 4 | 0% |
| Чёрная дыра | 0 | 1 | 0 | 2 | 0% |

**Формула количества планет**:

```
n_planets = min_planets + floor(random() * (max_planets - min_planets + 1))
```

Для звёзд главной последовательности существует корреляция массы и количества планет:

```
// Альтернативная формула (для программистов, предпочитающих непрерывные распределения):
n_planets = round(10 × M^0.3 × (0.7 + 0.6 × random()))
// Где M — масса звезды в M☉, результат ограничивается min/max из таблицы
```

### 3.2 Зона обитаемости

Зона обитаемости (Habitable Zone, HZ) — область вокруг звезды, где возможна жидкая вода на поверхности планеты.

**Формула расчёта границ зоны обитаемости**:

```
R_внутр = sqrt(L / 1.1)   // АЕ, внутренний край (эффект парника)
R_внешн = sqrt(L / 0.53)  // АЕ, внешний край (максимальный парник)
```

Где `L` — светимость звезды в L☉.

**Таблица зон обитаемости по типам звёзд**:

| Тип звезды | L (среднее, L☉) | Внутренний край (АЕ) | Внешний край (АЕ) | Ширина HZ (АЕ) | Средний радиус HZ (АЕ) |
|---|---|---|---|---|---|
| O | 200 000 | 426 | 614 | 188 | 520 |
| B | 500 | 21.3 | 30.7 | 9.4 | 26 |
| A | 12 | 3.3 | 4.8 | 1.5 | 4.0 |
| F | 2.5 | 1.51 | 2.17 | 0.66 | 1.84 |
| G | 1.0 | 0.95 | 1.37 | 0.42 | 1.16 |
| K | 0.3 | 0.52 | 0.75 | 0.23 | 0.64 |
| M | 0.02 | 0.13 | 0.19 | 0.06 | 0.16 |

**Для специальных типов**:

| Тип звезды | L (L☉) | Внутренний край (АЕ) | Внешний край (АЕ) | Примечание |
|---|---|---|---|---|
| Белый карлик | 0.005 | 0.067 | 0.097 | Практически непригодна |
| Красный гигант | 3 000 | 52 | 75 | Нестабильна |

**Определение «обитаемости» планеты**:

Планета считается потенциально обитаемой, если её орбитальный радиус попадает в HZ:

```
is_in_habitable_zone = (R_орбиты >= R_внутр) AND (R_орбиты <= R_внешн)
```

Дополнительные условия для **реальной** обитаемости (габитабельность > 0):
- Наличие атмосферы (нет / тонкая / плотная / инертная — подходит; токсичная — нет)
- Гравитация: 0.5–1.5 g (комфортная), 1.5–2.0 g (трудная, снижение эффективности)
- Температура поверхности: −50°C … +100°C (с учётом атмосферы)

### 3.3 Вероятность газовых гигантов vs скалистых планет

Распределение типов планет зависит от расстояния от звезды (по аналогии с линией снега / snow line):

**Линия снега** — расстояние, за которым летучие вещества замерзают:

```
R_snow = 2.7 × sqrt(L)   // АЕ
```

| Положение орбиты | Тип планеты | Вероятность |
|---|---|---|
| Внутри 0.5 × R_snow | Скалистая | 90% |
| Внутри 0.5 × R_snow | Газовый гигант | 10% (горячий юпитер) |
| 0.5–1.0 × R_snow | Скалистая | 70% |
| 0.5–1.0 × R_snow | Газовый гигант | 30% |
| 1.0–2.0 × R_snow | Скалистая | 40% |
| 1.0–2.0 × R_snow | Газовый гигант | 50% |
| 1.0–2.0 × R_snow | Ледяной гигант | 10% |
| За 2.0 × R_snow | Скалистая | 10% |
| За 2.0 × R_snow | Газовый гигант | 40% |
| За 2.0 × R_snow | Ледяной гигант | 50% |

**Примечание**: Ледяные гиганты (Уран, Нептун) в игре классифицируются как отдельный тип планет с собственной таблицей ресурсов (H, He, N, CH₄, NH₃ из атмосферы; лёд из недр).

**Коррекция по типу звезды**:

Массивные звёзды имеют более массивные протопланетные диски → больше газовых гигантов:

| Тип звезды | Множитель вероятности газового гиганта |
|---|---|
| O | ×2.0 |
| B | ×1.7 |
| A | ×1.4 |
| F | ×1.1 |
| G | ×1.0 |
| K | ×0.8 |
| M | ×0.5 |

Пример расчёта:
```
// Планета на орбите 1.5 АЕ вокруг звезды класса A (L = 12 L☉)
R_snow = 2.7 × sqrt(12) = 9.35 АЕ
Позиция = 1.5 / 9.35 = 0.16 × R_snow  → внутри 0.5 × R_snow
Базовая вероятность газового гиганта = 10%
С коррекцией класса A: 10% × 1.4 = 14%
```

### 3.4 Влияние двойных/тройных систем на стабильность

**Правило стабильности**: планета стабильна, если её орбита удовлетворяет критерию:

**Для циркумбинарных (близкая двойная)**:
```
a_min = 2–4 × a_binary   // минимальная стабильная орбита
// a_binary — расстояние между компонентами двойной
```

Планеты ближе `a_min` не формируются — область внутри `a_min` пуста.

**Для S-типа (планета вокруг одного компонента широкой двойной)**:
```
a_max = 0.2–0.5 × a_binary   // максимальная стабильная орбита
```

Планеты дальше `a_max` нестабильны и выбрасываются.

**Практическое следствие для генерации**:

| Тип системы | Ограничение |
|---|---|
| Одиночная | Нет ограничений |
| Широкая двойная | Планеты каждого компонента ограничены радиусом `0.3 × d` (где d — расстояние между компонентами) |
| Близкая двойная | Планеты только дальше `3 × d_binary` от центра масс; внутренняя область пуста |
| Тройная | Для близкой пары — как близкая двойная; для удалённого компонента — как широкая двойная |

## 4. Астероидные поля

### 4.1 Определение в контексте игры

**Астероидное поле** — это не единичные астероиды, а скопление из тысяч и миллионов мелких тел, образующее протяжённую зону в звёздной системе. Аналог — пояс астероидов между Марсом и Юпитером (миллионы тел от 1 км до 1000 км).

В игре астероидное поле — это **игровой объект** с координатами, ресурсным потенциалом и навигационными свойствами. Не путать с отдельными астероидами — единичные тела не моделируются.

### 4.2 Количество полей на систему

Количество астероидных полей зависит от типа звезды (см. таблицу в разделе 3.1).

**Формула генерации**:
```
n_asteroid_fields = min_af + floor(random() * (max_af - min_af + 1))
```

Дополнительно:
- Если в системе есть газовый гигант, вероятность дополнительного астероидного поля +10% (гравитация гиганта формирует пояса)
- Если система двойная/тройная, вероятность дополнительного поля +15% (гравитационные возмущения)

**Позиция астероидного поля в системе**:

```
// Астероидные поля обычно расположены:
// 1) Между внутренними планетами и газовыми гигантами (главный пояс) — 70% полей
// 2) За орбитой последнего газового гиганта (пояс Койпера) — 20% полей
// 3) В точках Лагранжа газовых гигантов (троянцы) — 10% полей

R_поля = R_ближайшей_планеты + random() × (R_следующей_планеты - R_ближайшей_планеты)
// Для пояса Койпера:
R_поля = R_последней_планеты × (1.5 + random() × 2.0)
```

### 4.3 Ресурсный потенциал астероидных полей

Астероидные поля — важнейший источник металлов и редких элементов, не требующий колонизации планеты.

**Категории ресурсов астероидного поля**:

| Категория | Основные элементы | Концентрация (относительно планеты) |
|---|---|---|
| **Металлы** | Fe, Ni, Co, Cr, Mn | ×1.5 (выше, чем на планетах) |
| **Редкие металлы** | Pt, Pd, Au, Ir, Os | ×3.0 (значительно выше) |
| **Лёгкие элементы** | Si, Al, Mg, Ca | ×0.8 (ниже) |
| **Летучие** | H, O, C, N | ×0.1 (очень мало — нет атмосферы) |
| **Уникальные** | Re, Ru, Rh, Hf | ×2.0 (выше) |

**Общая ресурсная ёмкость астероидного поля**:

```
// Ёмкость = количество ресурса, доступного для добычи (в условных единицах)
ёмкость_поля = базовая_ёмкость × множитель_типа_звезды

базовая_ёмкость = 5000 + floor(random() * 15000)   // 5000–20000 единиц

// Множитель типа звезды:
| Тип звезды | Множитель |
|---|---|
| O | 1.5 |
| B | 1.3 |
| A | 1.1 |
| F | 1.0 |
| G | 1.0 |
| K | 0.9 |
| M | 0.7 |
| Белый карлик | 2.0 |
| Нейтронная | 3.0 |
| Пульсар | 5.0 |
| Красный гигант | 1.2 |
| Чёрная дыра | 1.5 |
```

**Распределение элементов внутри ёмкости**:

```
// Для каждого поля генерируется набор доступных элементов
n_доступных_элементов = 5 + floor(random() * 10)   // 5–15 элементов

// Элементы выбираются взвешенным случайным выбором:
// Повсеместные: вес 10
// Распространённые: вес 5
// Редкие: вес 2
// Уникальные: вес 1

// Количество каждого элемента:
количество_i = ёмкость_поля × доля_категории_i × (0.5 + random())
// доля_категории: металлы = 0.4, редкие = 0.15, лёгкие = 0.25, летучие = 0.05, уникальные = 0.05
```

### 4.4 Добыча на астероидах

**Автоматические шахты** — единственный способ добычи на астероидных полях.

| Свойство | Значение |
|---|---|
| **Тип постройки** | Автоматическая шахта (не требует колонистов) |
| **Стоимость** | 50 × уровень стали + 20 × уровень Ti + 10 × уровень микрочипов |
| **Уровни** | 1–5 |
| **Добыча (ед./тик)** | `5 × уровень × множитель_элемента` |
| **Энергопотребление** | `10 × уровень` ед./тик |
| **Не требуют** | Колонистов, атмосферы, гравитации |
| **Макс. на поле** | 10 шахт |
| **Ограничение** | Требуют корабль-строитель для установки |

**Преимущества автоматических шахт на астероидах**:
1. Не нужна колония — не требуются колонисты
2. Не нужна атмосфера — астероиды безатмосферные
3. Высокая концентрация редких металлов
4. Быстрая установка (5 тиков на шахту vs 20 тиков на планете)

**Недостатки**:
1. Нет защиты — шахты уязвимы для нападения
2. Ограниченный ресурс — поле истощается со временем
3. Требуется транспорт для вывоза ресурсов
4. Нужна энергия (солнечные панели или ядерный генератор)

**Истощение поля**:

```
// Каждая шахта уменьшает ёмкость поля
ёмкость_поля -= добыча_за_тик

// При ёмкости < 1000:
//   - Добыча снижается пропорционально: множитель = ёмкость / 1000
//   - При ёмкости = 0 — поле считается истощённым, шахты останавливаются

// Скорость восполнения: 0 (астероиды не восстанавливаются)
// Исключение: поле в системе с газовым гигантом получает +1 ед./1000 тиков
//   (гравитация разрушает мелкие тела, пополняя пояс)
```

### 4.5 Опасность навигации в астероидных полях

| Эффект | Значение | Условие |
|---|---|---|
| **Замедление скорости** | −30% к скорости корабля | Корабль внутри астероидного поля |
| **Урон от столкновений** | 0.5 HP/тик | Корабль без щитов внутри поля |
| **Урон (со щитами)** | 0 (щиты поглощают) | Корабль со щитами внутри поля |
| **Вероятность аварии** | 0.1% за тик | Корабль с неисправным двигателем |
| **Урон аварии** | 10% от макс. HP | При аварии |

**Модуль «Астероидный обходчик»** (улучшение корабля):
- Снижает замедление: −30% → −10%
- Снижает вероятность аварии: 0.1% → 0.01%
- Требует технологию «Инженерия 4»

## 5. Кометные облака

### 5.1 Облако Оорта

**Описание**: Сферическое облако ледяных тел на периферии звёздной системы (2 000–100 000 АЕ от звезды). В Солнечной системе оценивается в 10¹² кометных ядер.

**В игре**: упрощённая модель — облако Оорта присутствует в большинстве систем как источник льда и летучих элементов.

| Свойство | Значение |
|---|---|
| **Наличие** | 80% одиночных систем, 50% двойных, 30% тройных |
| **Радиус внутреннего края** | 1 000 АЕ |
| **Радиус внешнего края** | 50 000 АЕ |
| **Игровое отображение** | Не отображается на обычной карте; доступно через экран системы |

### 5.2 Ресурсы кометных облаков

| Категория | Основные элементы | Концентрация |
|---|---|---|
| **Лёд** | H₂O (H, O) | Очень высокая |
| **Летучие элементы** | N, C, CO₂, CH₄, NH₃ | Высокая |
| **Органика** | C, H, O, N (аминокислоты, простые органические молекулы) | Умеренная |
| **Металлы** | Fe, Ni, Si (в пылевой компоненте) | Низкая |

**Добыча из кометного облака**:

В отличие от астероидных полей, добыча из облака Оорта требует специальных кораблей-сборщиков:

| Свойство | Значение |
|---|---|
| **Тип добычи** | Корабль-сборщик комет (специальный модуль) |
| **Добыча (ед./тик)** | 2 × уровень_модуля |
| **Доступные элементы** | H, O, C, N — в пропорции 4:3:2:1 |
| **Органика** | 5% шанс найти органику за каждую единицу добычи |
| **Время полёта к облаку** | 100 тиков (один путь) |
| **Рейс туда-обратно** | 200 тиков + время загрузки |

**Стратегическое значение**:
- Лёд (H₂O) — источник воды для пустынных и вулканических колоний
- Летучие (N, C) — критически важны для биологии и сельского хозяйства
- Органика — редкий ресурс для биотехнологий

### 5.3 Периодические кометы

**Описание**: Из облака Оорта периодически «выпадают» кометы — ледяные тела на сильно вытянутых орбитах, проходящие через внутреннюю систему.

**Генерация периодических комет**:

```
// Каждая система с облаком Оорта генерирует 0–5 периодических комет
n_comets = floor(random() * 6)

// Параметры каждой кометы:
R_перигелий = 0.1 + random() × 2.0   // АЕ (ближайшая к звезде точка)
R_афелий = 500 + random() × 49500     // АЕ (самая дальняя точка)
период = sqrt(R_афелий³)              // годы (по третьему закону Кеплера)
// В игровых тиках: период_тиков = период × 365 × 24 × 60 / длительность_тика
```

**Периодические кометы как событие**:

Когда комета проходит через внутреннюю систему (раз в свой период):
1. **Научное событие**: +100 очков исследований биологии (изучение кометы)
2. **Ресурсное событие**: можно отправить корабль-сборщик к комете → добыча ×10 за рейс
3. **Атмосферное событие**: если комета проходит близко к обитаемой планете, +5% воды в атмосферу

**Для баланса**: средний период комет измеряется тысячами лет, поэтому в игровой кампании большинство комет не успеют вернуться. Но короткопериодические кометы (период < 20 лет, R_афелий < 35 АЕ — пояс Койпера) встречаются чаще.

```
// Вероятность короткопериодической кометы: 30% от всех комет
if (random() < 0.3) {
    R_афелий = 10 + random() × 25   // 10–35 АЕ
    период = sqrt(R_афелий³)         // 30–200 лет
}
```

## 6. Алгоритм генерации звёздной системы

### 6.1 Обзор алгоритма

Генерация звёздной системы — пошаговый детерминированный процесс на основе seed. Одинаковый seed + одинаковые параметры → одинаковая система.

**Входные данные**:
- `seed` — целое число (из генератора галактики)
- `galaxy_age` — возраст галактики (млрд лет, влияет на вероятности)
- `force_type` — опционально: принудительный тип звезды (для стартовой системы игрока)

**Выходные данные**:
- Звезда (или звёзды) с параметрами
- Массив планет с параметрами
- Массив астероидных полей с ресурсами
- Кометное облако (если есть)
- Периодические кометы (если есть облако)

#### Шаг 7: Генерация орбит планет

```
// Распределение орбит по закону Титиуса-Боде (модифицированное)
// R_n = R_0 × k^n, где k ≈ 1.5–2.0

k = 1.5 + rng.nextFloat() * 0.5   // 1.5–2.0, вариация для каждой системы

    R_орбиты[i] = R_0 × k^i
    
    // Добавить случайное отклонение ±15%

    // Ограничение стабильности для двойных:
```

#### Шаг 8: Тип каждой планеты

```
R_snow = 2.7 × sqrt(primary_luminosity)   // линия снега

for (i = 0; i < n_planets; i++) {
    R = R_орбиты[i]
    ratio = R / R_snow
    
    // Определяем вероятность газового гиганта (раздел 3.3)
    if (ratio < 0.5)       → p_gas = 0.10
    else if (ratio < 1.0)  → p_gas = 0.30
    else if (ratio < 2.0)  → p_gas = 0.50
    else                   → p_gas = 0.40
    
    // Коррекция по типу звезды
    p_gas ×= gas_giant_multiplier[primary_type]
    p_gas = min(p_gas, 0.90)   // не более 90%
    
    // Определяем тип
    roll = rng.nextFloat()
    if (roll < p_gas) {
        if (ratio < 2.0 && rng.nextFloat() < 0.1) {
            planet_type[i] = ICE_GIANT    // ледяной гигант (10% шанс за линией снега)
        } else {
            planet_type[i] = GAS_GIANT
        }
    } else {
        // Скалистая — конкретный подтип зависит от температуры
        T_равновесная = 278 × (primary_luminosity ^ 0.25) / sqrt(R)   // Кельвин
        
        if (T_равновесная > 500)        → planet_type[i] = VOLCANIC
        else if (T_равновесная > 350)   → planet_type[i] = DESERT
        else if (T_равновесная > 250)   → planet_type[i] = ROCKY   // потенциально обитаемая
        else if (T_равновесная > 150)   → planet_type[i] = OCEANIC // если есть вода
        else if (T_равновесная > 50)    → planet_type[i] = ICE
        else                            → planet_type[i] = ICE
    }
    
    // Проверка обитаемости (раздел 3.2)
    is_in_hz = (R >= R_hz_inner) && (R <= R_hz_outer)
    
    // Форсирование обитаемой планеты для стартовой системы
    if (is_start_system && i == форсированный_индекс) {
        planet_type[i] = ROCKY
        R_орбиты[i] = (R_hz_inner + R_hz_outer) / 2
        is_in_hz = true
    }
}

```

```
    // Радиус (в R⊕)
    if (planet_type[i] == GAS_GIANT) {
        radius[i] = 4 + rng.nextFloat() * 8    // 4–12 R⊕ (Юпитер ≈ 11 R⊕)
    } else if (planet_type[i] == ICE_GIANT) {
        radius[i] = 3 + rng.nextFloat() * 2    // 3–5 R⊕ (Нептун ≈ 3.9 R⊕)
    } else {
        radius[i] = 0.5 + rng.nextFloat() * 2.5 // 0.5–3.0 R⊕
    }
    
    // Масса (в M⊕)
    if (planet_type[i] == GAS_GIANT) {
        mass[i] = 50 + rng.nextFloat() * 450    // 50–500 M⊕ (Юпитер ≈ 318 M⊕)
    } else if (planet_type[i] == ICE_GIANT) {
        mass[i] = 10 + rng.nextFloat() * 30     // 10–40 M⊕ (Нептун ≈ 17 M⊕)
    } else {
        // Скалистая: масса ≈ радиус³ (плотность ≈ земной)
        mass[i] = radius[i] ^ 3
    }
    
    // Гравитация
    gravity[i] = mass[i] / (radius[i] ^ 2)   // в g (земных)
    
    // Атмосфера
    if (planet_type[i] == GAS_GIANT || planet_type[i] == ICE_GIANT) {
        atmosphere[i] = DENSE
    } else if (gravity[i] < 0.3) {
        atmosphere[i] = NONE      // слишком слабая гравитация
    } else if (is_in_hz[i] && gravity[i] >= 0.5) {
        roll = rng.nextFloat()
        if (roll < 0.3)      → atmosphere[i] = NONE
        else if (roll < 0.5) → atmosphere[i] = THIN
        else if (roll < 0.8) → atmosphere[i] = DENSE
        else if (roll < 0.95) → atmosphere[i] = TOXIC
        else                 → atmosphere[i] = INERT
    } else {
        roll = rng.nextFloat()
        if (roll < 0.5)      → atmosphere[i] = NONE
        else if (roll < 0.7) → atmosphere[i] = THIN
        else if (roll < 0.85) → atmosphere[i] = TOXIC
        else                 → atmosphere[i] = INERT
    }
    
    // Габитабельность (0–100%)
    habitability[i] = calculate_habitability(gravity[i], atmosphere[i], T_равновесная[i], planet_type[i])
    // (формула в отдельном документе о планетах)
}
```

### 6.3 Формулы-справочники

#### Закон Стефана-Больцмана (температура звезды)

```
T = 5778 × (L / R²)^0.25   [K]
```

#### Равновесная температура планеты

```
T_eq = 278 × L^0.25 / sqrt(R_орбиты)   [K]
// L — светимость в L☉, R_орбиты — в АЕ
// Без учёта парникового эффекта
```

#### Зона обитаемости

```
HZ_inner = sqrt(L / 1.1)   [АЕ]
HZ_outer = sqrt(L / 0.53)  [АЕ]
```

#### Линия снега

```
R_snow = 2.7 × sqrt(L)   [АЕ]
```

#### Третий закон Кеплера (период обращения)

```
P² = a³ / M   [P в годах, a в АЕ, M в M☉]
P = sqrt(a³ / M)
```

#### Масса-светимость (главная последовательность)

```
L = M^3.5    для M < 2 M☉
L = M^2.3    для M ≥ 2 M☉
```

#### Масса-радиус (главная последовательность)

```
R = M^0.8    для M < 1 M☉
R = M^0.57   для M ≥ 1 M☉
```

#### Время жизни звезды

```
τ ≈ 10 × (M / L)   [млрд лет]
// или с подстановкой: τ ≈ 10 × M^(-2.5) для M < 2 M☉
// или: τ ≈ 10 × M^(-1.3) для M ≥ 2 M☉
```

## Приложение B: Балансировочные заметки

### Проблема: красные карлики слишком скучные

76% звёзд — M-класс. Если у них мало планет и узкая HZ, игрок будет пропускать большинство систем.

**Решение**:
1. У M-карликов высокие шансы на астероидные поля с редкими элементами → даже необитаемые системы M-класса стоят посещения ради добычи
2. «Приливно-захваченные» планеты M-карликов имеют уникальную механику: терминаторная зона (граница день/ночь) — обитаемая полоса шириной 10–50 км с габитабельностью до 30%
3. Вспышки M-карликов — случайное событие: раз в 100–500 тиков, снижает габитабельность на −10% на 10 тиков

### Проблема: чёрные дыры бесполезны

Чёрные дыры опасны и не имеют обитаемых планет — зачем туда лететь?

**Решение**:
1. Уникальное исследование: +1 технология физики (одноразово) — мотивация для научных кораблей
2. Аккреционный диск: визуальный эффект + потенциальный источник антиматерии (на поздних этапах игры)
3. Гравитационная аномалия: корабли с варп-двигателем могут использовать чёрную дыру как «гравитационную рогатку» — +50% к дальности варп-прыжка при старте из системы с чёрной дырой

### Проблема: двойные системы сложны в отображении

**Решение**: на карте галактики — одна иконка. При зуме в систему — две (или три) подсистемы, каждая с планетами. UI переключается между подсистемами вкладками.
