## 1. Архитектура AI

### 1.1 Фундаментальные принципы

Каждый AI-противник представляет собой полноценную «империю» — самостоятельную сущность с собственными:

- **Ресурсами** — запасы элементов, материалов, компонентов, энергии, кредитов
- **Территорией** — звёздные системы, планеты, постройки
- **Флотом** — корабли, сгруппированные во флоты с приказами
- **Технологиями** — исследованные технологии и текущие проекты
- **Целями** — стратегические и тактические задачи, приоритеты
- **Отношениями** — числовые значения отношений с каждой другой фракцией (включая игрока)

**Ключевое правило**: AI работает на тех же правилах, что и игрок. У AI нет доступа к скрытой информации, нет бесплатных ресурсов, нет мгновенного строительства. Единственное исключение — стартовые бонусы на высоких уровнях сложности (см. раздел 7).

### 1.3 Подходы к принятию решений

Для реализации AI рассматриваются два архитектурных подхода. Ниже — сравнение и рекомендация.

#### 1.3.1 GOAP (Goal-Oriented Action Planning)

**Суть**: AI задаёт цель → система планирования ищет цепочку действий для достижения цели → AI выполняет план.

```
Цель: «Захватить систему Alpha-7»
  │
  ├─ Нужен флот силой > 500
  │   ├─ Построить крейсер (нужны: верфь L3 + компоненты)
  │   │   ├─ Нужна верфь L3 → построить/улучшить верфь
  │   │   └─ Нужны компоненты → произвести на заводе
  │   └─ Собрать флот из существующих кораблей
  │
  ├─ Нужна разведка системы
  │   └─ Отправить разведчик
  │
  └─ Нужен путь к системе
      └─ Проверить гипермаршруты
```

**Преимущества**:
- Гибкие многошаговые планы
- AI может реагировать на изменения, пересчитывая план
- Естественная декомпозиция сложных целей
- Хорошо сочетается с Aurora 4X-стилем игры (долгосрочное планирование)

**Недостатки**:
- Вычислительно дорого при большом пространстве действий
- Сложно отлаживать (план может быть неожиданным)
- Требует чёткого определения предусловий и эффектов для каждого действия

**Формальная модель действия GOAP**:

```typescript
// Псевдокод: структура действия в GOAP
interface GOAPAction {
  id: string;                    // Уникальный идентификатор
  preconditions: WorldState;     // Необходимые условия
  effects: WorldState;           // Результат выполнения
  cost: number;                  // Стоимость (для A* поиска)
  execute: () => ActionResult;   // Исполнение
}

// Пример: действие «Построить шахту»
const buildMine: GOAPAction = {
  id: "build_mine",
  preconditions: {
    has_energy: true,
    credits_available: { ">=": 100 },
    planet_slots_available: { ">=": 1 },
  },
  effects: {
    mine_count: { "+": 1 },
    credits_available: { "-": 100 },
    planet_slots_available: { "-": 1 },
  },
  cost: 100, // базируется на стоимости в кредитах
};
```

#### 1.3.2 Utility AI (Система полезностей)

**Суть**: Для каждого возможного действия AI вычисляет «полезность» (utility) — числовую оценку от 0 до 1, учитывающую текущее состояние мира. Выбирается действие с максимальной полезностью.

```
Оценка действий для Конфедерации (тик 1500):

Действие                    | Потребность | Срочность | Осуществимость | Utility
────────────────────────────|─────────────|───────────|────────────────|────────
Построить шахту Fe          | 0.8         | 0.6       | 0.9            | 0.432
Построить крейсер           | 0.5         | 0.4       | 0.3            | 0.060
Улучшить лабораторию        | 0.6         | 0.3       | 0.7            | 0.126
Предложить торговлю Синдикату| 0.7        | 0.5       | 0.8            | 0.280
Колонизировать систему B-4  | 0.4         | 0.2       | 0.6            | 0.048
════════════════════════════╪═════════════╪═══════════╪════════════════╪════════
→ Выбрано: Построить шахту Fe (макс. utility = 0.432)
```

**Преимущества**:
- Простота реализации и отладки
- Вычислительно эффективно
- Легко балансировать (подкручивать веса)
- Естественная приоритизация
- Хорошо работает для Economics AI и Diplomacy AI

**Недостатки**:
- Нет долгосрочного планирования (только «лучшее действие прямо сейчас»)
- Может «метаться» между действиями без последовательной стратегии
- Требует тщательной настройки кривых полезности

**Формула принятия решения**:

```
utility(action) = need(action) × urgency(action) × feasibility(action)

Где:
  need       = насколько фракция нуждается в результате этого действия [0..1]
  urgency    = насколько быстро нужно действовать [0..1]
  feasibility = насколько действие выполнимо при текущих ресурсах [0..1]
```

#### 1.3.3 Рекомендация: Гибридная архитектура

**Рекомендуемый подход**: **Utility AI как основа + GOAP для стратегического планирования**.

| Компонент AI | Подход | Обоснование |
|---|---|---|
| Экономика (что строить, что производить) | **Utility AI** | Каждый тик нужно выбирать из десятков вариантов — Utility эффективнее |
| Военный AI (атаковать/защищаться/отступать) | **Utility AI** | Быстрая оценка тактической ситуации |
| Дипломатия (предложить/отклонить) | **Utility AI** | Числовые оценки отношений хорошо ложатся на utility |
| Стратегические цели (долгосрочные) | **GOAP** | Планирование цепочек: «колонизировать → построить → исследовать» |
| Экспансия (куда колонизировать) | **Utility AI** с весами фракции | Оценка кандидатов-систем по формуле |

**Архитектура гибрида**:

```
Каждые N тиков (стратегический цикл):
  1. GOAP-планировщик обновляет/создаёт стратегический план
  2. План формирует набор активных целей (Goal Queue)

Каждый тик (тактический цикл):
  1. Utility AI оценивает доступные действия
  2. Действия, соответствующие активным целям GOAP, получают бонус к utility
  3. Выбирается действие с максимальной utility
  4. Действие исполняется через Action Executor
```

**Бонус GOAP-целей к utility**:

```
utility_with_plan = base_utility × (1 + plan_alignment_bonus)

plan_alignment_bonus = 0.5  если действие прямо способствует активной GOAP-цели
plan_alignment_bonus = 0.0  если действие не связано с планом
```

Это гарантирует, что AI действует целенаправленно (GOAP), но при этом адаптивно реагирует на текущую ситуацию (Utility AI).

### 1.4 Цикл обновления AI

| Тип решения | Частота | Примеры |
|---|---|---|
| Стратегический цикл | Каждые 100 тиков | Пересмотр целей, планирование экспансии, пересмотр дипломатии |
| Тактический цикл | Каждый 10 тиков | Строительство, производство, перемещение флотов, торговля |
| Реактивный цикл | Каждый тик | Реакция на атаки, нарушения границ, срочные угрозы |
| Дипломатический цикл | Каждый 50 тиков | Оценка отношений, генерация предложений, ответы на предложения |

## 2. Типы AI-фракций

### 2.1 Обзор фракций

В игре присутствуют 5 AI-фракций. Не все фракции обязательно присутствуют в каждой игре — количество и состав настраивается при старте (по умолчанию: 3 из 5).

| Фракция | Архетип | Бонус | Штраф | Стиль |
|---|---|---|---|---|
| **Конфедерация** | Сбалансированная | +10% производство | Нет | Равномерное развитие, дипломатия |
| **Хегемония** | Военная | +15% боевая мощь | −20% скорость исследований | Агрессия, экспансия силой |
| **Синдикат** | Торговая | +20% торговые доходы | −10% боевая мощь | Торговля, наёмники, мир |
| **Орден** | Научная | +25% скорость исследований | −15% боевая мощь | Технологии, артефакты, знания |
| **Рой** | Ксено-агрессивная | +30% скорость колонизации | Нет дипломатии, −10% эффективность шахт | Быстрая экспансия, биотех, война |

### 2.2 Конфедерация (сбалансированная)

> *«Единство — сила. Раздельность — слабость.»*

**Концепция**: Демократический союз колоний, стремящийся к балансу во всех сферах. Не проявляет агрессии первым, но даёт отпор. Предпочитает дипломатические решения.

#### Стартовые параметры

| Параметр | Значение |
|---|---|
| Стартовые системы | 3 |
| Стартовый флот | 2 истребителя, 1 транспорт, 1 разведчик |
| Стартовые ресурсы | Стандартные × 1.0 |
| Бонус производства | +10% ко всем видам производства |
| Штраф | Нет |

#### Приоритеты строительства

| Приоритет | Здание | Вес (доля от бюджета строительства) |
|---|---|---|
| 1 | Электростанция | 20% |
| 2 | Шахта | 25% |
| 3 | Завод | 20% |
| 4 | Лаборатория | 15% |
| 5 | Верфь | 10% |
| 6 | Оборона | 5% |
| 7 | Торговый хаб | 5% |

#### Поведение

- **Экономика**: Равномерное развитие всех секторов. При дефиците —优先 строительство недостающего.
- **Военное дело**: Оборонительная доктрина. Строит флот до уровня паритета с сильнейшим соседом. Не атакует первым, если отношения > −20.
- **Дипломатия**: Активно предлагает торговые соглашения и обмен технологиями. Ищет союзников против общих угроз.
- **Экспансия**: Умеренная. Колонизирует системы с габитабельностью > 40%. Предпочитает компактную территорию.

#### Бюджетирование ресурсов

```
budget_allocation = {
  economy:  0.40,  // шахты, заводы, электростанции
  military: 0.25,  // верфи, флот, оборона
  science:  0.20,  // лаборатории, исследования
  trade:    0.10,  // торговые хабы, караваны
  reserve:  0.05,  // запас на непредвиденные расходы
}
```

#### Стратегические цели GOAP (в порядке приоритета)

1. Обеспечить экономическую стабильность (доход > расход по всем ключевым ресурсам)
2. Поддерживать обороноспособность (флот ≥ паритет с соседями)
3. Исследовать технологии уровня текущего + 1
4. Колонизировать пригодные системы в пределах 2 прыжков от границ
5. Заключить торговые соглашения с 2+ фракциями

### 2.3 Хегемония (военная)

> *«Мир — это промежуток между войнами.»*

**Концепция**: Милитаристская империя, верящая в силу как единственный аргумент. Быстро расширяется, подавляет слабых, игнорирует дипломатию.

#### Стартовые параметры

| Параметр | Значение |
|---|---|
| Стартовые системы | 3 |
| Стартовый флот | 4 истребителя, 2 крейсера, 1 транспорт |
| Стартовые ресурсы | Стандартные × 0.9 (меньше ресурсов, больше кораблей) |
| Бонус боевой мощи | +15% к урону и HP всех боевых кораблей |
| Штраф | −20% скорость исследований |

#### Приоритеты строительства

| Приоритет | Здание | Вес (доля от бюджета строительства) |
|---|---|---|
| 1 | Верфь | 30% |
| 2 | Электростанция | 20% |
| 3 | Шахта | 20% |
| 4 | Завод | 15% |
| 5 | Оборона | 10% |
| 6 | Лаборатория | 3% |
| 7 | Торговый хаб | 2% |

#### Поведение

- **Экономика**: Экономика обслуживает военные нужды. Строит шахты и заводы только для обеспечения верфей.
- **Военное дело**: Агрессивная доктрина. Атакует при превосходстве 1.3:1. Приоритет — быстрые набеги на слабозащищённые системы. Не отступает, пока потери < 50% флота.
- **Дипломатия**: Минимальная. Предлагает ультиматумы. Принимает мир только с позиции силы. Не заключает союзов (кроме случаев крайней угрозы).
- **Экспансия**: Агрессивная. Захватывает системы силой, если колонизация невозможна. Не обращает внимания на габитабельность — строит на любой планете.

#### Бюджетирование ресурсов

```
budget_allocation = {
  economy:  0.25,  // минимум для обеспечения верфей
  military: 0.50,  // верфи, флот, оборона — ПРИОРИТЕТ
  science:  0.05,  // минимальные исследования
  trade:    0.05,  // почти не торгует
  reserve:  0.15,  // запас для военных кампаний
}
```

#### Пороги военных решений

| Решение | Условие |
|---|---|
| Атака | `моя_сила > цель_сила × 1.2` (сниженный порог — Хегемония агрессивна) |
| Набег | `моя_сила > цель_сила × 0.8` (готова рисковать) |
| Отступление | `моя_сила < враг_сила × 0.4` (отступает только при подавляющем превосходстве врага) |
| Ультиматум | Отношения < −10 И `моя_сила > их_сила × 1.5` |

#### Стратегические цели GOAP

1. Построить флот, превосходящий каждого соседа минимум на 30%
2. Захватить ближайшую слабую систему
3. Подавить экономику врага набегами (уничтожить шахты, верфи)
4. Контролировать 5+ систем к середине игры
5. Уничтожить столицу weakest-соседа

### 2.4 Синдикат (торговая)

> *«Война — это просто плохо организованная торговля.»*

**Концепция**: Торговая сеть корпораций, максимизирующая прибыль. Избегает прямых конфликтов, предпочитает наёмников и экономическое давление.

#### Стартовые параметры

| Параметр | Значение |
|---|---|
| Стартовые системы | 2 (меньше территорий, больше капитала) |
| Стартовый флот | 1 истребитель, 3 транспорта, 2 разведчика |
| Стартовые ресурсы | Стандартные × 1.3 (больше начального капитала) |
| Бонус торговли | +20% к доходам от торговых соглашений |
| Штраф | −10% боевая мощь кораблей |

#### Приоритеты строительства

| Приоритет | Здание | Вес (доля от бюджета строительства) |
|---|---|---|
| 1 | Торговый хаб | 25% |
| 2 | Электростанция | 15% |
| 3 | Шахта | 15% |
| 4 | Завод | 20% |
| 5 | Верфь | 10% (строит транспорты, а не боевые корабли) |
| 6 | Лаборатория | 10% |
| 7 | Оборона | 5% |

#### Поведение

- **Экономика**: Специализация на дефицитных ресурсах. Продаёт избыток по завышенным ценам. Покупает дефицит по рыночным.
- **Военное дело**: Минимальный собственный флот. Нанимает наёмников (мощные временные флоты за кредиты). Атакует только торговые караваны врагов и защищает свои маршруты.
- **Дипломатия**: Очень активная. Предлагает торговые соглашения всем. Использует экономическое давление (эмбарго). Легко покупает лояльность.
- **Экспансия**: Выборочная. Колонизирует только системы с редкими ресурсами или на перекрёстках торговых путей.

#### Бюджетирование ресурсов

```
budget_allocation = {
  economy:  0.35,  // шахты, заводы — основа экспорта
  military: 0.10,  // минимальный собственный флот
  science:  0.15,  // торговые технологии
  trade:    0.30,  // хабы, караваны, наёмники — ПРИОРИТЕТ
  reserve:  0.10,  // на наёмников и экстренные закупки
}
```

#### Торговый AI Синдиката

```
price_sell  = base_price × (1 + demand_factor × 0.5) × faction_bonus
price_buy   = base_price × (1 − supply_factor × 0.3) × faction_bonus

Где:
  demand_factor = дефицит данного ресурса на рынке [0..1]
  supply_factor = избыток данного ресурса на рынке [0..1]
  faction_bonus = 1.20 (бонус фракции к торговле)

Синдикат продаёт по ценам выше рынка, а покупает — ниже.
Разница = торговая прибыль.
```

#### Наёмники

Синдикат может нанимать наёмные флоты:
- Стоимость: 500 кредитов за крейсер-эквивалент за 100 тиков
- Максимум: 3 наёмных флота одновременно
- Наёмники появляются из нейтральных систем
- При нехватке средств наёмники дезертируют

1. Установить торговые соглашения со всеми фракциями
2. Контролировать 3+ торговых хаба в ключевых системах
3. Обеспечить монополию на 2+ редких ресурсах
4. Построить флот наёмников для защиты торговых маршрутов
5. Достичь экономической победы (60% торговых потоков)

## 3. Экономический AI

### 3.1 Определение потребностей

Экономический AI определяет, каких ресурсов не хватает для текущих целей, и предпринимает действия для устранения дефицита.

#### Алгоритм оценки потребностей

```
Для каждого ресурса R:
  need_score(R) = deficit(R) × weight(R)

Где:
  deficit(R) = max(0, required(R) − available(R))

  required(R) = Σ расход_R_на_текущие_задачи + Σ расход_R_на_планы_GOAP + buffer(R)

  buffer(R) = base_buffer × (1 + urgency_factor)
  // buffer — минимальный запас на непредвиденные нужды
  // urgency_factor растёт при войне или угрозе

  weight(R) — вес ресурса, зависит от типа фракции:
    Конфедерация: все веса ≈ 1.0
    Хегемония:    вес(компоненты_кораблей) = 2.0, вес(научные_ресурсы) = 0.5
    Синдикат:     вес(редкие_ресурсы) = 2.0, вес(военные) = 0.5
    Орден:        вес(научные_ресурсы) = 2.5, вес(военные) = 0.5
    Рой:          вес(базовые_ресурсы) = 1.5, вес(редкие) = 0.3
```

#### Матрица потребностей по фракциям

| Ресурс | Конфедерация | Хегемония | Синдикат | Орден | Рой |
|---|---|---|---|---|---|
| Базовые (Fe, C, Si) | 1.0 | 1.5 | 0.8 | 0.8 | 1.5 |
| Распространённые (Ti, Cu, Ni) | 1.0 | 1.5 | 0.8 | 1.0 | 1.2 |
| Редкие (W, Pt, U) | 1.0 | 1.2 | 2.0 | 2.0 | 0.3 |
| Уникальные (Ir, Os) | 1.0 | 0.8 | 2.5 | 2.5 | 0.2 |
| Энергия | 1.0 | 1.2 | 1.0 | 1.2 | 1.0 |
| Кредиты | 1.0 | 0.8 | 2.0 | 0.8 | 0.5 |

### 3.2 Приоритеты строительства

AI строит здания в порядке приоритетности. Базовый алгоритм:

```
function selectNextBuilding(planet, faction):
  candidates = []

  for building_type in BUILDING_TYPES:
    // Пропускаем если слоты заняты
    if planet.slots_used >= planet.slots_total:
      break

    // Пропускаем если уже достаточно зданий этого типа
    if countBuildings(planet, building_type) >= desiredCount(planet, building_type):
      continue

    // Вычисляем utility
    u = utility(building_type, planet, faction)
    candidates.push({ type: building_type, utility: u })

  // Сортируем по utility и берём лучшее
  candidates.sort(by: utility, descending)
  return candidates[0]?.type ?? null
```

#### Формула utility для зданий

```
utility(building) = need(building) × urgency(building) × feasibility(building)

need(building):
  mine:       deficit(target_resource) / stockpile(target_resource + 1)
  factory:    deficit(components) / (components_produced + 1)
  shipyard:   (desired_fleet_size − current_fleet_size) / (desired_fleet_size + 1)
  power:      (energy_consumption − energy_production) / (energy_consumption + 1)
  lab:        research_deficit / (current_research_speed + 1)
  defense:    threat_level / 100
  trade_hub:  trade_opportunity_count / 10
  scanner:    unexplored_nearby_systems / 10

urgency(building):
  // Время до критического дефицита
  ticks_until_crisis = stockpile(target_resource) / consumption_rate(target_resource)
  urgency = 1.0 / (1.0 + ticks_until_crisis / 100)
  // Чем ближе кризис — тем выше срочность

feasibility(building):
  // Хватает ли ресурсов для строительства
  feasibility = 1.0 если все требования выполнены
  feasibility = 0.5 если не хватает ≤ 20% ресурсов
  feasibility = 0.0 если не хватает > 20% ресурсов
```

#### Желаемое количество зданий на планете

```
desiredCount(planet, building_type):

  mine:
    base = количество_залежей_на_планете  // не больше залежей
    adjusted = base × faction.mine_weight
    min(adjusted, planet.slots_total × 0.4)  // не более 40% слотов

  factory:
    base = countBuildings(planet, "mine") × 0.8  // ≈ 1 завод на 1.25 шахт
    min(base, planet.slots_total × 0.25)

  shipyard:
    base = 1 если planet.has_shipyard_resources else 0
    + 1 если planet.is_border_system  // верфи на границе
    min(base, 2)  // максимум 2 верфи на планету

  power:
    // Строим столько, сколько нужно для питания всех зданий + 10% запас
    total_energy_need = Σ energy_consumption(building) for all buildings
    total_energy_prod = Σ energy_production(power_plant) for all power plants
    needed = ceil((total_energy_need × 1.1 − total_energy_prod) / energy_per_power_plant)
    max(0, needed)

  lab:
    base = 1  // минимум 1 лаборатория на планете с населением
    + 1 если planet.habitability > 70  // хорошие планеты — научные центры
    min(base, planet.slots_total × 0.2)

  defense:
    base = 1 если planet.is_colony_capital else 0
    + threat_level / 50  // больше обороны при высоких угрозах
    min(base, 5)

  trade_hub:
    base = 1 если planet.is_trade_crossroad else 0
    + countNeighbouringFactions(planet) × 0.5  // больше хабов у границ
    min(base, 2)

  scanner:
    base = 1  // всегда 1 сканер
    min(base, 1)
```

### 3.3 Торговля

AI генерирует торговые предложения на основе дефицита и избытка ресурсов.

#### Генерация торговых предложений

```
function generateTradeProposals(faction):
  proposals = []

  for resource in ALL_RESOURCES:
    surplus = available(resource) − required(resource)

    if surplus > trade_threshold(resource):
      // Предлагаем продать избыток
      price = calculateSellPrice(resource, faction)
      proposals.push({
        type: "sell",
        resource: resource,
        quantity: surplus × 0.5,  // продаём до 50% избытка
        price: price,
      })

    elif deficit(resource) > 0:
      // Предлагаем купить дефицит
      price = calculateBuyPrice(resource, faction)
      proposals.push({
        type: "buy",
        resource: resource,
        quantity: deficit(resource) × 0.8,  // хотим покрыть 80% дефицита
        price: price,
      })

  return proposals
```

#### Формирование цен

```
base_price(resource) = rarity_factor(resource) × base_unit_value

rarity_factor:
  Повсеместные:  1.0
  Распространённые: 3.0
  Редкие:       10.0
  Уникальные:   50.0

calculateSellPrice(resource, faction):
  market_demand = global_demand(resource) / global_supply(resource)
  price = base_price(resource) × market_demand × faction.trade_modifier
  // Синдикат: faction.trade_modifier = 1.2 (продаёт дороже)
  // Остальные: faction.trade_modifier = 1.0
  return price

calculateBuyPrice(resource, faction):
  market_supply = global_supply(resource) / global_demand(resource)
  price = base_price(resource) × market_supply × faction.trade_modifier
  // Синдикат: faction.trade_modifier = 0.8 (покупает дешевле)
  return price
```

#### Принятие/отклонение торговых предложений

Когда AI получает предложение от другой фракции (или игрока):

```
function evaluateTradeProposal(proposal, faction):
  if faction == Рой:
    return REJECT  // Рой не торгует

  utility = 0

  if proposal.type == "sell_to_us":
    // Нам продают ресурс
    need_score = deficit(proposal.resource) / (stockpile(proposal.resource) + 1)
    price_fairness = base_price(proposal.resource) / proposal.price
    // price_fairness > 1 = цена ниже базовой = выгодно
    utility = need_score × price_fairness

  elif proposal.type == "buy_from_us":
    // У нас покупают ресурс
    surplus_score = surplus(proposal.resource) / (stockpile(proposal.resource) + 1)
    price_fairness = proposal.price / base_price(proposal.resource)
    // price_fairness > 1 = цена выше базовой = выгодно
    utility = surplus_score × price_fairness

  // Дипломатический модификатор
  utility ×= (1 + relations(proposal.from_faction) / 200)
  // Хорошие отношения → более склонны к торговле

  if utility > 0.6:  return ACCEPT
  if utility > 0.3:  return COUNTER_OFFER
  return REJECT
```

### 3.4 Бюджетирование

Распределение ресурсов между экономикой, флотом, наукой и торговлей.

#### Базовый алгоритм бюджетирования

```
function allocateBudget(faction):
  // Базовые пропорции зависят от типа фракции (см. раздел 2)
  base = faction.budget_allocation

  // Модификаторы текущей ситуации
  mods = { economy: 1.0, military: 1.0, science: 1.0, trade: 1.0, reserve: 1.0 }

  // Угроза войны → увеличить военный бюджет
  if max_threat_level > 0.7:
    mods.military ×= 1.5
    mods.economy ×= 0.8
    mods.science ×= 0.8
    mods.trade *= 0.7

  // Экономический кризис → увеличить экономический бюджет
  if has_resource_deficit:
    mods.economy ×= 1.3
    mods.military ×= 0.9

  // Мирное время → увеличить научный и торговый бюджет
  if avg_relations > 30 AND max_threat_level < 0.3:
    mods.science ×= 1.3
    mods.trade ×= 1.2

  // На грани поражения → всё в военный бюджет
  if capital_threatened:
    mods.military ×= 3.0
    mods.economy ×= 0.5
    mods.science ×= 0.3
    mods.trade *= 0.2

  // Применяем модификаторы и нормализуем
  adjusted = {}
  total = 0
  for key in base:
    adjusted[key] = base[key] × mods[key]
    total += adjusted[key]

  for key in adjusted:
    adjusted[key] /= total  // нормализация к 1.0

  return adjusted
```

### 3.5 Формула принятия экономического решения

Итоговая формула для выбора экономического действия:

```
utility(action) = need(action) × urgency(action) × feasibility(action)

Где:

need(action) ∈ [0, 1]:
  Насколько фракция нуждается в результате действия.
  Определяется дефицитом целевого ресурса или желаемостью результата.

urgency(action) ∈ [0, 1]:
  Насколько быстро нужно действовать.
  Зависит от времени до критического дефицита:
    urgency = 1 / (1 + ticks_until_crisis / urgency_scale)
    urgency_scale = 100 (настраиваемый параметр)

feasibility(action) ∈ [0, 1]:
  Насколько действие выполнимо прямо сейчас.
  Определяется наличием ресурсов, энергии, слотов, технологий:
    feasibility = Π requirement_met_i
    Где requirement_met_i = 1 если требование выполнено, 0.5 если частично, 0 если нет
```

#### Дерево решений: выбор экономического действия

```
START: Экономический тик
  │
  ├─ Есть критический дефицит энергии? (stockpile < 10 тиков потребления)
  │   └─ ДА → Строить электростанцию (utility += 0.5)
  │
  ├─ Есть критический дефицит базового ресурса? (stockpile < 20 тиков)
  │   └─ ДА → Строить шахту для этого ресурса (utility += 0.4)
  │
  ├─ Все базовые потребности покрыты?
  │   ├─ ДА → Проверить фракционные приоритеты:
  │   │   ├─ Конфедерация: Лаборатория или Торговый хаб
  │   │   ├─ Хегемония: Верфь
  │   │   ├─ Синдикат: Торговый хаб
  │   │   ├─ Орден: Лаборатория
  │   │   └─ Рой: Инкубатор (верфь) или Био-экстрактор (шахта)
  │   └─ НЕТ → Устранить дефицит с наивысшим need_score
  │
  ├─ Есть свободные заводы?
  │   └─ ДА → Назначить производство по цепочке: сырьё → материалы → компоненты
  │
  └─ Есть избыток ресурсов?
      └─ ДА → Сгенерировать торговое предложение
```

## 4. Военный AI

### 4.1 Оценка угроз

Военный AI непрерывно сканирует окружение и оценивает уровень угроз.

#### Алгоритм оценки угроз

```
function assessThreats(faction):
  threats = []

  for system in faction.owned_systems:
    for fleet in system.neighbouring_fleets:  // в радиусе сканера
      if fleet.owner == faction:
        continue  // свой флот — не угроза

      threat_score = calculateThreatScore(fleet, system, faction)
      threats.push({
        fleet: fleet,
        target_system: system,
        score: threat_score,
        time_to_arrival: distance(fleet, system) / fleet.speed,
      })

  return threats.sort(by: score, descending)

function calculateThreatScore(fleet, system, faction):
  power_ratio = fleet.combat_power / system.defense_power

  // Базовая угроза на основе соотношения сил
  base_threat = power_ratio × 50  // 0–100+

  // Модификатор отношений
  relation_mod = (−relations(fleet.owner) + 100) / 200  // 0 (союз) — 1 (враг)

  // Модификатор близости к столице
  capital_mod = 1.0
  if system == faction.capital:
    capital_mod = 2.0
  elif distance(system, faction.capital) < 3:
    capital_mod = 1.5

  // Модификатор ценности системы
  value_mod = system.economic_value / 100  // 0–2+

  return base_threat × relation_mod × capital_mod × value_mod
```

#### Уровни угрозы

| Уровень | Диапазон score | Действие AI |
|---|---|---|
| **Нет угрозы** | 0–10 | Стандартное поведение |
| **Низкая** | 10–30 | Усиление патрулей |
| **Средняя** | 30–60 | Переброска флота к границе |
| **Высокая** | 60–80 | Мобилизация, запрос союзников |
| **Критическая** | 80+ | Полная военная готовность, экстренная верфь |

### 4.2 Формирование флотов

| Тип | Назначение | Состав | Минимальная сила |
|---|---|---|---|
| **Оборонительный** | Защита системы | 30% истребители, 40% крейсера, 30% поддержки | Равна угрозе × 1.2 |
| **Ударный** | Атака системы | 20% истребители, 50% крейсера, 30% тяжёлые | > цель × 1.3 |
| **Набеговый** | Быстрый удар и отход | 60% истребители, 20% крейсера, 20% транспорты | > цель × 0.8 |
| **Патрульный** | Контроль территории | 50% истребители, 30% крейсера, 20% разведчики | Произвольная |
| **Транспортный** | Переброска войск | 80% транспорты, 20% эскорт | Не вступает в бой |

```
function formFleet(faction, mission_type, target_power):
  available_ships = getAvailableShips(faction)

  fleet_composition = FLEET_TEMPLATES[mission_type]

  selected_ships = []
  current_power = 0

  // Выбираем корабли, пока не достигнем целевой мощи
  for priority_class in fleet_composition.priority_order:
    needed_fraction = fleet_composition[priority_class]
    needed_power = target_power × needed_fraction

    ships_of_class = available_ships
      .filter(s => s.class == priority_class)
      .sort(by: combat_power, descending)

    for ship in ships_of_class:
      if current_power_for_class < needed_power:
        selected_ships.push(ship)
        current_power_for_class += ship.combat_power

      if current_power >= target_power:
        break

  // Если не набрали нужную мощь — строим недостающие корабли
  if current_power < target_power:
    deficit = target_power − current_power
    queueShipConstruction(faction, deficit, fleet_composition)

  return createFleet(selected_ships, mission_type)
```

### 4.3 Решение об атаке

AI принимает решение об атаке на основе оценки соотношения сил и стратегической ценности цели.

#### Формула решения об атаке

```
function shouldAttack(faction, target_system):
  my_power = getFleetPowerInRange(faction, target_system, max_jump_distance)
  enemy_power = getSystemDefensePower(target_system)

  // Базовое условие
  attack_threshold = faction.attack_threshold
  // Конфедерация: 1.5, Хегемония: 1.2, Синдикат: ∞, Орден: 1.5, Рой: 1.0

  if my_power <= enemy_power × attack_threshold:
    return { decision: NO, reason: "insufficient_force" }

  // Оценка стратегической ценности
  strategic_value = evaluateStrategicValue(target_system, faction)

  // Оценка риска контратаки
  counterattack_risk = evaluateCounterattackRisk(faction, target_system)

  // Итоговое решение
  attack_utility = (my_power / enemy_power) × strategic_value × (1 − counterattack_risk)

  return {
    decision: attack_utility > ATTACK_UTILITY_THRESHOLD ? YES : NO,
    utility: attack_utility,
    my_power: my_power,
    enemy_power: enemy_power,
  }

evaluateStrategicValue(system, faction):
  value = 0

  // Столица врага — наивысший приоритет
  if system == enemy.capital:
    value += 10.0

  // Верфи — критическая инфраструктура
  value += countBuildings(system, "shipyard") × 3.0

  // Редкие ресурсы
  value += countRareDeposits(system) × 2.0

  // Близость к нашим границам (легче удержать)
  value += max(0, 3 − distanceToNearestOwnedSystem(system, faction)) × 1.0

  // Слабая защита — лёгкая добыча
  defense_factor = 1 / (system.defense_power / 100 + 1)
  value += defense_factor × 2.0

  return value / 20  // нормализация к [0, 1+]
```

| Хегемония | > 1.2 | +0.5 если цель — столица, −0.1 если воюем на 2+ фронтах |
| Синдикат | > 3.0 | Атакует только если угроза торговым маршрутам |
| Орден | > 1.5 | +0.3 если в системе артефакт, −0.3 если война на 2+ фронтах |

function shouldRetreat(faction, fleet, enemy_fleet):
  my_power = fleet.combat_power
  enemy_power = enemy_fleet.combat_power

  retreat_threshold = faction.retreat_threshold
  // Конфедерация: 1.5, Хегемония: 2.5, Синдикат: 1.2, Орден: 1.3, Рой: ∞

  if enemy_power > my_power × retreat_threshold:
    return { decision: RETREAT, reason: "outgunned" }

  // Дополнительные факторы
  losses_ratio = fleet.current_hp / fleet.max_hp
  if losses_ratio < 0.3 AND enemy_power > my_power:
    return { decision: RETREAT, reason: "heavy_losses" }

  // Защита столицы — отступление только при подавляющем превосходстве
  if fleet.mission == "defend_capital":
    if enemy_power < my_power × 3.0:
      return { decision: FIGHT_TO_DEATH, reason: "capital_defense" }

  return { decision: FIGHT, reason: "favorable_odds" }
```

function selectAttackTargets(faction, target_system):
  targets = []

  for planet in target_system.planets:
    for building in planet.buildings:
      priority = TARGET_PRIORITIES[building.type]

      // Модификатор ценности
      value = building.level × priority.base_value

      // Модификатор защиты (слабозащищённые — привлекательнее)
      defense_penalty = 1 / (planet.defense_power / 50 + 1)
      value ×= defense_penalty

      // Фракционные модификаторы
      value ×= faction.target_preference[building.type]
      // Хегемония: верфи × 2.0, Орден: лаборатории × 2.0, Рой: всё × 1.0

      targets.push({ building, planet, value })

  targets.sort(by: value, descending)
  return targets.slice(0, 3)  // атакуем до 3 целей за рейд
```

function combatTactics(fleet, enemy_fleet):
  // 1. Приоритет целей по угрозности
  enemy_ships = enemy_fleet.ships.sort(by: threat_score, descending)
  // threat_score = firepower × (1 + technology_bonus) × hp_ratio

  // 2. Концентрация огня
  primary_target = enemy_ships[0]  // самая опасная цель
  for ship in fleet.ships:
    ship.target = primary_target
  // Все корабли стреляют по одной цели — быстро уничтожаем

  // 3. Переназначение при уничтожении цели
  if primary_target.destroyed:
    primary_target = next_alive(enemy_ships)
    for ship in fleet.ships:
      if ship.alive:
        ship.target = primary_target

  // 4. Приоритет по типу:
  //   а) Корабли с наибольшим уроном (крейсера, артиллерия)
  //   б) Корабли-носители щитов
  //   в) Транспорты (если цель — набег)
  //   г) Остальные

  // 5. Отход при потерях (кроме Роя)
  if fleet.loss_ratio > faction.retreat_loss_threshold:
    if faction != Рой:
      return RETREAT
    else:
      return CONTINUE_FIGHT  // Рой не отступает

  // Условия для объявления войны
  if relations(target) > −20:
    return FALSE  // нет повода

  if faction == Синдикат AND relations(target) > −50:
    return FALSE  // Синдикат не начинает войны

  if faction == Рой:
    return TRUE   // Рой всегда воюет

  // Оценка готовности
  readiness = military_readiness(faction)
  if readiness < 0.5 AND faction != Хегемония:
    return FALSE  // не готовы к войне

  // Объявление войны
  setRelations(faction, target, max(relations(target), −50))
  // Все союзники цели получают −20 к отношениям
  for ally of target:
    modifyRelations(faction, ally, −20)

  return TRUE
```

function proposePeace(faction, target):

  willingness = 0

  // Потери делают мир привлекательнее
  willingness += faction.war_losses / 100

  // Экономический кризис
  if hasResourceDeficit(faction):
    willingness += 0.3

  if activeWarCount(faction) >= 2:

  willingness += relations(target) / 100  // + если отношения не совсем враждебные

  if willingness > 0.5:
    // Генерируем условия мира
    terms = generatePeaceTerms(faction, target, willingness)
    return { decision: PROPOSE_PEACE, terms: terms }

  return { decision: CONTINUE_WAR }

generatePeaceTerms(faction, target, willingness):

  if willingness > 0.9:

  their_offer = selectTechForOffer(their_surplus_techs, my_surplus_techs)

  my_gain = techValue(their_offer)

## 6. Экспансионистский AI

### 6.1 Выбор систем для колонизации

AI оценивает кандидатов на колонизацию по комплексной формуле.

#### Формула оценки системы для колонизации

```
colonization_score(system, faction) =
    habitability_factor × 0.30
  + resource_factor × 0.25
  + proximity_factor × 0.20
  + strategic_factor × 0.15
  + threat_factor × 0.10

Где:

habitability_factor ∈ [0, 1]:
  avg_habitability = Σ(planet.habitability for planet in system.planets) / planet_count
  factor = avg_habitability / 100
  // Рой: factor = 1.0 (всегда максимальный — адаптируется к любой среде)

resource_factor ∈ [0, 1]:
  rare_count = countRareDeposits(system)
  common_count = countCommonDeposits(system)
  total_count = countAllDeposits(system)
  factor = min(1.0, (rare_count × 3 + common_count) / 20)
  // Синдикат: rare_count × 5 вместо × 3 (ищет редкие ресурсы)

proximity_factor ∈ [0, 1]:
  distance_to_border = minDistanceToOwnedSystem(system, faction)
  factor = 1.0 / (1.0 + distance_to_border / 3)
  // Близкие системы = фактор ≈ 1.0, далёкие = фактор ≈ 0

strategic_factor ∈ [0, 1]:
  is_choke_point = countAccessRoutes(system) <= 2  // узловая система
  is_trade_route = isOnTradeRoute(system)
  has_artifacts = hasStrangerArtifacts(system)
  factor = (is_choke_point ? 0.4 : 0) + (is_trade_route ? 0.3 : 0) + (has_artifacts ? 0.3 : 0)

threat_factor ∈ [0, 1]:
  nearest_enemy_distance = minDistanceToEnemySystem(system, faction)
  factor = min(1.0, nearest_enemy_distance / 5)
  // Далеко от врагов = фактор ≈ 1.0, рядом = фактор ≈ 0
  // Хегемония: фактор инвертирован (ближе к врагам = лучше, фактор = 1 − factor)
```

#### Приоритет колонизации по типу системы

| Приоритет | Тип системы | Пример | Вес |
|---|---|---|---|
| 1 | Система с редкими/уникальными ресурсами | Ir, Os, Re залежи | × 3.0 |
| 2 | Обитаемая система (габитабельность > 60%) | Землеподобная планета | × 2.0 |
| 3 | Ресурсная система (много распространённых) | Fe, Ti, Cu залежи | × 1.5 |
| 4 | Система с артефактами Странников | Заброшенный корабль | × 2.5 (Орден: × 4.0) |
| 5 | Пустая система | Минимум ресурсов | × 0.5 |
| 6 | Узловая (choke point) | 1–2 гипермаршрута | × 2.0 |

### 6.2 Скорость колонизации

Скорость колонизации зависит от типа фракции, наличия колонизационных кораблей и ресурсов.

#### Время колонизации

```
colonization_time(system, faction) =
    base_time
  × faction_colonization_modifier
  × (1 + distance_penalty)
  × habitability_modifier

Где:
  base_time = 200 тиков (базовое время колонизации)

  faction_colonization_modifier:
    Конфедерация: 1.0
    Хегемония:    0.9  (быстрее за счёт военной дисциплины)
    Синдикат:     1.1  (медленнее — найми специалистов)
    Орден:        1.2  (медленнее — мало людей)
    Рой:          0.7  (быстрее — биоразмножение, бонус +30%)

  distance_penalty = distance_to_nearest_colony / 10
  // Дальние колонии колонизируются медленнее (логистика)

  habitability_modifier = 1 / (0.5 + avg_habitability / 100)
  // Высокая габитабельность → модификатор ≈ 0.67 (быстрее)
  // Низкая габитабельность → модификатор ≈ 2.0 (медленнее)
  // Рой: habitability_modifier = 0.8 (всегда быстро, адаптация)
```

### 6.3 Алгоритм экспансии

```
function expansionCycle(faction):
  // Проверяем, есть ли свободные слоты для колонизации
  if activeColonizations(faction) >= faction.max_colonizations:
    return  // все колонизационные ресурсы заняты

  // Получаем список кандидатов
  candidates = getUncolonizedSystemsInRange(faction, max_range: 5)

  if candidates.length == 0:
    // Нет свободных систем — рассмотреть захват (кроме Синдиката)
    if faction != Синдикат:
      considerMilitaryExpansion(faction)
    return

  // Оцениваем каждую систему
  scored = []
  for system in candidates:
    score = colonizationScore(system, faction)
    if score > faction.colonization_threshold:
      // Конфедерация: 0.4, Хегемония: 0.2, Синдикат: 0.6, Орден: 0.5, Рой: 0.1
      scored.push({ system, score })

  // Сортируем и колонизируем лучшую
  scored.sort(by: score, descending)

  if scored.length > 0:
    best = scored[0]
    launchColonization(faction, best.system)
```

## 7. Уровни сложности AI

### 7.1 Обзор уровней сложности

Уровень сложности влияет на эффективность принятия решений AI, скорость развития и стартовые условия. **AI не получает информационных преимуществ** (не видит скрытые данные игрока) — разница только в качестве решений и экономических бонусах.

| Параметр | Лёгкий | Нормальный | Сложный | Безумный |
|---|---|---|---|---|
| **Бонус производства** | −20% | 0% | +10% | +50% |
| **Бонус исследований** | −10% | 0% | +10% | +30% |
| **Бонус торговли** | −10% | 0% | +10% | +20% |
| **Стартовые ресурсы** | × 0.7 | × 1.0 | × 1.2 | × 2.0 |
| **Атакует первым** | Нет | При преимуществе | Агрессивно | Максимально |
| **Качество решений** | Субоптимальные | Разумные | Оптимальные | Оптимальные |
| **Частота обновления** | Каждый 20 тиков | Каждый 10 тиков | Каждый 5 тиков | Каждый тик |
| **Стратегическое планирование** | Нет | Базовое | Полное | Полное + адаптация |

### 7.2 Лёгкий (Easy)

**Целевая аудитория**: Новички, первый контакт с жанром 4X.

#### Поведение AI

- **Экономика**: Медленное развитие. Строит здания неоптимально (случайный выбор из верхних 50% utility). Не реинвестирует прибыль.
- **Военное дело**: Не атакует первым. Строит минимальный оборонительный флот. Отступает при любом перевесе врага (порог 1:1.2).
- **Дипломатия**: Пассивная. Принимает большинство предложений игрока. Не инициирует союзы.
- **Экспансия**: Медленная. Колонизирует только системы с габитабельностью > 50%.

#### Модификаторы решений

```
// Лёгкий AI принимает субоптимальные решения
function easyDecisionModifer(candidates):
  // Вместо лучшего — случайный из верхней половины
  top_50_percent = candidates.slice(0, ceil(candidates.length / 2))
  return randomChoice(top_50_percent)

// Задержка реакции
REACTION_DELAY = 20 тиков  // AI «думает» дольше
```

#### Ограничения

- Не формирует ударные флоты (только оборонительные и патрульные)
- Не использует набеги
- Не инициирует обмен технологиями
- Не объявляет ультиматумы
- Колонизирует максимум 1 систему одновременно

### 7.3 Нормальный (Normal)

**Целевая аудитория**: Опытные игроки стратегий.

#### Поведение AI

- **Экономика**: Стандартное развитие. Строит здания по utility (выбирает лучшее). Балансирует бюджет.
- **Военное дело**: Атакует при преимуществе (порог зависит от фракции). Строит сбалансированные флоты.
- **Дипломатия**: Умеренно активная. Предлагает торговлю и обмен. Отвечает на угрозы.
- **Экспансия**: Умеренная. Колонизирует системы с габитабельностью > 30%.

#### Модификаторы решений

```
function normalDecisionModifier(candidates):
  // Выбирает лучшее действие (макс utility)
  return candidates[0]  // после сортировки по utility

// Стандартная частота
REACTION_DELAY = 10 тиков
```

### 7.4 Сложный (Hard)

**Целевая аудитория**: Ветераны 4X, игроки Aurora 4X.

#### Поведение AI

- **Экономика**: Оптимальное развитие. Выбирает лучшие решения. Опережает средний темп игрока на 10–15%.
- **Военное дело**: Агрессивная стратегия. Атакует при минимальном преимуществе. Координирует несколько флотов.
- **Дипломатия**: Стратегическая. Заключает союзы против сильнейшего (баланс сил). Использует эмбарго.
- **Экспансия**: Быстрая. Колонизирует все доступные системы. Конкурирует за ключевые системы.

#### Модификаторы решений

```
function hardDecisionModifier(candidates):
  // Выбирает лучшее + учитывает долгосрочные последствия
  for candidate in candidates:
    candidate.utility += evaluateLongTermBenefit(candidate) × 0.3
  return candidates[0]

// Быстрая реакция
REACTION_DELAY = 5 тиков

// Координация флотов
ENABLE_FLEET_COORDINATION = true  // несколько флотов атакуют одну цель
ENABLE_STRATEGIC_TARGETING = true  // приоритет верфям и столицам
```

#### Дополнительные возможности (Сложный)

- **Координация флотов**: 2+ флота атакуют одну систему одновременно
- **Ложные цели**: Разведчиками имитирует подготовку атаки (заставляет врага распылять силы)
- **Экономическая война**: Эмбарго, скупка дефицитных ресурсов, монополизация
- **Адаптация**: Меняет стратегию при неудачах (если атака провалилась → усилить флот перед повторной)

### 7.5 Безумный (Insane)

**Целевая аудитория**: Мазохисты, мастера жанра.

#### Поведение AI

- **Экономика**: Максимально эффективная. Каждый тик — оптимальное решение. Бонус +50% к производству.
- **Военное дело**: Тотальная война с первого контакта. Атакует при равенстве сил. Координирует все флоты.
- **Дипломатия**: Минимальная (использует только для выгоды). Заключает союз, чтобы ударить в спину.
- **Экспансия**: Максимальная. Колонизирует быстрее всех. Конкурирует за каждую систему.

#### Модификаторы решений

```
function insaneDecisionModifier(candidates):

    candidate.utility += evaluateAntiPlayerBenefit(candidate) × 0.3

// Полная координация

ENABLE_ADAPTATION = true  // меняет стратегию после каждого поражения

// Бонусы
PRODUCTION_BONUS = 1.50  // +50% к производству

## 8. Оптимизация AI для производительности

### 8.1 Проблема производительности

При 3–5 AI-фракциях, каждая из которых владеет 5–30 системами с 3–10 планетами и десятками флотов, полный пересчёт каждого AI каждый тик недопустим. Необходима система оптимизации.

#### Оценка нагрузки

| Сущность | Количество (среднее) | Операций на тик | Итого |
|---|---|---|---|
| AI-фракции | 4 | 1 стратегический цикл | 4 |
| Системы AI | 80 | 5 (экономика, строительство) | 400 |
| Планеты AI | 400 | 3 (добыча, производство, очереди) | 1200 |
| Флоты AI | 20 | 10 (перемещение, приказы, бои) | 200 |
| Дипломатия | 6 пар | 5 (оценка, предложения) | 30 |
| **Итого без оптимизации** | | | **~1834 операций/тик** |
| **Итого с оптимизацией** | | | **~200 операций/тик** |

### 8.2 Стратегия 1: Удалённые AI-империи — редкое обновление

AI-системы, расположенные далеко от игрока, обновляются реже.

#### Зоны обновления

```
Для каждой AI-фракции определяем расстояние до игрока:

distance = minDistance(any_owned_system, any_player_system)

if distance <= 2:
  zone = ACTIVE       // полный пересчёт каждый тик
elif distance <= 5:
  zone = VISIBLE      // упрощённый пересчёт каждые 5 тиков
elif distance <= 10:
  zone = DISTANT      // редкий пересчёт каждые 10 тиков
else:
  zone = REMOTE       // модель прироста каждые 50 тиков
```

| Зона | Расстояние | Частота обновления | Модель |
|---|---|---|---|
| **Активная** | ≤ 2 прыжков | Каждый тик | Полная симуляция (как игрок) |
| **Видимая** | 3–5 прыжков | Каждые 5 тиков | Упрощённая: строительство + производство |
| **Дальняя** | 6–10 прыжков | Каждые 10 тиков | Формульный прирост |
| **Удалённая** | > 10 прыжков | Каждые 50 тиков | Формульный прирост + макро-решения |

### 8.3 Стратегия 2: Упрощённая модель для удалённых систем

Вместо посимвольного производства (каждая шахта добывает, каждый завод перерабатывает) — прирост по формуле.

#### Формула прироста ресурсов (упрощённая модель)

```
// Для каждой AI-системы в дальней/удалённой зоне:
function simplifiedGrowth(system, ticks_elapsed):
  // Прирост базовых ресурсов
  for resource in system.produced_resources:
    base_rate = system.mine_count × mine_output_per_level × faction.production_modifier
    growth = base_rate × ticks_elapsed
    system.stockpile[resource] += growth

  // Потребление заводами
  for recipe in system.active_recipes:
    input_consumed = recipe.input × ticks_elapsed × 0.8  // 80% эффективности (не оптимально)
    output_produced = recipe.output × ticks_elapsed × 0.8
    system.consume(input_consumed)
    system.produce(output_produced)

  // Упрощённое строительство
  if system.has_build_queue:
    build_progress += system.factory_output × ticks_elapsed
    if build_progress >= required_progress:
      completeBuilding(system)
```

#### Сравнение моделей

| Аспект | Полная модель | Упрощённая модель |
|---|---|---|
| Точность | 100% | ~90% (приемлемая погрешность) |
| Операций/тик на систему | ~50 | ~5 |
| Когда видимый результат важен | Активная + Видимая зоны | Нет |
| Когда результат «за кадром» | Нет | Дальняя + Удалённая зоны |

### 8.4 Стратегия 3: Ленивое вычисление решений

AI не просчитывает все возможные варианты — он выбирает из 3–5 приоритетных кандидатов.

#### Алгоритм ленивого выбора

```
function lazyDecision(faction, decision_type):
  // Вместо перебора всех действий (может быть 100+):
  // 1. Предфильтрация: берём только релевантные действия
  candidates = getCandidateActions(faction, decision_type)

  // 2. Ограничиваем до топ-5 по грубой оценке
  for candidate in candidates:
    candidate.rough_score = roughUtility(candidate)  // быстрая формула без деталей
  candidates.sort(by: rough_score, descending)
  candidates = candidates.slice(0, 5)

  // 3. Точная оценка только для топ-5
  for candidate in candidates:
    candidate.utility = preciseUtility(candidate)  // полная формула

  // 4. Выбираем лучшее
  return candidates[0]
```

#### Грубая vs точная оценка

```
// Грубая оценка (быстрая, без деталей)
roughUtility(action):
  return need(action) × 0.5  // только потребность, без срочности и осуществимости

// Точная оценка (полная)
preciseUtility(action):
  return need(action) × urgency(action) × feasibility(action)
```

#### Лимиты вычислений

| Тип решения | Максимум кандидатов | Частота |
|---|---|---|
| Строительство (что строить) | 5 | Каждый 10 тик |
| Производство (что производить) | 3 | Каждый 5 тик |
| Военное (атаковать/защищаться) | 3 | Каждый тик (если угроза) / каждый 10 тик |
| Дипломатия (предложить/ответить) | 2 | Каждый 50 тик |
| Экспансия (куда колонизировать) | 3 | Каждый 100 тик |

### 8.5 Стратегия 4: Кэширование решений

AI кэширует результаты вычислений и переиспользует их, если ситуация не изменилась существенно.

```
interface DecisionCache {
  last_evaluation_tick: number;
  situation_hash: string;    // хеш ключевых параметров состояния
  decision: Action;
  utility: number;
}

function getCachedDecision(faction, decision_type, current_tick):
  cache = faction.decision_cache[decision_type]

  if cache == NULL:
    return NULL  // нет кэша

  if current_tick − cache.last_evaluation_tick > MAX_CACHE_AGE:
    return NULL  // кэш устарел

  // Проверяем, изменилась ли ситуация
  current_hash = computeSituationHash(faction, decision_type)
  if current_hash != cache.situation_hash:
    return NULL  // ситуация изменилась, кэш недействителен

  return cache.decision  // используем кэшированное решение

// Хеш считается по ключевым параметрам (быстро):
function computeSituationHash(faction, decision_type):
  switch decision_type:
    case "economy":
      return hash(faction.resources, faction.building_queue, faction.energy_balance)
    case "military":
      return hash(faction.fleet_power, faction.threat_level, faction.active_wars)
    case "diplomacy":
      return hash(faction.relations, faction.active_agreements)
    case "expansion":
      return hash(faction.colonization_progress, faction.available_systems)
```

#### Максимальный возраст кэша

| Тип решения | MAX_CACHE_AGE (тиков) |
|---|---|
| Экономика | 10 |
| Военное | 5 |
| Дипломатия | 25 |
| Экспансия | 50 |

### 8.6 Стратегия 5: Пакетная обработка

Несколько AI-фракций обрабатываются в разных тиках, а не все одновременно.

```
// Распределение AI по тикам (пример для 4 фракций):
tick % 4 == 0 → Обновить Конфедерацию (стратегический цикл)
tick % 4 == 1 → Обновить Хегемонию (стратегический цикл)
tick % 4 == 2 → Обновить Синдикат (стратегический цикл)
tick % 4 == 3 → Обновить Орден (стратегический цикл)

// Тактические циклы — чаще, но тоже распределены:
tick % 2 == 0 → Обновить Конфедерацию + Синдикат (тактика)
tick % 2 == 1 → Обновить Хегемонию + Орден (тактика)

// Реактивные циклы — ВСЕ фракции каждый тик (но только при событиях):
on_event(ATTACK, BORDER_INCIDENT) → Обновить все фракции (реактивно)
```

### 8.7 Итоговая архитектура обновления AI

```
Каждый тик:
  ├─ Реактивные события (для ВСЕХ фракций)
  │   └─ Атака на систему → немедленная оборона
  │   └─ Нарушение границ → оценка угрозы
  │   └─ Дипломатическое предложение → ответ
  │
  ├─ Тактический цикл (2 фракции, ротация)
  │   └─ Строительство
  │   └─ Производство
  │   └─ Перемещение флотов
  │
  ├─ Стратегический цикл (1 фракция, ротация)
  │   └─ Пересмотр целей GOAP
  │   └─ Бюджетирование
  │   └─ Дипломатия
  │
  └─ Упрощённое обновление (дальние системы)
      └─ Формульный прирост ресурсов
      └─ Упрощённое строительство
```

### 8.8 Бюджет производительности

| Категория | Лимит (мс/тик) | Доля |
|---|---|---|
| Реактивные события | 2 | 10% |
| Тактический AI | 8 | 40% |
| Стратегический AI | 4 | 20% |
| Упрощённое обновление | 4 | 20% |
| Дипломатия | 2 | 10% |
| **Итого** | **20** | **100%** |

> При скорости x50 (50 тиков/сек) бюджет = 20мс/тик = 1000мс/сек. Web Worker справится.
