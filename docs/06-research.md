## 1. Обзор системы

Система исследований — одна из ключевых механик игры. Она определяет прогресс цивилизации игрока: от примитивных химических двигателей и стальных корпусов до варп-прыжков, антиматерии и тайн Предтеч.

> **Двухуровневая система**: специализированные ветки (описаны ниже) опираются на фундаментальные ветки (описаны в [ARCHITECTURE.md](./ARCHITECTURE.md) §3.2.1). Уровень фундаментальной ветки определяет максимальный доступный уровень технологий в специализированной. Подробности — в [research-unification.md](../doc_temp/research-unification.md).

### Принципы проектирования

| Принцип | Описание |
|---------|----------|
| **Глубина, не ширина** | 6 веток с 10–12 технологиями каждая, но каждая технология имеет уровни 1–10+, что даёт сотни комбинаций |
| **Осмысленный выбор** | Невозможно исследовать всё параллельно; приоритеты определяют стиль игры |
| **Экспоненциальный рост** | Высокие уровни стоят всё дороже — игрок решает, углубляться ли или расширяться |
| **DAG-зависимости** | Направленный ациклический граф; нет циклов, есть чёткие пути развития |
| **Открытия извне** | Артефакты и руины дают альтернативные пути прогресса, минуя дерево |

### Связь с другими системами

| Система | Влияние исследований |
|---------|---------------------|
| **Производство** | Новые сплавы и материалы открывают рецепты крафта |
| **Корабли** | Новые двигатели, щиты, оружие, модули доступны после исследования |
| **Планеты** | Терраформирование улучшает габитабельность; автоматизация снижает потребность в рабочей силе |
| **Дипломатия** | Обмен технологиями — важный инструмент; редкие технологии — предмет торга |
| **Бой** | Уровни оружия и защиты напрямую определяют исход сражений |
| **Разведка** | Датчики и сканеры позволяют находить руины и артефакты |

## 3. Механика исследований

### 3.1 Очки исследований (RP — Research Points)

**Базовая единица** — RP (Research Points). Лаборатории производят RP в секунду игрового времени.

| Параметр | Формула | Пример |
|----------|---------|--------|
| **RP/сек одной лаборатории** | `base_output × level × (1 + biology_hab_bonus)` | Лаб. ур.3, база 5, бонус 0.1 → 5 × 3 × 1.1 = 16.5 RP/сек |
| **Суммарный RP/сек** | `Σ(лаборатории на всех планетах)` | 3 лаб. ур.3 + 2 лаб. ур.5 = 49.5 + 55 = 104.5 RP/сек |
| **Базовый выход лаборатории** | 5 RP/сек на ур.1 | — |
| **Бонус габитабельности** | `hab% / 500` (макс. +0.2 при 100% габ.) | 80% габ. → +0.16 |

### 3.2 Стоимость и время исследования

| Параметр | Формула | Описание |
|----------|---------|----------|
| **Стоимость уровня N** | `base_cost × 1.5^(N-1)` | Экспоненциальный рост |
| **Минимальное время** | `base_cost × 1.5^(N-1) / (RP_allocated × 1.2)` | При 100% фокусе |
| **Расчётное время** | `remaining_RP / RP_per_sec_allocated` | Реальное время завершения |

> **Минимальное время исследования** — защита от мгновенного «закупа» технологии при огромном RP/сек. Даже если RP хватает, технология не может быть исследована быстрее чем за `min_time = base_cost / 1000` секунд.

### 3.3 Распределение RP

Игрок распределяет доступный RP между активными исследованиями в процентах.

```
Пример: всего 100 RP/сек

Слот 1: Ионный двигатель (ур.3)  →  40%  →  40 RP/сек
Слот 2: Лазерное оружие (ур.2)   →  35%  →  35 RP/сек
Слот 3: Сталеплавильное дело (ур.5) → 25% → 25 RP/сек
─────────────────────────────────────────────────────
Итого: 100%  →  100 RP/сек
```

**Правила распределения**:
- Сумма процентов по всем активным слотам = 100%
- Минимальное выделение на один слот — 5% (нельзя «заморозить» исследование, выделив 0%)
- Перераспределение возможно в любой момент, уже вложенные RP не теряются
- При завершении исследования в слоте RP перераспределяются автоматически между оставшимися

### 3.4 Параллельные исследования

| Формула | Описание |
|---------|----------|
| `max_slots = 1 + floor(total_labs / 10)` | Количество одновременных исследований |

**Таблица слотов по количеству лабораторий**:

| Лабораторий | Слотов | Комментарий |
|-------------|--------|-------------|
| 1–9 | 1 | Ранняя игра: фокус критически важен |
| 10–19 | 2 | Можно вести 2 параллельных проекта |
| 20–29 | 3 | Средняя игра |
| 30–39 | 4 | — |
| 40–49 | 5 | — |
| 50–59 | 6 | Поздняя игра |
| 60–69 | 7 | — |
| 70–79 | 8 | — |
| 80–89 | 9 | — |
| 90+ | 10 | Максимум |

> **Мягкий лимит**: даже при 100 лабораториях максимальное количество параллельных исследований = 10. Это предотвращает распыление усилий и делает выбор значимым.

### 3.5 Ускорение от фокуса

Если игрок выделяет **100% RP на одну технологию** (все слоты кроме одного пусты), он получает **бонус +20% к скорости**.

| Выделено RP | Эффективность | Пример (100 RP/сек) |
|-------------|--------------|---------------------|
| 100% на 1 слот | ×1.2 (фокус-бонус) | 120 RP/сек эффективных |
| 50% + 50% на 2 слота | ×1.0 каждый | 50 + 50 = 100 RP/сек |
| 25% × 4 слота | ×1.0 каждый | 25 × 4 = 100 RP/сек |

**Формула фокус-бонуса**:

```
if (active_slots == 1 && allocation_percent == 100) {
    efficiency_multiplier = 1.2;
} else {
    efficiency_multiplier = 1.0;
}
```

> **Дизайнерское обоснование**: Бонус вознаграждает сфокусированную стратегию, но не делает её единственно верной — параллельные исследования дают гибкость.

## 4. Уровни технологий

### 4.1 Базовые правила

- Каждая технология имеет **уровни от 1 до 10+**
- Уровень 1 — базовый, открывает технологию и её первичные эффекты
- Каждый следующий уровень **усиливает базовый параметр на 10–25%** (зависит от технологии)
- Стоимость уровня растёт экспоненциально

### 4.2 Формула стоимости уровня

```
cost(level_N) = base_cost × 1.5^(N - 1)
```

**Пример: Ионный двигатель (base_cost = 800 RP)**

| Уровень | Множитель | Стоимость (RP) | Кумулятивно (RP) | Эффект |
|---------|-----------|----------------|-------------------|--------|
| 1 | 1.0 | 800 | 800 | Тяга: 100 кН, Расход: 1.0 |
| 2 | 1.5 | 1 200 | 2 000 | Тяга: 120 кН (+20%), Расход: 0.95 |
| 3 | 2.25 | 1 800 | 3 800 | Тяга: 144 кН (+20%), Расход: 0.90 |
| 4 | 3.375 | 2 700 | 6 500 | Тяга: 173 кН (+20%), Расход: 0.86 |
| 5 | 5.06 | 4 050 | 10 550 | Тяга: 207 кН (+20%), Расход: 0.81 |
| 6 | 7.59 | 6 075 | 16 625 | Тяга: 249 кН (+20%), Расход: 0.77 |
| 7 | 11.39 | 9 112 | 25 737 | Тяга: 299 кН (+20%), Расход: 0.73 |
| 8 | 17.09 | 13 669 | 39 406 | Тяга: 359 кН (+20%), Расход: 0.70 |
| 9 | 25.63 | 20 503 | 59 909 | Тяга: 430 кН (+20%), Расход: 0.66 |
| 10 | 38.44 | 30 755 | 90 664 | Тяга: 516 кН (+20%), Расход: 0.63 |

> **Уровни выше 10** — доступны, но стоимость продолжает расти по формуле. Это для сверхпоздней игры / песочницы.

### 4.3 Типы улучшений по уровням

| Тип улучшения | Формула | Пример |
|---------------|---------|--------|
| **Линейный** | `base × (1 + 0.10 × (level - 1))` | +10% за уровень |
| **Прогрессивный** | `base × (1 + 0.10 × (level - 1))^1.2` | Ускоряющийся рост |
| **Пороговый** | Эффект на уровнях 1, 3, 5, 8, 10 | Новые возможности на ключевых уровнях |
| **Убывающий** | `base × (1 + 0.25 × (1 - 0.9^(level - 1)))` | Быстрый старт, замедляющийся рост |

> Конкретный тип улучшения указан в описании каждой технологии (раздел 6).

## 5. Зависимости (пре-реквизиты)

### 5.1 Структура зависимости

Технология может требовать:

| Тип требования | Обозначение | Пример |
|----------------|-------------|--------|
| **Уровень другой технологии** | `tech_id >= N` | Ионный двигатель ≥ ур.5 |
| **Технология целиком (любой уровень)** | `tech_id >= 1` | Микроэлектроника изучена |
| **Уровень своей ветки** | `branch_level >= N` | Энергетика ≥ ур.3 (сумма уровней ≥ 3) |

**Определение `branch_level`**: сумма максимальных уровней всех изученных технологий в данной ветке. Не путать с уровнем конкретной технологии.

### 5.2 Правила DAG

1. **Направленный ациклический граф** — нет циклических зависимостей
2. **Однонаправленность** — если A → B, то не может быть B → A
3. **Кросс-веточные связи** — технологии одной ветки могут зависеть от технологий другой
4. **Множественные пре-реквизиты** — технология может требовать до 3 пре-реквизитов одновременно (логическое AND)
5. **Нет OR-зависимостей** — все перечисленные пре-реквизиты должны быть выполнены

### 5.3 Визуализация формата зависимости

```
Варп-двигатель (ур.1)
├── ПРЕ-РЕКВИЗИТ 1: Ионный двигатель ≥ ур.5     ← ветка Энергетика
├── ПРЕ-РЕКВИЗИТ 2: Энергетика (branch_level) ≥ 10  ← суммарный уровень ветки
└── ПРЕ-РЕКВИЗИТ 3: Сверхпроводники ≥ ур.3      ← ветка Материаловедение
```

### 5.4 Состояния технологий в UI

| Состояние | Цвет | Условие |
|-----------|------|---------|
| **Изучена** | 🟢 Зелёный | Текущий уровень ≥ 1 |
| **Доступна** | 🔵 Синий | Все пре-реквизиты выполнены |
| **Частично доступна** | 🟡 Жёлтый | Часть пре-реквизитов выполнена |
| **Заблокирована** | 🔴 Красный | Ни один пре-реквизит не выполнен |
| **В процессе** | 🔄 Пульсирующий | Активно исследуется |

## 7. Артефакты и руины

### 7.1 Обнаружение руин

Руины древних цивилизаций обнаруживаются при исследовании планет. Шанс находки зависит от нескольких факторов:

| Фактор | Влияние |
|--------|---------|
| **Тип планеты** | Скалистые: +5%, Океанические: +3%, Вулканические: +1%, Газовые: 0% |
| **Уровень Археологической разведки (X1)** | +3% за уровень |
| **Уровень Глубокого сканера (C8)** | +5% за уровень |
| **Размер планеты** | Крупные планеты имеют больше руин |
| **Удалённость от ядра галактики** | Дальние системы: +10% (реже посещались) |

**Базовый шанс обнаружения**: 5% при первом сканировании планеты.

**Формула**:
```
ruin_discovery_chance = 0.05
    + planet_type_bonus
    + (x1_level × 0.03)
    + (c8_level × 0.05)
    + distance_bonus
```

### 7.2 Типы руин

| Тип руин | Вероятность | Описание | Мин. уровень X1 |
|----------|-------------|----------|-----------------|
| **Мелкие остатки** | 60% | Обломки строений, бытовые артефакты | 1 |
| **Научная станция** | 20% | Заброшенная лаборатория с данными | 2 |
| **Военный комплекс** | 10% | Арсенал с оружием и чертежами | 3 |
| **Хранилище Предтеч** | 7% | Запечатанное хранилище с технологиями | 5 |
| **Мегаполис Странников** | 3% | Целый город с множеством артефактов | 7 |

### 7.3 Награды за руины

При успешном изучении руин можно получить:

| Награда | Шанс | Описание |
|---------|------|----------|
| **Готовая технология** | 15% | Случайная технология ур.1–3, для которой выполнены пре-реквизиты |
| **Бонус RP** | 30% | Одноразовый приток 500–5000 RP (зависит от типа руин) |
| **Уникальный рецепт** | 20% | Рецепт недоступный из обычного дерева технологий |

### 7.4 Процесс изучения руин

```
1. ОБНАРУЖЕНИЕ → сканирование планеты (автоматическое при наличии X1 ≥ 1)
2. РАЗВЕДКА   → отправка экспедиции (требуется корабль с модулем «Научная команда»)
3. РАСКОПКИ    → время: 30–300 сек. игрового времени (зависит от размера руин)
                 модификатор: X3 уровень × 0.9 (каждый уровень ускоряет на 10%)
4. ИЗУЧЕНИЕ    → время: 60–600 сек. игрового времени
                 модификатор: X2 уровень × 0.85 (каждый уровень ускоряет на 15%)
5. РЕЗУЛЬТАТ   → расчёт награды по таблице выше
                 модификатор: X4 уровень → +5% шанс редкой награды за уровень
```

### 7.5 Уникальные рецепты из руин

Эти рецепты **недоступны** через обычное дерево технологий и могут быть получены **только** из руин:

## 8. Торговля технологиями

### 8.1 Общие правила

| Правило | Описание |
|---------|----------|
| **Двусторонний обмен** | Можно предложить технологию и запросить технологию взамен |
| **Односторонняя передача** | Можно подарить или продать технологию за ресурсы/деньги |
| **Проверка пре-реквизитов** | Нельзя передать технологию, у которой у получателя не выполнены пре-реквизиты |
| **Уровень передачи** | Передаётся конкретный уровень; получатель получает max(свой_уровень, переданный_уровень) |
| **Кулдаун** | После обмена — 600 сек. игрового времени до следующего предложения |

### 8.2 Стоимость технологий в торговле

**Формула стоимости**:
```
trade_value = base_cost × 1.5^(level - 1) × rarity_multiplier × relation_modifier
```

| Множитель редкости | Условие |
|--------------------|---------|
| ×1.0 | Стартовые технологии (P1, M1, W1, C1, B1) |
| ×1.5 | Технологии ур. 2–3 в своей ветке |
| ×2.0 | Технологии ур. 4–5, или кросс-веточные |
| ×3.0 | Технологии ур. 6+ |
| ×5.0 | Терминальные технологии (P12, M12, W12, C12, B12, X12) |
| ×10.0 | Технологии, полученные из руин |

**Модификатор отношений**:

| Отношения | Модификатор |
|-----------|------------|
| Война (−100 to −50) | Торговля невозможна |
| Холод (−49 to −20) | ×3.0 |
| Нейтральные (−19 to +19) | ×2.0 |
| Дружеские (+20 to +60) | ×1.5 |
| Союз (+61 to +100) | ×1.0 |

### 8.3 Примеры обмена

**Пример 1: Обмен технологиями (союз)**
```
Игрок предлагает: Лазерное оружие (W2) ур.3
  → Стоимость: 700 × 1.5^2 × 2.0 × 1.0 = 3 150 очков торговли

Игрок запрашивает: Ионный двигатель (P2) ур.2
  → Стоимость: 800 × 1.5^1 × 1.5 × 1.0 = 1 800 очков торговли

Разница: 3 150 − 1 800 = +1 350 → AI доплачивает ресурсами
```

**Пример 2: Односторонняя продажа (нейтральные отношения)**
```
Игрок продаёт: Сталеплавильное дело (M1) ур.5
  → Стоимость: 300 × 1.5^4 × 1.0 × 2.0 = 3 037.5 очков торговли
  → AI предлагает: 3 000 единиц ресурсов (Fe, C, Ti — случайный набор)
```

### 8.4 AI-поведение при торговле

| Фактор | Влияние |
|--------|---------|
| **Военная фракция (Хегемония)** | Не продаёт военные технологии; охотно покупает их |
| **Торговая фракция (Синдикат)** | Продаёт любые технологии по рыночной цене; предлагает скидки при больших объёмах |
| **Научная фракция (Орден)** | Охотно обменивается технологиями; требует экзотические технологии взамен |
| **Сбалансированная (Конфедерация)** | Стандартное поведение |

**AI не будет**:
- Передавать технологии, которые являются единственным преимуществом (например, уникальные боевые ИИ при военном превосходстве)
- Принимать технологии, для которых нет пре-реквизитов
- Обмениваться технологиями с фракцией, с которой идёт война

## 9. Формулы и данные для программиста

### 9.1 Структура данных технологии (TypeScript)

```typescript
interface Technology {
  id: string;                    // Уникальный ID (напр. "ion_engine")
  name: string;                  // Отображаемое имя
  nameEn: string;                // Английское имя (для кода)
  branch: BranchId;              // Ветка: 'power' | 'materials' | 'weapons' | 'computing' | 'biology' | 'xenoarch'
  baseCost: number;              // Базовая стоимость RP (уровень 1)
  maxLevel: number;              // Максимальный уровень
  improvementType: 'linear' | 'progressive' | 'threshold' | 'diminishing';
  improvementPerLevel: number;   // Процент улучшения за уровень
  prerequisites: Prerequisite[]; // Массив пре-реквизитов
  effects: TechEffect[];         // Массив эффектов
  description: string;           // Описание для UI
  icon: string;                  // Имя иконки
  sortOrder: number;             // Порядок в ветке (для UI)
}

type BranchId = 'power' | 'materials' | 'weapons' | 'computing' | 'biology' | 'xenoarch';

interface Prerequisite {
  techId: string;                // ID технологии
  minLevel: number;              // Минимальный требуемый уровень
}

interface TechEffect {
  target: string;                // Цель: 'ship_engine_thrust' | 'reactor_output' | 'shield_hp' | ...
  operation: 'multiply' | 'add' | 'unlock';
  value: number;                 // Значение (множитель или абсолютное)
  perLevel: boolean;             // Применяется за каждый уровень?
  thresholdLevel?: number;       // Для типа 'threshold' — уровень, на котором срабатывает
}
```

### 9.7 Расчёт эффективного RP/сек для исследования

```typescript
function getEffectiveRPPerSec(
  totalRPPerSec: number,
  allocationPercent: number,
  activeSlots: number,
  quantumComputingLevel: number = 0
): number {
  const allocated = totalRPPerSec * (allocationPercent / 100);
  const focusMultiplier = getFocusBonus(activeSlots, allocationPercent);
  const quantumBonus = 1 + (quantumComputingLevel * 0.25); // +25% за уровень C10
  return allocated * focusMultiplier * quantumBonus;
}
```

### 9.9 Расчёт времени завершения

```typescript
function getEstimatedCompletionTime(
  remainingRP: number,
  effectiveRPPerSec: number,
  baseCost: number

  const timeByRP = remainingRP / effectiveRPPerSec;
  const minTime = getMinResearchTime(baseCost);
  return Math.max(timeByRP, minTime);
}
```

### 9.10 Проверка пре-реквизитов

```typescript
function arePrerequisitesMet(
  tech: Technology,
  researchedTechs: Map<string, number>  // techId → currentLevel
): { met: boolean; details: PrerequisiteCheck[] } {
  const details = tech.prerequisites.map(prereq => {
    const currentLevel = researchedTechs.get(prereq.techId) ?? 0;
    return {
      techId: prereq.techId,
      requiredLevel: prereq.minLevel,
      currentLevel,
      met: currentLevel >= prereq.minLevel,
    };
  });

  return {
    met: details.every(d => d.met),
    details,
  };
}
```

### 9.11 Валидация DAG (при загрузке данных)

```typescript
  const techIds = new Set(technologies.map(t => t.id));

  for (const tech of technologies) {
    for (const prereq of tech.prerequisites) {
      // Проверка существования
      if (!techIds.has(prereq.techId)) {
        errors.push(`${tech.id}: prerequisite ${prereq.techId} does not exist`);
      }
      // Проверка самозависимости
      if (prereq.techId === tech.id) {
        errors.push(`${tech.id}: self-dependency detected`);
      }
    }
  }

  // Проверка циклов (DFS)
  const visited = new Set<string>();
  const inStack = new Set<string>();

  function dfs(techId: string): boolean {
    if (inStack.has(techId)) return true; // цикл найден
    if (visited.has(techId)) return false;
    
    visited.add(techId);
    inStack.add(techId);
    
    const tech = technologies.find(t => t.id === techId);
    if (tech) {
      for (const prereq of tech.prerequisites) {
        if (dfs(prereq.techId)) {
          errors.push(`Cycle detected involving ${techId} and ${prereq.techId}`);
          return true;
        }
      }
    }
    
    inStack.delete(techId);
    return false;
  }

  for (const tech of technologies) {
    dfs(tech.id);
  }

  return errors;
}
```

### 9.12 Расчёт уровня ветки (branch_level)

```typescript
function getBranchLevel(
  branchId: BranchId,
  researchedTechs: Map<string, number>,
  allTechs: Technology[]
): number {
  return allTechs
    .filter(t => t.branch === branchId)
    .reduce((sum, t) => sum + (researchedTechs.get(t.id) ?? 0), 0);
}
```

## Прогрессия по этапам игры

### Ранняя игра (0–50 000 RP)

**Доступные технологии**: P1–P3, M1–M4, W1–W5, C1–C5, B1–B4, X1

**Типичная стратегия**:
- Фокус на M1 (сталь) → M2 (лёгкие сплавы) → базовое производство
- W1 → W3 (ракеты) для защиты от ранних угроз
- C1 → C2 (датчики) для обнаружения ресурсов
- P1 → P2 (ионный двигатель) для первых кораблей

### Средняя игра (50 000–500 000 RP)

**Доступные технологии**: P1–P8, M1–M8, W1–W9, C1–C9, B1–B9, X1–X6

**Ключевые развилки**:
- **Варп-двигатель (P5)** vs **Фотонный двигатель (P8)** — разные стили перемещения
- **Терраформирование-β (B7)** vs **Биополимеры (B9)** — расширение или укрепление
- **Пушка Гаусса (W6)** vs **Торпеды (W8)** — разные боевые доктрины
- Начало ксеноархеологии: X1–X4 для поиска руин

### Поздняя игра (500 000–5 000 000 RP)

**Доступные технологии**: Все, включая терминальные

**Ключевые цели**:
- Антиматериальный реактор (P7) → Прыжковый двигатель (P9)
- Квантовые вычисления (C10) → мультипликатор RP
- Терраформирование-γ (B10) → Панспермия (B11)
- Тайна Предтеч (X12) → разблокировка терминальных технологий

### Сверхпоздняя игра (5 000 000+ RP)

- Гипердвигатель (P12) — мгновенные прыжки через галактику
- Сверхоружие (W12) — планетарное разрушение
- Сверхразум (C12) — полная автоматизация
- Трансгуманизм (B12) — симбиоз с ИИ
- Все терминальные технологии на максимальных уровнях
